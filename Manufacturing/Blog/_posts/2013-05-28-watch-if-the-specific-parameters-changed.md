<p>By <a href="http://adndevblog.typepad.com/manufacturing/xiaodong-liang.html" target="_self">Xiaodong Liang</a></p>  <p>In default, if the option [<strong>Don’t run automatically</strong>] is not checked,&#160; the iLogic rule will execute automatically, updating with the newest parameters value when the parameters changed. </p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b019102aa0362970c-pi"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="image" border="0" alt="image" src="/assets/image_621233.jpg" width="383" height="314" /></a> </p>  <p></p>  <p>Or, you could specify which rule needs to execute by iLogic Trigger.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b019102aa03a6970c-pi"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="image" border="0" alt="image" src="/assets/image_1a50d6.jpg" width="409" height="370" /></a> </p>  <p></p>  <p>But, sometimes, you want only to execute the rule when the specific parameter(s) changed. The two ways above cannot help. </p>  <p>The ModelingEvents.OnParameterChange of Inventor API can know which parameter changed, but it means you would have more effort to work with Inventor events, which may be a challenge to some iLogic users. </p>  <p>Here is one workaround: </p>  <p>You could provide one global file (say &quot;GlobalV.txt&quot;)&#160; to store the last values of parameters, e.g. assume we have two parameters: Width and Length. When parameter(s) changed,&#160; the rule runs. It will check the values in &quot;GlobalV.txt&quot; with current values of parameters. If only Width changed, run the corresponding workflow. If Length changed, run the other corresponding workflow.
