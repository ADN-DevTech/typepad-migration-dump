<p>By <a href="http://adndevblog.typepad.com/manufacturing/xiaodong-liang.html" target="_self">Xiaodong Liang</a></p>  <p>In default, Inventor has provided a bunch of translators of various formats such as *.dwg, *.dxf, *.sat etc. Each translator is actually an add-in which manages the Open or Save Copy As of the corresponding file format. e.g. when you open a *.sat file, the translator add-in of SAT will handle the open process to translate the file data to the geometries of Inventor. When you save copy as an Inventor file, the translator add-in will handle the save process to save geometies to the file.</p>  <p>The translator add-ins are grouped at the [Translator] tab of AddIn Manager. While what we normally create is a kind of standard add-in which is listed at [Applications] tab.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01a73d7c82e0970d-pi"><img alt="clip_image001[4]" border="0" height="269" src="/assets/image_cf235e.jpg" style="background-image: none; border-right-width: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="clip_image001[4]" width="351" /></a></p>  <p>You can find their *.addin file at:    <br /><em>\ProgramData\Autodesk\Inventor ***\Addins\. </em></p>  <p>The *.dll are deployed at    <br /><em>&lt;Inventor Installation Path&gt;\bin\</em></p>  <p>I believe you have known API allows you to drive the default add-in to customize the Open/Save options to achieve the specific requirements.</p>  <p>API also allows you to create custom translator add-in, by which you can resolve custom file format to the geometries in Inventor, or conversely.</p>  <p>The basic structure of the stardard add-in and translator add-in are same:</p>  <p>1. It is a class library (*.dll )&#0160; <br />2. A *.X.Manifest file is used to build with the *.dll&#0160; <br />3. A *.addin file configures the behaviors of the add-in.</p>  <p>The difference are:&#0160; <br />1.&#0160; The class of stardard add-in derives from ApplicationAddInServer, while the class of translator add-in derives from <strong>TranslatorAddInServer.</strong></p>  <p>2. Except the Activate and Deactivate methods, the class must override more methods of TranslatorAddInServer:</p>  <p>&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; <em>Open</em>: Open the data specified by the data-source.     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; SaveCopyAs: Save the document to the specified data-&#0160; <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; source.     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; <em>HasOpenOptions</em>: Gets whether the translator has&#0160; <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; options available for opening the specified data-source.     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; <em>HasSaveCopyAsOptions</em>: Gets whether the translator has     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; options available for saving the specified data-source.     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; <em>ShowSaveCopyAsOptions</em>: Show the save options for the     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; specified data-source. This method is only called if True was     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; returned from HasSaveCopyAsOptions     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; <em>GetThumbnail</em>: Obtains the thumbnail, if any, for the given     <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; data-source</p>  <p>3.
