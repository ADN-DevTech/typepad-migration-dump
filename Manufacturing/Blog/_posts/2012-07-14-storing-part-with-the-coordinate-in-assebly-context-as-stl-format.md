<p>By <a href="http://adndevblog.typepad.com/manufacturing/xiaodong-liang.html" target="_self">Xiaodong Liang</a></p>  <p><b>Issue     <br /></b>I need to export the Assembly document to a set of STL files. I traveled in assembly using SubOccurrences and Occurrences properties and store every part document as STL. This work fine but every STL is stored using its private coordinate system. I need store all of them using the same coordinate system used in assembly or alternatively to receive the matrix for every part to transform from the part coordinates to the assembly coordinates. Is there a solution for this?</p>  <p><a name="section2"></a></p>  <p><b>Solution     <br /></b>The Inventor API provides the ComponentOccurrence.transformation property that defines the matrix used to transform the component from the part context to the assembly context, this does not however provide the coordinate value used by the part, this means there is not a way to change the coordinate of the solids in the part. </p>  <p>The workaround is to take advantage of a derived part. For a sub-assembly, you would travel through its referenced files to find every leaf part and do the part derivation.
