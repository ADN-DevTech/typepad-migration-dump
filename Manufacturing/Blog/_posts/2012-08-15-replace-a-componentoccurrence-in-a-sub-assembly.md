<p>By <a href="http://adndevblog.typepad.com/manufacturing/barbara-han.html" target="_self">Barbara Han</a></p>  <p><b>Issue</b></p>  <p>I have the following structure: &quot;assembly-&gt;subassembly-&gt;part&quot; and I'm trying to replace part with another part. The part occurrence is in Edit mode when I run the ComponentOccurrence.Replace() function, but instead of replacing the part, my code adds the new part to subassembly. What is wrong?</p>  <p><a name="section2"></a></p>  <p><b>Solution</b></p>  <p>It's always worth having a look at the behaviour of the user interface.    <br />In this particular case, when the SubPart is in Edit mode, then in the user interface the &quot;Component-&gt;Replace&quot; command is not available in the context menu of the Model Browser.     <br />So, if you want to replace the subcomponent then edit its parent instead.
