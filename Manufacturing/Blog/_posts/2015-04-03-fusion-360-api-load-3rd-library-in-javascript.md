<p>By <a href="http://adndevblog.typepad.com/manufacturing/xiaodong-liang.html" target="_self">Xiaodong Liang</a></p>
<p>Fusion 360 encapsulates JavaScript and Python API that allow developers to extend the abilities of Fusion. The <a href="http://fusion360.autodesk.com/resources">API manual</a> introduces how to write and debug the script.</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b8d0f98d40970c-pi"><img alt="image" border="0" height="204" src="/assets/image_f16b20.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" width="315" /></a></p>
<p>As a developer of JavaScript, you would be wondering where the source script is and whether it could load the 3rd library.</p>
<p>In [Script and Add-Ins] dialog, click the script, Details&gt;&gt;Full Path tells where the script is.</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c76fe99a970b-pi"><img alt="image" border="0" height="311" src="/assets/image_5e9c1c.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" width="167" /></a></p>
<p>Generally, the built-in sample scripts reside in</p>
<p><em>C:\Users\kh\AppData\Roaming\Autodesk\Autodesk Fusion 360\API\JavaScript\Samples</em></p>
<p>While the custom scripts locate in</p>
<p><em>C:\Users\kh\AppData\Roaming\Autodesk\Autodesk Fusion 360\MyJS\Scripts\</em></p>
<p>you can set the Folder Location of your script when creating it:</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c76fe99e970b-pi"><img alt="clip_image002" border="0" height="244" src="/assets/image_32aa5d.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border: 0px;" title="clip_image002" width="195" /></a></p>
<p>In the older version of Fusion, you can find each ‘script’ is actually consist of two companion files: <br />- *.HTML file <br />- *.js file <br /> The HTML is the skeleton like what we develop a web program. The *.js file is what you created in Fusion. e.g. the below is a snapshot of a script named <strong>Bottle.</strong></p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c76fe9a7970b-pi"><img alt="image" border="0" height="188" src="/assets/image_777f8a.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; margin: 0px; display: inline; padding-right: 0px; border: 0px;" title="image" width="244" /></a></p>
<p>This is the HTML file of this sample:</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb0813e35a970d-pi"><img alt="image" border="0" height="182" src="/assets/image_254f13.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border: 0px;" title="image" width="428" /></a></p>
<p>Its HTML loads the necessary libraries of Fusion (in the relative path of <em>../src/adsk/Fusion</em>, and <em>Bottle.js</em> file of this sample. When we run the sample within Fusion, it will turn to the HTML to load the libraries.</p>
<p>In the latest version, Fusion provides the mechanism that a *.manifest file defines the profile of the sample such as the libraries. So 3 companion files now:</p>
<p>- *.HTML file <br />- *.js file <br />- *.manifest&#0160; file <br /><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c76fe9ab970b-pi"><img alt="image" border="0" height="177" src="/assets/image_be676a.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border: 0px;" title="image" width="244" /></a></p>
<p>The *.manifest file configures in a format of <a href="http://json.org/">Json</a>. The libraries to be loaded are defined at the node of <strong>autodeskLibraries.</strong></p>
<p><span style="font-size: xx-small;">{ <br />&#0160;&#0160;&#0160; &quot;autodeskProduct&quot;:&#0160;&#0160;&#0160; &quot;Fusion360&quot;, <br />&#0160;&#0160;&#0160; &quot;type&quot;:&#0160;&#0160;&#0160; &quot;script&quot;, <br />&#0160;&#0160;&#0160; &quot;author&quot;:&#0160;&#0160;&#0160; &quot;Autodesk&quot;, <br />&#0160;&#0160;&#0160; &quot;description&quot;:&#0160; <br />&#0160;&#0160;&#0160;&#0160; { <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; &quot;&quot;:&#0160;&#0160;&#0160; &quot;This is a test&quot; <br />&#0160;&#0160;&#0160;&#0160; }, <br />&#0160;&#0160;&#0160; &quot;supportedOS&quot;:&#0160;&#0160;&#0160; &quot;windows|mac&quot;, <br />&#0160; <span style="background-color: #ffff00;">&#0160; &quot;autodeskLibraries&quot;:</span>&#0160;&#0160;&#0160; <br />&#0160;&#0160;&#0160;&#0160; [&quot;application&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;dashboard&quot;, <br />&#0160;&#0160;&#0160;&#0160; &quot;geometry&quot;, <br />&#0160;&#0160;&#0160;&#0160; &quot;materials&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;userInterface&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;utilities&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;bRep&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;components&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;construction&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;features&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;fusion&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160; &quot;meshBody&quot;,&#0160; <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; &quot;meshData&quot;,&#0160; <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; &quot;sketch&quot;, <br />&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;&#0160; &quot;tSpline&quot;] <br />}</span></p>
<p>now, the *.HTML is much simpler. We just need to load the *.js file only.</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b8d0f98d61970c-pi"><img alt="image" border="0" height="72" src="/assets/image_41b51a.jpg" style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border: 0px;" title="image" width="351" /></a></p>
<p>Well, let us come back to the question of this blog title: how to load the 3rd library. The answer is very straightforward. It is like what you did in a common web programming, linking it through a relative path or the path on internet.</p>
<p>However currently the libraries for web UI have not been supported in Fusion 360. In another word, we cannot create the web page. While we can still link to some libraries that are algorism related.</p>
<p>I played with a math library <a href="http://www.numericjs.com/">Numeric</a>. It allows you to get the result of&#0160; linear algebra, complex numbers, splines, ODE solver and so on. I downloaded the mini library <a href="http://www.numericjs.com/lib/numeric-1.2.6.min.js">numeric-1.2.6.min.js</a> to the same folder of my sample NewScript1. NewScript1.html loads the library. This sample gets the spline values from <a href="http://www.numericjs.com/">Numeric</a> and draws a spline in one sketch.</p>
