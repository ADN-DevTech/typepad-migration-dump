<p>By <a href="http://adndevblog.typepad.com/manufacturing/Daniel-Du.html">Daniel Du</a></p> <p>You probablay have noticed that Vault 2014 releases a new client – Thin Client with completely new UI. It is really a light one, you do not need to install anything on client side, a broswer is good enough. Thin Client performs read-access tasks in a vault, such as searching the vault and viewing file history using a web browser. If you have not noticed this, try it now, goto <a href="http://servername/AutodeskTC">http://<em>servername</em>/AutodeskTC</a> in your broswer, where <em>servername</em> is either the IP address or the name of the computer hosting Autodesk Data Management Server.</p> <p>In this post, I will introduce how to integerate Vault Explorere and Vault ThinClient. I will create a Vault Explorer plugin, which enable users to right click a file or folder in Explorer and view it in Vault ThinClient. This is a just a demonstration, you may have better idea to use this permnent link, for example, save it into your database or ERP systems. In Autodesk Vault 2014 API, KnowledgeVaultService provides functions to get/set the persistent ID of entites.&nbsp; GetPersistentIds and ResolvePersistentIds in the KnowledgeVaultService are the functions to use for converting between file IDs and the persistent IDs.&nbsp; Keep in mind that there are two types of persistent IDs for files.&nbsp; If EntPersistOpt is <b>History</b>, the Persistent ID is for a specific file version.&nbsp; If EntPersistOpt is <b>Latest</b>, the Persistent ID is for the latest version of the file.&nbsp; The caller has to know which type to use.&nbsp; There is no way to tell the type by looking at the Persistent ID itself. Another tricky part is that, the PersistentId retrieved from Vault API is not the exact id of ThinClient URL, here is the core snippet of generating url of files and folders in ThinClient: </p><pre style="font-size: 13px; font-family: consolas; background: white; color: black"><span style="color: blue">string</span>[] ids = webMgr.KnowledgeVaultService.GetPersistentIds(<br>&nbsp; VDF.Vault.Currency.Entities.<span style="color: #2b91af">EntityClassIds</span>.Files,<br>&nbsp; <span style="color: blue">new</span>&nbsp;<span style="color: blue">long</span>[] { file.Id },<br>&nbsp; Autodesk.Connectivity.WebServices.<span style="color: #2b91af">EntPersistOpt</span>.Latest);<br> <br><span style="color: blue">string</span> id = ids[0];<br>id = id.TrimEnd(<span style="color: #a31515">'='</span>);<br>url = <span style="color: blue">string</span>.Format(<span style="color: #a31515">"{0}://{1}/AutodeskTC/{1}/{2}#/Entity/<br>            Details?id=<strong>m{3}=&amp;</strong>itemtype=File"</span>,<br>&nbsp; serverUri.Scheme, serverUri.Host, currentConnection.Vault, id);</pre>
