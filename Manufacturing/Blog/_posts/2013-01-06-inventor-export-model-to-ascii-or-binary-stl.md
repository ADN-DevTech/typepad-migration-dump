<p>by <a href="http://adndevblog.typepad.com/manufacturing/vladimir-ananyev.html">Vladimir Ananyev</a></p>  <p><b>Issue</b></p>  <p>In Inventor 2013 the SaveAs method of the PartDocument by default creates binary format STL. Can I control output file format – binary or ASCII?</p>  <p><a name="section2"></a></p>  <p><b>Solution</b></p>  <p>SaveAs method to translate files is very easy to use and supports a wide variety of formats but has limited flexibility with respect to controlling the various options when doing the translation. </p>  <p>Another approach takes advantage of the fact that translators in Inventor are implemented as a special type of Add-In. All translator Add-Ins are required to support a common set of methods and properties which allows you to write a program that drives the translator. The best resource for this information is the programming help.</p>  <p>&#160;</p>  <p><strong>Related link:&#160;&#160; </strong>Good <a href="http://modthemachine.typepad.com/my_weblog/2009/01/translating-files-with-the-api.html">overview</a> on Inventor translators.&#160; <br />You may download here very useful utility developed by Brian Ekins.
