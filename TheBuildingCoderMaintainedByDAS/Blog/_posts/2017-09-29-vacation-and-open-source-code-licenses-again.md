<p>Let me take leave from you for a short vacation.</p>

<p>Before disappearing for the next ten days, I would like to take this opportunity to revisit the topic of licenses for open source software.</p>

<p>I <a href="http://thebuildingcoder.typepad.com/blog/2017/09/extentelem-and-square-face-dimensioning-references.html#3">recently repeated the importance of a license</a> when sharing source code for reuse by others and pointed to these previous discussions:</p>

<ul>
<li><a href="http://thebuildingcoder.typepad.com/blog/2016/10/how-to-create-a-new-line-style.html#4">A readme is a must</a>!</li>
<li><a href="http://thebuildingcoder.typepad.com/blog/2013/10/the-building-coder-samples-on-github.html#2">People cannot use the code unless you specify a license</a>!</li>
<li><a href="http://thebuildingcoder.typepad.com/blog/2013/10/wall-compound-layer-and-other-geometry.html#2">More on licenses</a>...</li>
</ul>

<p>I myself use the <a href="http://opensource.org/licenses/MIT">MIT License</a> for my samples, "a lax, permissive non-copyleft free software license. For substantial programs, it is better to use the Apache 2.0 license since it blocks patent treachery".</p>

<p>My samples are not substantial.</p>

<p>Just recently, after criticism of the one previosly used,
<a href="https://medium.freecodecamp.org/facebook-just-changed-the-license-on-react-heres-a-2-minute-explanation-why-5878478913b2">Facebook changed the license on React</a>,
also switching to the MIT one.
The article explains why and once again underlines the importance of this aspect and its effect on the code reuse.</p>

<p>If you want to explore open source licenses for your own product, GitHub created
<a href="https://choosealicense.com">a tool to choose an open source license</a> to
help select the right one.</p>

<p>Bye bye, enjoy the autumn (northerners) and springtime (down under).</p>
