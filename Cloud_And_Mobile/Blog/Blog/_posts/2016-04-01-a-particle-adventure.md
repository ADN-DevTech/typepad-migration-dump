<p><span style="font-family: arial, helvetica, sans-serif;">By&nbsp;<a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html" target="_blank">Philippe Leefsma</a> <a href="https://twitter.com/F3lipek" target="_blank">(@F3lipek)</a></span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;It all started with my interest for physic based simulations: since I'm a programmer&nbsp;I always found it so cool to write computer programs where objects&nbsp;are ruled by the laws of physics!</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp; &nbsp; This time I wanted to play with a <a href="https://en.wikipedia.org/wiki/Particle_system" target="_blank">particle system</a> and see how it could perform in the browser, trying to integrate it into our viewer and have some fun with it. Browsing through the web for some existing samples, I found <a href="http://jarrodoverson.com/static/demos/particleSystem/" target="_blank">this cool demo</a>&nbsp;from <a href="https://github.com/jsoverson" target="_blank">Jarrod Overson</a>&nbsp;and decided to use it as starting point.</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Although working nicely,&nbsp;Jarrod's demo is&nbsp;2D and a bit too tied to his UI implementation for my needs. My first steps were to add a third dimension to the&nbsp;simulation&nbsp;and also decouple&nbsp;the particle system from the UI so it only takes care about managing the physic and&nbsp;becomes easily pluggable with any other web&nbsp;viewing solution - A.K.A. View &amp; Data API.</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">The&nbsp;system has four&nbsp;main components:</span></p>
<ul style="list-style-type: disc;">
<li>Particle objects which hold each&nbsp;particle properties (position, velocity and so on...)</li>
</ul>
<ul style="list-style-type: disc;">
<li>Particle emitters which are initializing the particle properties upon emission</li>
</ul>
<ul style="list-style-type: disc;">
<li>Magnetic&nbsp;fields which are imposing their force to the particles at each time step</li>
</ul>
<ul style="list-style-type: disc;">
<li>Finally the particle system itself which acts as a manager, owning all the objects, it is responsible for updating them and filtering out particles if they go out of defined bounds or their lifetime expire.&nbsp;</li>
</ul>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;My other enhancement to the initial code is&nbsp;a recycling mechanism for the particles: in my&nbsp;sample each particle is referencing&nbsp;a THREE.Mesh in the scene to represent it.&nbsp;It would be too memory intensive&nbsp;to add a mesh to the scene every time a new particle is emitted. So once an existing particle is filtered out, it is pushed to a recycle stack, the mesh is hidden from the scene and when the system needs a new particle, it will first pop out a recycled particle. Only if the recycled bin is empty, a new particle and mesh are created.</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;A side note on performance and meshes: the View &amp; Data API renderer does not yet support the <strong><em>THREE.PointCloud</em></strong>&nbsp;object (formerly named <em>THREE.ParticleSystem</em>) but the development team&nbsp;does have that&nbsp;in the pipe so it will be interesting to migrate the demo to use point cloud once it's available! In the meantime you can take a look at what is already achievable using THREE.js with the cool&nbsp;<a href="http://potree.org/" target="_blank">potree</a>&nbsp;project.&nbsp;</span></p>
<p>&nbsp;&nbsp;&nbsp;&nbsp;The first basic demo I created using that particle system is available <a href="https://lmv-react.herokuapp.com/embed?id=56fa69306a97dfd407dcda0c&amp;extIds=Viewing.Extension.Particle" target="_blank">there</a>&nbsp;-&nbsp;Running on a free Heroku dyno, you may have to reload twice if the first request gets timed out. It's a simple scene with one emitter and two fields so you can play around.</p>
<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c82c5e01970b-pi" style="display: inline;" target="_blank"><img class="asset  asset-image at-xid-6a0167607c2431970b01b7c82c5e01970b image-full img-responsive" title="Screen Shot 2016-03-30 at 23.22.23" src="/assets/image_2e680d.jpg" alt="Screen Shot 2016-03-30 at 23.22.23" border="0" /></a></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Like Jarrod, I used a library named&nbsp;<a href="http://workshop.chromeexperiments.com/examples/gui/#1--Basic-Usage" target="_blank">dat.GUI control</a>&nbsp;from <a href="http://workshop.chromeexperiments.com/" target="_blank">Chome Experiments</a> in order to create the&nbsp;interfaces to tweak the simulation properties. In the viewer you can select emitters or fields&nbsp;and specific properties are appended to a&nbsp;panel dynamically. I found the dat.GUI control pretty slick and really easy to use! Highly recommended it if you need an interface in your project to modify a JavaScript object and don't want to spend hours coding the UI.</span></p>
<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c82c5e32970b-pi" style="display: inline;" target="_blank"><img class="asset  asset-image at-xid-6a0167607c2431970b01b7c82c5e32970b img-responsive" title="Screen Shot 2016-03-30 at 23.24.24" src="/assets/image_3f3383.jpg" alt="Screen Shot 2016-03-30 at 23.24.24" width="328" height="552" /></a></p>
<p>&nbsp;</p>
<p>&nbsp;&nbsp;<span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;I was happy with that sample and didn't plan much modifications, however it caught the attention of a group of scientists at the <a href="http://home.cern" target="_blank">CERN</a> working on the <a href="http://home.cern/topics/large-hadron-collider" target="_blank">LHC</a> particle accelerator (Large Hadron Collider),&nbsp;one of the most famous and successful scientific experiments of all time, also the&nbsp;biggest&nbsp;machine on earth.&nbsp;</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Running a single experiment&nbsp;in the LHC costs thousands of&nbsp;dollars, as well as&nbsp;several days of calibration and once finished equally as long to analyze the results. This team, called <em><strong>LHC-C</strong></em> (stands for&nbsp;<em>LHC - Cloud</em>) is investigating new cutting edge&nbsp;ways to simulate particle acceleration using cloud technologies. This could not only save a huge amount of preparation&nbsp;time,&nbsp;money but also reduce the risks of producing an <a href="https://www.youtube.com/watch?v=BXzugu39pKM" target="_blank">earth-wiping blackhole</a>!&nbsp;<a href="https://www.youtube.com/watch?v=BXzugu39pKM" id="LPlnk634638" target="_blank"></a>&nbsp;</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;They asked if View &amp; Data was up for that task and if we could create a demo project based on the particle engine that would simulate such experiments. I happily went&nbsp;to spend three&nbsp;days with the <em>LHC-C</em> team in order to gather requirements in the field, unfortunately not allowed to take any picture of the premises as only official ones are allowed.</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp; &nbsp; I&nbsp;came up with a pretty convincing demo that you can run from <a href="https://lmv-react.herokuapp.com/embed?id=56f1d7c19344b52810bac677&amp;extIds=Viewing.Extension.Particle;_Viewing.Extension.Particle.LHC" target="_blank">that link</a> and looks as below:</span></p>

<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp; &nbsp; Since this one is just a proof of concept, the CERN allowed me to share publicly the source code, so you can take a look at it there:&nbsp;<a href="https://github.com/Developer-Autodesk/library-javascript-viewer-extensions/tree/master/src/Viewing.Extension.Particle" target="_blank">Viewing.Extension.Particle</a></span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Because it involves several files, I added a build step to my workflow&nbsp;using <a href="https://webpack.github.io" target="_blank">webpack</a>, a fantastic tool that is able to bundle all the&nbsp;.js and .css into a single extension file that can be loaded by the viewer. Also highly recommended if you work on medium to large web projects!</span></p>

<p>&nbsp;</p>
<p><span style="font-family: arial, helvetica, sans-serif;"><strong>The outcome:</strong></span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;The <em>LHC-C</em> team started to use more heavily View &amp; Data API but they needed more than just viewing or gathering particle collisions results. They deal with massive amount of data and also require huge computing power to simulate those experiments... which both sounded like a good candidate for processing in the cloud!&nbsp;</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Looking at this success Autodesk headquarters decided to create a new branch in our expending web-services&nbsp;collection, A.K.A. <a href="http://developer.autodesk.com/" target="_blank">Forge</a> and this is how the <strong><em>Autodesk&nbsp;Physics Web&nbsp;API</em>&nbsp;</strong>was created.&nbsp;</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">&nbsp;&nbsp;&nbsp;&nbsp;Since couple of months, a dedicated team of physicists and programmers are working to the ground&nbsp;to get the<em> Autodesk Physics API</em> ready for June 15, launch of our <a href="http://forge.autodesk.com/conference/" target="_blank">Forge conference</a>&nbsp;in&nbsp;San Francisco.&nbsp;This API is said&nbsp;to&nbsp;open a whole new perspective&nbsp;in how particle experiments are are conducted allowing for cheaper, faster and better analysis of the results.</span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">The first API to be unveiled will be&nbsp;<em><strong>/collide:</strong></em></span></p>
<p><span style="font-family: arial, helvetica, sans-serif;"><em><strong>POST -&nbsp;https://developer.api.autodesk.com/physics/v1/collide</strong></em></span></p>
<p><span style="font-family: arial, helvetica, sans-serif;">A REST POST endpoint&nbsp;where a client can send&nbsp;a payload describing two particles colliding at near light-speed velocity&nbsp;and returning a set of&nbsp;new particles being generated as a result of the collision in JSON&nbsp;format.&nbsp;</span></p>
