<p style="text-align: left;">Here is a quick post that demo a new feature added to the viewer client JavaScript API: viewer states. A viewer state holds information about the currently loaded model, the visible/isolated components, viewport information and rendering options.</p>
<p style="text-align: left;">Here is a basic way to use the API: save current state, do some modification and restore the state if needed:</p>

Finally you notice the third parameter of "restoreState". This is used if you want to have a smooth transition while restoring the view. If immediate = false there will be a transition. <strong><u>However a word of caution concerning the transition: at the time of my testing renderOptions needs to be false in the state filter in order for the transition to work.</u></strong> If you restore the renderOptions, you wont have the view transition or a workaround would be to do a two-step approach: restore view without renderOptions then restore renderOptions or the other way around...
