<p>By <a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p>Let’s continue the implementation of our cloud based viewer. Today we are going to add RESTful support to our service. Thanks to WCF, this is going to be very straightforward!</p>  <p>There are number of reasons why you would like to provide a <a href="http://en.wikipedia.org/wiki/Representational_state_transfer">REST</a> API from your web services. Nowadays most of the public web services will provide REST access to their clients, so it is very popular, if not the dominant web service design model. Main advantages of REST are the following:</p>  <ul>   <li>Lightweight – heavy xml formatting of SOAP removed</li>    <li>Human Readable Results </li>    <li>Easy to build - no toolkits required</li>    <li>Supported by a broad range of clients – mobile platforms, web browsers, …</li> </ul>  <p>The data format we will use for our REST API is <a href="http://www.json.org/">JSON</a>, a lightweight data-interchange format based on a subset of the <a href="http://javascript.crockford.com/">JavaScript Programming Language</a>. </p>  <p><strong>I – Adding REST support to a WCF Service</strong></p>  <p>The WCF Framework makes it very easy to support REST, all we have to do is prefixing the declaration of our <em>OperationContract</em> methods with the <em>WebInvoke</em> attribute and set the appropriate fields.</p>  <p>I created a new Interface that exposes our two methods in order to access available cloud models and retrieve data for a given modelId.
