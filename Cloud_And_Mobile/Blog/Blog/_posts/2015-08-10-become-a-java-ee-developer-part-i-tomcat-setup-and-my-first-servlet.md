<p>By&nbsp;<a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html">Philippe Leefsma</a></p>
<p>Being a complete newbie when it comes to web development in Java, I decided to take a step forward and build some expertise around JEE servers and web services. Another motivation is that we have some web technology coming up at Autodesk - more about that when it will turn public - that can be leveraged by, among other things, Java web services, so it makes sense to become a bit more familiar with it.&nbsp;</p>
<p>I'm starting this series of posts where I will report my progresses as I learn along the path. To get started, let's take a look at running a local Tomcat server and deploy our first Java servlet.</p>
<p>Prerequisites:</p>
<p>- <a href="http://www.oracle.com/technetwork/java/javaee/downloads/index.html" target="_self">Java EE SDK </a>&nbsp;</p>
<p>- <a href="http://www.eclipse.org/downloads/packages/eclipse-ide-java-ee-developers/marsr" target="_self">Eclipse IDE for Java EE developers</a></p>
<p>- <a href="http://tomcat.apache.org/download-70.cgi" target="_self">Apache Tomcat 7</a></p>
<p>Let's start with installing the Tomcat server. I'm assuming you are running a Linux based system. I found that <a href="https://wolfpaulus.com/jounal/mac/tomcat7/" target="_self">this blog post</a> was very straightforward and easy to follow, so I'm simply going to copy the detailed steps, you should be able to install Tomcat on your system without pain:</p>
<ol>
<li>Download a binary distribution of the core module: apache-tomcat-7.0.47.tar.gz from&nbsp;<a title="Apache Tomcat Download Page" href="http://tomcat.apache.org/download-70.cgi" target="_blank">here</a>. I picked the tar.gz in Binary Distributions / Core section.</li>
<li>Unarchiving the archive will create a folder structure in your&nbsp;<em>Downloads</em>&nbsp;folder:<br />~/Downloads/apache-tomcat-7.0.47</li>
<li>Open to Terminal app to move the unarchived distribution to /usr/local<br /><code>sudo mkdir -p /usr/local</code><br /><code>sudo mv ~/Downloads/apache-tomcat-7.0.47 /usr/local</code></li>
<li>To make it easy to replace this release with future releases, we are going to create a symbolic link that we are going to use when referring to Tomcat (after removing the old link, you might have from installing a previous version):<br /><code>sudo rm -f /Library/Tomcat<br />sudo ln -s /usr/local/apache-tomcat-7.0.47 /Library/Tomcat<br /></code></li>
<li>Change ownership of the /Library/Tomcat folder hierarchy:<br /><code>sudo chown -R &lt;your_username&gt; /Library/Tomcat</code></li>
<li>Make all scripts executable:<br /><code>sudo chmod +x /Library/Tomcat/bin/*.sh</code></li>
</ol>
<p><a href="http://www.c-sharpcorner.com/UploadFile/fd0172/how-to-configure-and-install-apache-tomcat-server-in-windows/" target="_self">Here</a> is a tutorial that looks descent for Windows users.</p>
<p>Issue a&nbsp;<strong><em>/Library/Tomcat/bin/startup.sh</em></strong> from a terminal to start the server and open the following url in your browser:&nbsp;<em>http://localhost:8080</em></p>
<p>You should see the Tomcat homepage:</p>

<p>Let's create our first servlet: "File" &gt; "New Project" &gt; "Dynamic Web Project"</p>
<p><a class="asset-img-link" style="display: inline;" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c7bd0118970b-pi"><img class="asset  asset-image at-xid-6a0167607c2431970b01b7c7bd0118970b image-full img-responsive" title="Dynamic web project" src="/assets/image_3243b6.jpg" alt="Dynamic web project" border="0" /></a></p>
<p>Then click "New Runtime" and select the appropriate server version, fill up the "Tomcat installation directory" and create the project.</p>
