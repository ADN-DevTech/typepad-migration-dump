<p>By <a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p>Hi Folks! Today I ‘m presenting you how to create an Android client for our WCF cloud hosted viewer.</p>  <p>Let me be clear about the purpose of this post: I am NOT going to teach you how to create an Android app from scratch, neither show how to get your development environment ready or your SDK installed. If you have never programmed on that platform, I would suggest you take a day or two to go through one of the numerous tutorial you can easily find on the web in order to get you started and comeback once you know how to run a basic “Hello World” application.</p>  <p>The focus of this post is to illustrate how to perform web service calls, parse the retrieved data and exploit them in our client app. </p>  <p><strong>I – Performing REST web request from Android</strong></p>  <p>The nice thing with REST requests is that they don’t require any additional third party API, it’s all part of the native Android SDK. You need to use the <em>HttpClient.execute(HttpGet request)</em> method with correct REST address of your webservice and you should be good to go for retrieving the Json message it returns. The Android platform will however require you to perform the call asynchronously, which is anyway a good thing, so you need to perform this call from an AsyncTask.</p>  <p>In order to make it easy to use, I created the following <em>WebServiceConnector</em> class, that wraps the webservice call in that convenient utility. I added the <em>WebServiceResponseListener</em> interface to encapsulate the asynchronous response, the the clients can just listen to that event and get notified when the service call succeed or fails.
