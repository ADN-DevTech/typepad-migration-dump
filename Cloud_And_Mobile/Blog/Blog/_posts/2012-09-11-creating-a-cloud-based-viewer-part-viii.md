I decided to follow a two step process: first I will migrate the service as it was previously, that is it will still rely on the AWS API to store and retrieve the data but it will be deployed on Azure, then in the next post I will migrate the code in order to replace Amazon SimpleDb and S3 by Azure Tables and Blob services.</p>  <p><strong>I – Creating a new Azure project</strong></p>  <p>If you haven’t done it yet, you will first need to install the Azure SDK, which is rather straightforward, so I can just skip that step.
