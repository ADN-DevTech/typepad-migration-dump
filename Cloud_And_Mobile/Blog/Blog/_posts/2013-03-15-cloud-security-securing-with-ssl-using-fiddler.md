<p><a href="http://adndevblog.typepad.com/cloud_and_mobile/gopinath-taget.html">Gopinath Taget</a></p>  <p>As promised in my <a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/03/cloud-security-securing-with-ssl-rest-based-wcf-application.html">previous post</a>, I will now talk about using <a href="http://www.fiddler2.com/fiddler2/">Fiddler</a> to monitor network traffic and see the encrypted data of a <a href="http://en.wikipedia.org/wiki/Representational_state_transfer">REST</a> based web service encrypted with <a href="http://en.wikipedia.org/wiki/Transport_Layer_Security">SSL</a>.</p>  <p>When I run the REST based web service I created in my previous post (from Visual Studio 2010 in debug mode), I immediately found that Fiddler does not monitor the web service. The problem is not with Fiddler but with internet explorer and .NET framework as mentioned <a href="http://www.fiddler2.com/fiddler/help/hookup.asp">here</a> (Look at the topic: <em>Why don't I see traffic sent to <u>http://localhost</u> or <u><a href="http://127.0.0.1">http://127.0.0.1</a></u>?</em>). The workaround was to use my machine name instead of “localhost” in the URL for the REST web service.</p>  <p>When you run the Fiddler application, here is what it typically looks like:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017d41ebbfb1970c-pi"><img style="background-image: none; border-right-width: 0px; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto; padding-top: 0px" title="image" border="0" alt="image" src="/assets/image_b37972.jpg" width="487" height="357" /></a></p>  <p>The Fiddler window is divided into three major portions (apart from the menus and toolbar). The Pane on the left is where all the web service and web page URLs are listed each time a client application (typically the web browser) accesses that URL. The pane on the top right allows you to monitor the request and the data traffic from the client application to the web server:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017c37bc7c33970b-pi"><img style="background-image: none; border-right-width: 0px; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto; padding-top: 0px" title="image" border="0" alt="image" src="/assets/image_66500c.jpg" width="482" height="351" /></a></p>  <p>The pane on the bottom right allows you to monitor the responses and the data traffic from the web server to the client application:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017d41ebbfc6970c-pi"><img style="background-image: none; border-right-width: 0px; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto; padding-top: 0px" title="image" border="0" alt="image" src="/assets/image_8589d1.jpg" width="486" height="420" /></a></p>  <p>In order to make sure that my data is indeed getting encrypted with SSL, I should see what it looks without SSL. So SSL successfully encrypted my request and and response for the same web service while the browser (the client application) is still able to do its job and display the response.</p>  <p>Now, you will observe the yellow note at the top of the request pane that says “HTTPS decryption is disabled. You will find more information about this <a href="http://www.fiddler2.com/Fiddler/help/httpsdecryption.asp">here</a>.</p>
