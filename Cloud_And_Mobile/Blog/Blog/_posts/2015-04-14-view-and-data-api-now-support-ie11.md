<p>By <a href="http://adndevblog.typepad.com/cloud_and_mobile/daniel-du.html">Daniel Du</a></p>  <p>After a long time waiting, IE11 finally supports WebGL, which enables us viewing our 3D/2D models on Internet Explorer with view and data API if IE is your favorite browser or your only option for whatever reason. </p>  <p>Here are some tips to make it work. Firstly, it should be IE11, previous version does not work, and you are recommended to install the new versions automatically to get the latest patches if there are any. </p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb081d8555970d-pi"><img title="image" style="border-left-width: 0px; border-right-width: 0px; border-bottom-width: 0px; display: inline; border-top-width: 0px" border="0" alt="image" src="/assets/image_51f509.jpg" width="388" height="306" /></a> </p>  <p>But you may get “<strong>Load Error : 4</strong>”, or <strong>HTTP 401</strong> error when loading a model. For whatever reason IE does not send the “ads-token” cookie when the web request is made from a web worker thread. “ads-token”&#160; is a session cookie of view and data API, It works fine if the request is made from the main thread. Setting the session cookie toggle works around the problem. Here is how to do it in your IE11 browser setting. </p>  <p>Go to Internet Options, and switch to “<strong>Privacy</strong>” tab:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb081d8566970d-pi"><img title="image" style="border-left-width: 0px; border-right-width: 0px; border-bottom-width: 0px; display: inline; border-top-width: 0px" border="0" alt="image" src="/assets/image_8ef84f.jpg" width="353" height="450" /></a> </p>  <p>Open the Advanced Privacy Settings by clicking “<strong>Advanced</strong>” button, and check “<strong>Override automatic cookie handling</strong>” , make sure the two “<strong>Accept</strong>” options are selected and “<strong>Always allow session cookies</strong>” are checked:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb081d857a970d-pi"><img title="image" style="border-left-width: 0px; border-right-width: 0px; border-bottom-width: 0px; display: inline; border-top-width: 0px" border="0" alt="image" src="/assets/image_deb8f7.jpg" width="408" height="326" /></a> </p>  <p>Click OK to accept the changes, now you should be able to view 3D models in IE11. Here is a test site: <a title="http://checkoutmymodel.autodesk.io/" href="http://checkoutmymodel.autodesk.io/">http://checkoutmymodel.autodesk.io/</a> You can drag and drop your models to upload and view, or try <a href="http://viewer.autodesk.io/checkoutmymodel/?urn=dXJuOmFkc2sub2JqZWN0czpvcy5vYmplY3Q6Y2hlY2tvdXRtb2RlbHNidWNrZXQyMDE1MDEzMS9EcmlsbC5kd2Z4">this link</a> directly.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb081d857f970d-pi"><img title="image" style="border-left-width: 0px; border-right-width: 0px; border-bottom-width: 0px; display: inline; border-top-width: 0px" border="0" alt="image" src="/assets/image_9448a1.jpg" width="485" height="294" /></a></p>  <p>Yes, It is not good to ask users to change their browser setting, more investigation is undergoing for this issue. You may want to add a reminder banner to your site, telling your user to change their settings. Here is what I did with JavaScript and <a href="http://getbootstrap.com/">Bootstrap</a>. Please note that $('#alert_placeholder') is the element where you want to show the banner, it could be body or other HTML element, a &lt;div&gt; for example.</p>  <pre style="overflow: scroll; font-size: 13px; font-family: consolas; background: white; color: black"><p><span style="color: blue">var</span> _isIE11 = !!navigator.userAgent.match(<span style="color: maroon">/Trident\/7\./</span>);<br /><span style="color: blue">if</span> (_isIE11) {<br /> <br /> <br />&#160;&#160;&#160; <span style="color: blue">var</span> message = <span style="color: #a31515">'It seems you are using IE11, please refer <br />      </span><span style="color: #a31515">to &lt;a href=&quot;http://adndevblog.typepad.com/cloud_and_mobile<br />      /2015/04/view-and-data-api-now-support-ie11.html&quot;&gt;this <br />      blog post&lt;/a&gt; to config your browser to view this sample <br />      correctly.'</span>;<br /> <br />&#160;&#160;&#160; $(<span style="color: #a31515">'#alert_placeholder'</span>).html(<span style="color: #a31515">'&lt;div id=&quot;alertDiv&quot; <br />      class=&quot;alert alert-danger&quot;&gt;<br />           &lt;button type=&quot;button&quot; class=&quot;close&quot; <br />              data-dismiss=&quot;alert&quot; aria-hidden=&quot;true&quot;&gt;&amp;times;<br />           &lt;/button&gt;'</span> + message + <span style="color: #a31515">'&lt;/div&gt;'</span>);<br />&#160;&#160;&#160; <span style="color: blue">var</span> alert = $(<span style="color: #a31515">'#alertDiv'</span>);<br />&#160;&#160;&#160; alert.alert();<br /> <br />}</p></pre>
