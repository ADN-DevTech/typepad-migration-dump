<p>By <a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html">Philippe Leefsma</a></p>  <p>An interesting approach in building Android native apps can be achieve through the use a WebView that will execute HTML and JavaScript code: by setting the WebView to fill the screen and using a local HTML page that contains our JavaScript code, we get the ability to develop entire apps based on JavaScript!</p>  <p>Another great feature is the ability to register an interface class that can be invoked from the JavaScript code and which is implemented inside the native app, hence providing to our script the full potential of Android OS. The only limitation at the moment is that WebView doesn’t support WebGL, however it seems that it can be worked around by relying on a technology named <a href="http://www.ludei.com/tech/cocoonjs">CocoonJS</a>, something to investigate in a future post…</p>  <p>The approach is greatly inspired from Android documentation that you can find <a href="http://developer.android.com/guide/webapps/webview.html">here</a>.</p>  <p>Below is the code that allows to load an html page stored in my app assets.
