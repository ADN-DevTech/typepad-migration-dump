<p>By <a href="http://adndevblog.typepad.com/cloud_and_mobile/philippe-leefsma.html">Philippe Leefsma</a></p>  <p>In that post I am presenting a WinRT or Windows Store App project I’ve been working on for the past few weeks:</p>  <p>It is a 3D Viewer using a custom data format and web service we’ve been demoing during our last year Developer Days world wide conference.</p>  <p>The project is interesting because it deals with a various number of topics:</p>  <blockquote>   <p>- 3D programming / DirectX on WinRT platform</p>    <p>- Local Storage in sandboxed WinRT Apps</p>    <p>- Touch / Gesture API</p>    <p>- REST / JSON Web Services </p>    <p>- XAML / UI </p>    <p>- Screen Selection / Ray Tracing</p> </blockquote>  <p>DirectX is the recommended technology by Microsoft in order to achieve 3D apps or games on the WinRT platform. </p>  <p>Using DirectX from a .Net language is doable using some extra framework. Several options are available at the time of this writing, some of them are more or less in beta phase, so they aren’t fully functional yet:</p>  <blockquote>   <p>- <a href="http://sharpdx.org/">SharpDX</a></p> </blockquote>  <blockquote>   <p>- <a href="http://slimdx.org/">SlimDX</a></p> </blockquote>  <blockquote>   <p>- <a href="http://msdn.microsoft.com/en-us/centrum-xna.aspx">Microsoft XNA</a></p> </blockquote>  <blockquote>   <p>- <a href="http://anxframework.codeplex.com/">AXN</a></p> </blockquote>  <p>After a bit of research, it appeared that SharpDX was the best working solution at the moment, especially that it provides a sub-framework named Toolkit, which makes it pretty straightforward to create 3D apps by abstracting most of the DirectX complexity away. For example, if like me you aren’t a DirectX game programmer expert and don’t know much about shaders, you don’t have to worry, as using the Toolkit framework will relieve you from putting yourself in that kind of trouble.</p>  <p>Here is an example of what a simple 3D app source code looks like when using that toolkit: <a href="https://github.com/sharpdx/SharpDX/blob/master/Samples/Toolkit/Common/MiniCube/MiniCubeGame.cs">MiniCube Demo</a></p>  <p>Doing web services REST requests on WinRT is pretty straightforward, especially that you can take advantage of the great async/await feature provided by .Net Framework 4.5.</p>  <p>Here is a sample code from the app that retrieves the list of models hosted by my web service:</p>  <p style="line-height: normal;
