<p><a href="http://adndevblog.typepad.com/cloud_and_mobile/gopinath-taget.html">Gopinath Taget</a></p>  <p>Continuing from my <a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/02/cloud-security-securing-with-ssl-securing-a-wcf-application.html">previous post</a>, I wanted to find a way to actually “see” the traffic data as it is transported so I could confirm that <a href="http://en.wikipedia.org/wiki/Secure_Sockets_Layer">SSL</a> is indeed encrypting (as well as for some cheap thrills). I have heard that there are several tools for monitoring web traffic, so I decided to use one of them called <a href="http://www.fiddler2.com/fiddler2/">Fiddler</a>. Fiddler is an excellent free tool that I have known about for some time but do not have any experience with. Monitoring SSL traffic seemed like a good excuse to learn how to use Fiddler. More about that in the next post.</p>  <p>The first step though is to have a client and a web application that I can monitor. In my previous post I created a <a href="http://en.wikipedia.org/wiki/Windows_Communication_Foundation">WCF</a> web service but I did not have a client application to use that service. So the normal thing to do would be to create a .NET client application for the WCF service. Right?</p>  <p>Well, I am not normal so I decided to create another WCF service. And this service is special because it is a <a href="http://en.wikipedia.org/wiki/Representational_state_transfer">REST</a> based service. REST is a very popular technology for creating web services because it allows clients and web servers to exchange data in human readable <a href="http://en.wikipedia.org/wiki/XML">XML</a> or <a href="http://en.wikipedia.org/wiki/JSON">JSON</a> form. This means any web browser can display the data and I don’t have to create a separate client to show the data received from the web service. This is – like we say in India – Clever no?</p>  <p>I used <a href="http://www.codeproject.com/Articles/105273/Create-RESTful-WCF-Service-API-Step-By-Step-Guide">this tutorial</a> to create a simple REST based web service using WCF.
