<p>I started looking into the ObjectOverrule class, this week, to see if I could use it to prevent erasure of certain objects. My thinking was that overruling Erase() would allow me to control whether an object was erased or not, simply by my decision whether or not to super-message to the base implementation.</p>  <p>That doesn’t appear to be the case, as whether I call the base implementation or not the object gets erased. I’ll have to confirm that this is expected behaviour, but in the meantime I thought I’d share a version of my code which adds a simple ObjectOverrule to watch for erasure/unerasure of database objects in AutoCAD.
