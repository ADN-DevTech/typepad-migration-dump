<p>Thanks to all who provided input on <a href="http://through-the-interface.typepad.com/through_the_interface/2009/10/design-choices-for-the-next-plugin-of-the-month.html">this last post</a>, where I asked for guidance on how best to design the command-line interface for the Screenshot application, the proposed <a href="http://labs.autodesk.com/utilities/ADN_plugins">Plugin of the Month</a> for November. The current code is based largely on <a href="http://through-the-interface.typepad.com/through_the_interface/2009/09/taking-a-screenshot-of-a-user-selected-portion-of-a-drawing-using-net.html">this previous post</a>, but may end up being extended - in time - to include better 3D support via the technique shown <a href="http://through-the-interface.typepad.com/through_the_interface/2007/04/taking_a_snapsh_1.html">here</a>.</p>  <p>Here are some design choices that came out of the comments on the last post:</p>  <ul>   <li>From Fred Dickinson      <ul>       <li>A separate “Settings” option, allowing us to use a sub-menu for our application settings </li>     </ul>   </li>    <li>From MJohnston      <ul>       <li>Display of the last used options when the command launches </li>        <li>The option to change the background colour to white for the capture </li>     </ul>   </li>    <li>From Mark Dubbelaar      <ul>       <li>Some nice design choices around command and options naming, as well as settings placement </li>        <li>The option to select a group of objects for which to capture the extents </li>     </ul>   </li> </ul>  <p>And an honourable mention goes to Chris Bray, Nikolay Poleschuk and James Meading, who made additional suggestions and suggested possible future directions. Thanks, all of you!
