<p>There was an interesting coincidence, earlier in the week. A blog comment came in on an old post on the exact day there was an internal email discussion on the same topic.</p>  <p><a href="http://through-the-interface.typepad.com/through_the_interface/2011/11/generating-preview-images-for-all-blocks-in-an-autocad-drawing-using-net.html" target="_blank">The post</a> was about using the BLOCKICON command on a document that gets loaded into the editor just for that purpose, generating a thumbnail image on disk for each of the contained block definitions. The problem with this approach – as was suggested by a couple of people in blog comments – is that the generated thumbnail is really small at just 32 x 32 pixels.</p>  <p>The exact same question – and concern – came up on an internal email discussion and the following solution was suggested by Kun Qian from our Shanghai office: to use CMLContentSearchPreviews.GetBlockTRThumbnail(BlockTableRecord blockTR) to generate them, instead.</p>  <p>I was instantly intrigued, as I hadn’t come across this function before (and thought it might be flagged as internal-only), but – sure enough – it’s part of the public .NET API, in the Autodesk.AutoCAD.Windows.Data namespace.</p>  <p>Here’s an adapted version of the C# code in the older post that allows us to compare and contrast the two techniques:</p>  <div style="font-family: courier new;
