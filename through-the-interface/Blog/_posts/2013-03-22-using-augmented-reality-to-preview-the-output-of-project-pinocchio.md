Project Pinocchio is an online character generator that helps you <em>“create, customize, and download your very own rigged 3D characters from a catalog of over 100 body types, outfits, hairstyles, and physical attributes in a few simple steps.”</em></p>  <p><em><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2017ee94beb96970d-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Project Pinocchio" border="0" alt="Project Pinocchio" src="/assets/image_933898.jpg" width="474" height="206" /></a></em></p>  <p>Once you’ve signed in, it’s a simple process to create a character. Project Pinocchio exports to .MB and .FBX formats. We want to get to .OBJ to load inside PointCloud Browser, so we’ll go for .FBX and make use of <a href="http://usa.autodesk.com/adsk/servlet/pc/item?siteID=123112&amp;id=10775855" target="_blank">a convertor to get to .OBJ</a>.</p>  <p>Here are the export options I chose:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2017d41d80cb2970c-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="And we publish" border="0" alt="And we publish" src="/assets/image_186746.jpg" width="390" height="293" /></a></p>  <p>Conversion to .OBJ is straightforward via the FBX convertor tool for Mac or Windows:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2017d41d80cce970c-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="FBX Convertor options" border="0" alt="FBX Convertor options" src="/assets/image_778140.jpg" width="394" height="278" /></a></p>  <p>Now we have an .OBJ it needs to be hosted on a web-server along with the .MTL material definition file and the corresponding texture map. One quirk I found: the version of PointCloud Browser I used crashed with a .JPG texture map, so I converted it to .PNG and adjusted the .MTL (which is a simple text format) to point to that, instead.</p>  <p>Here’s the texture map extracted from the .FBX by the conversion process (and if you think this looks freaky, try exporting without clothes: it looks <em>really</em> <a href="http://en.wikipedia.org/wiki/The_Silence_of_the_Lambs_(film)" target="_blank">Silence of the Lambs</a>):</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2017c37a8b779970b-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Texture map" border="0" alt="Texture map" src="/assets/image_752193.jpg" width="319" height="319" /></a></p>  <p>Converting to PNG stopped the crash but didn’t result in the material being applied properly, so we still get something that isn’t completely satisfactory (see below for the results).
