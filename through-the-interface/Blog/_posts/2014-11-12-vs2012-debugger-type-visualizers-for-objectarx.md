<p>The C++ developers among you may remember the <em>autoexp.dat</em> file, which tells older versions of Visual Studio how to visualize custom C++ types during a debug session. <a href="http://through-the-interface.typepad.com/through_the_interface/2006/07/advanced_visual.html" target="_blank">Here’s an ancient post</a> showing how we extended it for some basic ObjectARX types and <a href="http://through-the-interface.typepad.com/through_the_interface/2006/08/more_fun_with_a.html" target="_blank">another</a> showing how to do so via a custom plug-in.</p>  <p>In Visual Studio 2012, <a href="http://blogs.msdn.com/b/vcblog/archive/2012/07/12/10329460.aspx" target="_blank">a newer XML-based mechanism</a> was introduced to do something similar. In today’s post we’ll look at a custom <em>.natvis</em> file that exposes some basic ObjectARX types to the Visual Studio debugger.</p>  <p>This file was created by Davis Augustine in response to a query from Augusto Gonçalves. Augusto has also <a href="https://github.com/ADN-DevTech/ObjectARX-Natvis" target="_blank">posted the file to GitHub</a> – something we talked about in <a href="http://through-the-interface.typepad.com/through_the_interface/2014/11/time-to-go-git.html" target="_blank">the last post</a> – which will allow you all to contribute changes to the file, should you so wish.</p>  <p>For Visual Studio (2012 or later) to find the <em>.natvis</em> file, it has to be in one of these locations:</p>  <p><em>%VSINSTALLDIR%\Common7\Packages\Debugger\Visualizers</em></p>  <p><em>%USERPROFILE%\My Documents\Visual Studio 2012\Visualizers</em></p>  <p>[The first requires admin rights and the second should have the <em>2012</em> changed to the appropriate number for newer versions of VS, of course.]</p>  <p>Here’s a recent version of the <em>acad.natvis</em> file, reformatted for this blog. This version supports AcString, AcArray, AcRxClass, AcString, CAdUiPathName, CAdUiVolumeDescriptor and resbuf, with future candidates being AcRxValue, AcDbObject, AcDbObjectId, AcGe*, etc. You’ll continue to find <a href="https://github.com/ADN-DevTech/ObjectARX-Natvis/blob/master/acad.natvis" target="_blank">the latest &amp;
