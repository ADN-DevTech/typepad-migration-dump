<p>Over the weekend, I had more fun exploring the use of Kinect with AutoCAD. It was prompted by an email I had from a UK-based creative team who are interested in the potential of capturing <a href="http://en.wikipedia.org/wiki/Time-lapse" target="_blank">time-lapse</a> point clouds using Kinect. They were curious whether the quality of data coming from the Kinect device would be adequate for doing some interesting <a href="http://en.wikipedia.org/wiki/Trompe-l%27%C5%93il" target="_blank">trompe l’oeil</a> video compositions.</p>
<p>I started by taking the code from <a href="http://through-the-interface.typepad.com/through_the_interface/2011/04/enhanced-kinect-integration-for-3d-polyline-creation.html" target="_blank">last week’s Kinect post</a>: I removed the code related to gesture detection and beefed up the point-cloud related implementation to deal with composite point clouds that are built up over time.</p>
<p>Some notable changes:</p>
<ul>
<li>Given the potential number of points generated by a sequence of captures, it made sense to limit the volume to within a specific bounding box 
<ul>
<li>The KINBOUNDS command sets a clipping volume from the 3D extents of a selected object</li>
<li>We use LINQ (similar to the technique shown in <a href="http://through-the-interface.typepad.com/through_the_interface/2011/01/a-couple-of-streamlined-solutions-for-autocad-using-linq.html" target="_blank">this previous post</a>) to filter the points</li>
</ul>
</li>
<li>Our jig, used by the KINSNAPS command, now has a timer to measure the appropriate delay between snapshots
<ul>
<li>It shows the snapshot history in ascending colour indeces, to make them stand out from each other</li>
<li>As we now have fewer points to display we can display more of them
<ul>
<li>We’re now showing 50% – rather than 10% – of the points inside our bounding box</li>
</ul>
</li>
</ul>
</li>
<li>The points from the various capture get combined into a single LAS/PCG file
<ul>
<li>A design decision – they could easily have been kept separate</li>
</ul>
</li>
</ul>
