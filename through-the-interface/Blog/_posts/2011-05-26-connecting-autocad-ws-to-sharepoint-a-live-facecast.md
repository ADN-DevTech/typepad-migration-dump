<p>It’s been a while since I’ve talked about <a href="http://autocadws.com" target="_blank">AutoCAD WS</a>, even though the application is proving hugely popular on its various platforms: whether inside a web browser or on iOS and Android devices.</p>  <p>There have been some really interesting capabilities exposed, lately, which are making it increasingly interesting to the development community. One, in particular, that I’d like to talk about today is the ability for AutoCAD WS to connect to sites supporting the <a href="http://en.wikipedia.org/wiki/WebDAV" target="_blank">WebDAV</a> protocol. I’ve used WebDAV a number of times, over the years, mainly with ProjectPoint and SharePoint. It’s certainly possible to <a href="http://learn.iis.net/page.aspx/350/installing-and-configuring-webdav-on-iis-7/" target="_blank">set up your own WebDAV server</a> (thanks to Gopinath Taget for the link), and then <a href="http://www.autocadws.com/tutorials/connect-to-autocad-ws-with-sharepoint-or-other-windows-server/" target="_blank">connect it to AutoCAD WS</a>, but I’m going to focus on the simple steps needed to link AutoCAD WS with an existing SharePoint site.</p>  <p>Firstly, though, let’s discuss the idea behind this capability. You can now store your DWG files in a central repository – albeit one that is in some way accessible via <a href="http://en.wikipedia.org/wiki/Https" target="_blank">secure HTTP</a> over the web – and AutoCAD WS can then open the DWGs directly from it. In the past I’ve used SharePoint behind our corporate firewall, but as it is now hosted elsewhere it’s really simple to set up AutoCAD WS to work with it.</p>  <p>Once you’ve logged into AutoCAD WS you need to select “Drawings” in the left-sash and then the “Connect” button at the top:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e20154328dc150970c-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="Entering the SharePoint information" border="0" alt="Entering the SharePoint information" src="/assets/image_281743.jpg" width="404" height="241" /></a></p>  <p>And with any luck it should just work:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2014e88ae4b26970d-pi" target="_blank"><img style="background-image: none;
