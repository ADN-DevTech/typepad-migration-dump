<p>A friend in our Product Support team in Neuchâtel asked me to help develop a little application to sort drawing files on disk. He has some old <a href="http://en.wikipedia.org/wiki/Clipper_(programming_language)" target="_blank">Clipper</a> scripts that he used during a previous life as a CAD Manager to sort drawing files into different folders based on their version (held in the first six bytes of the DWG file, as mentioned in <a href="http://through-the-interface.typepad.com/through_the_interface/2006/08/breaking_it_dow.html" target="_blank">this previous post</a> and <a href="http://autodesk.blogs.com/between_the_lines/autocad-release-history.html" target="_blank">this post of Shaan’s</a>). We decided I’d implement it in VB.NET, to make it easier for him to understand and extend.</p>  <p>The application is simple enough, but could make a mess of your file system: we started with just a “Copy&quot; function but decided to extend it to cover “Move”. These functions place files in sub-folders of the selected root folder named after the version numbers of the drawings encountered:</p>  <p><font face="Courier New">DWGs folder</font></p>  <p><font face="Courier New">&#160; |_ R11</font></p>  <p><font face="Courier New">&#160; |_ R14</font></p>  <p><font face="Courier New">&#160; |_ 2000</font></p>  <p><font face="Courier New">&#160; |_ 2004</font></p>  <p><font face="Courier New">&#160; |_ 2007</font></p>  <p><font face="Courier New">&#160; |_ 2010</font></p>  <p>The “List” process parses sub-folders until it encounters an exception, at which point it defaults to listing only the drawings in the root folder. The DWGs from the various folders all then get either copied or moved into one of the version folders under the root.</p>  <p>Because of the potential for harm to folder hierarchies brought by introducing “Move”, I’ve decided to leave the code in source form only (and <a href="http://through-the-interface.typepad.com/files/SortOnVersion.zip" target="_blank">here it is</a>). Which means someone needs to build it to make use of it, and assuming they can do that then they can presumably take responsibility for whatever the code does to their (or their users’) file system(s). ;-)</p>  <p>Here’s the main VB.NET file:</p>  <div style="font-family: courier new;
