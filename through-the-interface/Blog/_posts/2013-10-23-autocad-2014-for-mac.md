Here’s some info on this release from my friend Micah, the AutoCAD for Mac Product Manager:</p>  <br /><center><iframe height="338" src="//www.youtube.com/embed/cUSmxdVIimw?rel=0" frameborder="0" width="450" allowfullscreen="allowfullscreen"></iframe></center>  <br />  <br />  <p>There's been quite a lot of press on the 2014 release already, both <a href="http://autodesk.blogs.com/between_the_lines/2013/10/autocad-2014-for-mac-is-released.html" target="_blank">from</a> <a href="http://autocadinsider.autodesk.com/my_weblog/2013/10/autocad-2014-for-mac-now-available.html" target="_blank">Autodesk</a> and beyond (<a href="http://cadpanacea.com/node/437" target="_blank">here</a> <a href="http://budweiser.cadstudio.cz/2013/10/autocad-2014-for-mac-new-apple.html" target="_blank">are</a>&#160;<a href="http://designandmotion.net/autodesk/autocad-2014-for-mac-releases-today/" target="_blank">but</a> a <a href="http://ltisacad.blogspot.com/2013/10/autocadlt-for-mac-2014-its-here-maybe.html" target="_blank">few</a> <a href="http://www.deskeng.com/virtual_desktop/?p=7691" target="_blank">examples</a> of <a href="http://architosh.com/2013/10/autodesk-releases-new-autocad-2014-for-mac-product-line/" target="_blank">the</a> <a href="http://architosh.com/2013/10/notes-on-the-call-autodesk-autocad-2014-for-mac-more-details/" target="_blank">coverage</a> to <a href="http://macacad.wordpress.com/2013/10/22/autocad-for-mac-2014-long-awaited-it-is-here/" target="_blank">date</a>).</p>  <p>The release schedule for our Mac version has now been synced with Apple’s annual announcements, allowing us to take advantage of the latest &amp; greatest OS X capabilities but also to be part of <a href="http://www.apple.com/apple-events/october-2013" target="_blank">the launch event</a> (however briefly: watch for AutoCAD and AutoCAD 360 around the 73.5 minute mark in the video). It’s interesting to note the <a href="http://www.apple.com/mac-pro" target="_blank">Mac Pro</a> announcement: AutoCAD must literally scream when running on that hardware.</p>  <p>I’m not using AutoCAD for Mac much, myself: its programming environments are still AutoLISP and ObjectARX – neither of which I do much with, these days – but I have hopes that we’ll get to see JavaScript running there at some point. Lee Ambrosius tells me this release has meant an update to the LISP Developer Documentation (as part of <a href="http://docs.autodesk.com/ACDMAC/2014/ENU/" target="_blank">the AutoCAD for Mac online help</a>), which is relevant to LISP developers working on Windows, too. <a href="http://hyperpics.blogs.com/beyond_the_ui/2013/10/autolisp-documentation-updates-windows-and-mac.html" target="_blank">More details over on Lee’s blog</a>.</p>
