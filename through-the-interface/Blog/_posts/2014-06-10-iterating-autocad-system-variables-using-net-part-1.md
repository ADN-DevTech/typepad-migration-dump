Hopefully someone will tell me I’ve made a mistake somewhere in my code and that it is, in fact, possible. :-)</p>  <p>The newer enumerator skips over “hidden” system variables – those with an asterisk (“*”) prefix on their name – which do get returned as part of the SystemObjects collection.</p>  <p>The SystemObjects collection does allow you to set the value of a variable, which is pretty interesting. In a future post we’ll take a look at this capability as well as comparing the performance of the two mechanisms, to see which is most efficient.</p>
