<p>As mentioned in <a href="http://through-the-interface.typepad.com/through_the_interface/2012/07/enjoyed-england.html" target="_blank">this previous post</a>, I was very keen to see how AutoCAD could be used to help streamline the process of generating what I’ve now found out can be classified as anamorphic street art. <a href="http://leonkeer.com" target="_blank">Leon Keer</a> had mentioned that the technique dates back to Leonardo Da Vinci and – sure enough – <a href="http://en.wikipedia.org/wiki/Anamorphosis" target="_blank">Wikipedia agrees</a>.</p>  <p>As part of my research, I found the original video that inspired my interest in Leon’s work, which should help put this post in context:</p>  <p><iframe height="265" src="https://www.youtube.com/embed/lm3bUEePRMU" frameborder="0" width="470" allowfullscreen="allowfullscreen"></iframe></p>  <p>To make the process reasonably realistic – and to some degree replicate the approach Leon has taken in his projects – I managed to track down <a href="http://www.turbosquid.com/3d-models/free-lego-minifigure-brick-figure-3d-model/605913" target="_blank">a free 3D model of a LEGO mini-figure</a>. It’s available in various formats, but I chose to use FBX to bring it into AutoCAD.</p>  <p>If I were generating geometry for a real project, I’d probably create the scene using a product that can effectively manipulate instances of our mini-figure before bringing the full scene into AutoCAD (which presumably means Autodesk Maya, 3ds Max or MotionBuilder).</p>  <p>The workflow for creating the art was pretty standard, all things considered. Adobe Flash is required. </object>  <p>Here’s the process used in more detail:</p>  <ol>   <li>Use FBXIMPORT to bring in the standard figure. </li>    <li>Use INSERT to create an instance of the imported block (having an elevation of –2.5 will make it look like it’s below the surface of the ground). </li>    <li>Use the ARRAY command to duplicate the block (I used a 3 x 6 grid). </li>    <li>Change the VISUALSTYLE to Realistic and make sure we’re viewing in perspective mode. </li>    <li>Use 3DORBIT to adjust the view to one you like. </li>    <li>Use FLATSHOT to generate a projection of the 3D geometry (without occluded lines) and place it anywhere. </li>    <li>Delete the flatshot block (you may want to get its name using LIST, first). </li>    <li>Set the UCS to the current View. </li>    <li>INSERT the flatshot block into this view at the middle of the 3D scene, then scale it to get it as close to the view as possible. </li>    <li>MOVE the inserted block so that it matches the view of the 3D geometry. </li>    <li>Place the flatshot block on its own layer (creating the layer using LAYER and changing the block’s layer property via PROPS or CHPROP). </li>    <li>You can use 3DORBIT to see that the block is really inserted on the view plane (and then UNDO or ZOOM P to get back to the 3D view). </li>    <li>Set the UCS back to World. </li>    <li>Use the custom SKEWSHOT command to generate a projected set of geometry from the flatshot block, adjusting the factor upwards or downwards to get a close fit. </li> </ol>  <p>And here’s the C# code defining the SKEWSHOT command:</p>  <div style="font-family: courier new;
