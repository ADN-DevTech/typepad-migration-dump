<p>This is a topic that I’ve covered to some degree in a couple of previous posts:</p>  <p><a href="http://through-the-interface.typepad.com/through_the_interface/2007/08/using-autocads-.html">Using AutoCAD's file selection dialog from .NET</a></p>  <p><a href="http://through-the-interface.typepad.com/through_the_interface/2007/03/replacing_the_o.html">Replacing AutoCAD's OPEN command using .NET</a></p>  <p>Neither focused on the question of allowing the user to select from a number of different file format filters (try saying “different file format filters” five times, quickly :-), so I thought I’d compare and contrast the two approaches in this post.</p>  <p>There are two primary mechanisms provided by AutoCAD’s .NET interface for file selection:</p>  <ol>   <li>Methods from the Editor class:      <ul>       <li>GetFileNameForOpen() </li>        <li>GetFileNameForSave() </li>     </ul>   </li>    <li>Classes in the Autodesk.AutoCAD.Windows namespace:      <ul>       <li>OpenFileDialog </li>        <li>SaveFileDialog </li>     </ul>   </li> </ol>  <p>Within the two mechanisms the open and save choices are broadly similar, the main differences being around prompting the user to overwrite (in the case of save) and the need for a file to exist (in the case of open). In this case we’ll focus on open: providing the same treatment for save is left as an exercise for the reader.</p>  <p>Here’s the C# code that compares the two approaches.
