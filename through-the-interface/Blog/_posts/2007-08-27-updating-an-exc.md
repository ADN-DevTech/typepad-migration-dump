<p>In <a href="http://through-the-interface.typepad.com/through_the_interface/2007/08/updating-an-aut.html">the last post</a> we saw some code to update an AutoCAD table linked to an Excel spreadsheet. In this post we go the other way, updating an Excel spreadsheet from a linked AutoCAD table.</p>

<p>Here's the C# code:</p>

<p>Tables with linked spreadsheets are locked by default and display this glyph when you hover over them: <img title="Locked_table" alt="Locked_table" src="/assets/locked_table_2.png" border="0" style="MARGIN: 0px" />. Before you run the code you will need to unlock the table by right-clicking the cell(s) you wish to edit:</p>

<p><a href="http://through-the-interface.typepad.com/photos/uncategorized/2007/08/27/unlocking_the_table.png"><img class="image-full" title="Unlocking_the_table" alt="Unlocking_the_table" src="/assets/unlocking_the_table.png" border="0" /></a> </p>

<p>One point to note is that the code will work even if the spreadsheet is open in Excel, but the contents will not be updated automatically - you have to close and reopen the file to see the results. And you will probably see this dialog come up twice:</p>

<p><a href="http://through-the-interface.typepad.com/photos/uncategorized/2007/08/27/overwrite_spreadsheet.png"><img class="image-full" title="Overwrite_spreadsheet" alt="Overwrite_spreadsheet" src="/assets/overwrite_spreadsheet.png" border="0" /></a> </p>

<p>For the best (most logical) results, the T2S command should really be run when the spreadsheet is not open in Excel. I expect it's possible to determine whether a spreadsheet is open in Excel from using standard file access functions in .NET (requesting exclusive access, to see whether it's possible to get it), but that's being left as an exercise for the reader (or for another day, at least :-).</p>
