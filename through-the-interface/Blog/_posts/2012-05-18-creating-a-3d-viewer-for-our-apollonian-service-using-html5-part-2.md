<p>In <a href="http://through-the-interface.typepad.com/through_the_interface/2012/05/creating-a-3d-viewer-for-our-apollonian-service-using-html5.html" target="_blank">the last post</a>, we created another, basic 3D viewer for the data from our Apollonian web-service – this time using <a href="http://en.wikipedia.org/wiki/HTML5" target="_blank">HTML5</a> via <a href="https://github.com/mrdoob/three.js/" target="_blank">Three.js</a>. In this post, we’ll extend the code to listen for keyboard events and manipulate the model according to user-input, as well as enabling feature detection of WebGL (allowing the same implementation to call into the canvas rendering code when WebGL isn’t present).</p>  <p>We’re introducing keyboard-based commands to enable zoom, rotate (meaning spin, although not continuously) and the change of levels:</p>  <ul>   <li><strong>+</strong>       <ul>       <li>Zoom in </li>     </ul>   </li>    <li><strong>-</strong>       <ul>       <li>Zoom out </li>     </ul>   </li>    <li><strong>,</strong> or <strong>%</strong>       <ul>       <li>Rotate left </li>     </ul>   </li>    <li><strong>.</strong> or <strong>/</strong>       <ul>       <li>Rotate right </li>     </ul>   </li>    <li><strong>&amp;</strong> or <strong>a</strong>       <ul>       <li>Rotate up </li>     </ul>   </li>    <li><strong>(</strong> or <strong>z</strong>       <ul>       <li>Rotate down </li>     </ul>   </li>    <li><strong>0</strong> – <strong>9</strong>       <ul>       <li>Change level (0 == 10) </li>     </ul>   </li> </ul>  <p>You’ll see some strange choices for some of the rotate keys, above: on browsers other than Chrome, the keypress event gets called with symbols representing the arrow keys (left: ‘%’ right: ‘/’ up: ‘&amp;’ down: ‘(‘). But because of <a href="http://bugs.jquery.com/ticket/7300" target="_blank">an issue with Chrome that means its doesn’t receive the keypress event for arrow keys</a>, we’re also checking for other characters, too (left: ‘,’ right ‘.’ up: ‘a’ down: ‘z”).</p>  <p>For changing levels: we might also have confirmed the change of level with the user via a message-box, but in the end I felt it wasn’t too disruptive to just do it. </p>  <ul><!--EndFragment--></ul>  <p>Before we see the code, here’s a quick demo video and <a href="http://through-the-interface.typepad.com/files/ApollonianViewerEvents.html" target="_blank">a link to the page</a>:</p>  <p>[Errata: I mistakenly said “level 5” in the below video instead of “level 10”, and I also should have reinforced that the view was really using WebGL rather than basic HTML5 Canvas.]</p> <object id="scPlayer" width="470" height="390" type="application/x-shockwave-flash"
