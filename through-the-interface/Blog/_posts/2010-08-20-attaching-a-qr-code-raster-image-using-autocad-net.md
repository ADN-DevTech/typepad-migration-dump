<p>This week I decided to get started on the <a href="http://en.wikipedia.org/wiki/QR_Code">QR Code</a> <a href="http://labs.autodesk.com/utilities/ADN_plugins" target="_blank">Plugin of the Month</a> I mentioned in <a href="http://through-the-interface.typepad.com/through_the_interface/2010/06/future-plugin-of-the-month-to-embed-qr-codes-in-autocad-drawings.html" target="_blank">this previous post</a>, and show a few interesting techniques along the way. The introductory post mentioned a few different approaches for creating/embedding the codes in a drawing. For now – as it’s by far the simplest – I’ve chosen to insert the code as a raster image. In this post we’ll show how to define and insert a raster image of a QR Code into the drawing – for now one with a hardcoded message string, placed at the origin – and we’ve evolve the code over a couple of different posts to provide more control over the contents and placement, first by using selection functions and then by implementing a jig.</p>  <p>At some point I’ll have to implement some kind of user interface for defining QR Codes: I’ll probably take inspiration for that from <a href="http://zxing.appspot.com/generator" target="_blank">the ZXing QR Code Generator</a>.</p>  <p>Here’s the C# code for this initial implementation, much of which was based on <a href="http://adn.autodesk.com/adn/servlet/devnote?siteID=4814862&amp;id=6183451&amp;linkID=4900509" target="_blank">this DevNote on the ADN site</a>:</p>  <div style="font-family: courier new;
