<p>A colleague in our Product Support team came to my desk yesterday afternoon with a problem he thought I might find interesting. He was right. :-)</p>  <p>It turns out that AutoCAD tables containing <a href="http://en.wikipedia.org/wiki/Unicode" target="_blank">Unicode</a> characters don’t export very well to <a href="http://en.wikipedia.org/wiki/Comma-separated_values" target="_blank">CSV</a> files (for later import into Excel, for instance).</p>  <p>Here’s a sample table with a number of Unicode characters in it:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201a5116c663f970c-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Cyrillic table" border="0" alt="Cyrillic table" src="/assets/image_158534.jpg" width="300" height="313" /></a></p>  <p>When exported to a CSV file using the standard right-click menu option it contains a bunch of ? characters:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201a3fcbcaf09970b-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Incorrect CSV export" border="0" alt="Incorrect CSV export" src="/assets/image_132981.jpg" width="394" height="186" /></a></p>  <p>The good news is that it’s easy to create your own version of the TABLEEXPORT command – the one that gets called when you right-click a table and select Export – that successfully writes the contents of a table to a Unicode-encoded CSV file.</p>  <p>Here’s the C# code that implements the TABLEEXPORT2 command.
