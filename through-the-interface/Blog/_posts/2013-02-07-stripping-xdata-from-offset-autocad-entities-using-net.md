<p>This question came in day or two ago:</p>  <blockquote>   <p><em>“I attach an XData to some AutoCAD entities. When the AutoCAD entity is offset by OFFSET command, the XData is cloned in the offset entity. What’s the way to control(stop) the cloning of my XData in OFFSET command?”</em></p> </blockquote>  <p>This is an interesting one. Many applications rely on External Entity Data (XData) providing unique references from AutoCAD objects to other locations, so when objects with XData attached get copied, it either needs to be removed or updated to refer to something different (an identifier to a new record in an external database table, for instance). So it certainly seemed to be a topic worth covering here.</p>  <p>Responding to the question, a colleague of mine suggested using the Copied event on the objects of interest, and then using ObjectClosed to see when the copy gets added to the database (storing its ID for later processing).</p>  <p>I gave this a try, but wasn’t able to get it working properly: I suspect the problem was related to watching for ObjectClosed on objects that hadn’t previously been added to the database (from .NET, anyway – my colleague was talking about ObjectARX).</p>  <p>After hitting my head against it for some time, I ended up choosing a simpler path: during the OFFSET command, watch for objects being appended to the database. If any get added that are entities and have XData we care about attached, then we go ahead and process them once the command is over. Which in our case means we strip rather than update the XData, but other applications may want to treat it differently.</p>  <p>As long as we’re careful about the entities we modify – i.e. we only look for our specific XData, as there may be XData that other applications are interested in and rely upon being copied – then this simpler approach should be just fine.</p>  <p>Here’s the C# code that implements this, building on the commands in <a href="http://through-the-interface.typepad.com/through_the_interface/2007/04/adding_xdata_to.html" target="_blank">this previous post</a> (which you can use to add XData to the entities you want to offset and test the existence of the XData once the OFFSET command has completed):</p>  <div style="font-family: courier new;
