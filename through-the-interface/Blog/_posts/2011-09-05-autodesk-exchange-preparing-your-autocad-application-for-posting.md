<p>Back when we were first creating the Exchange Store, I went through the process of preparing several of our <a href="http://labs.autodesk.com/utilities/ADN_plugins" target="_blank">Plugins of the Month</a> for posting to Autodesk Exchange. After a break of several months, I’ve now gone back and readied September’s plugin (<a href="http://through-the-interface.typepad.com/through_the_interface/2011/09/septembers-plugin-of-the-month-live-on-autodesk-labs-transtips.html" target="_blank">TransTips</a>, now on <a href="http://labs.autodesk.com" target="_blank">Autodesk Labs</a>) for posting, also.</p>  <p>It seemed to make sense to spend a couple of posts documenting the process I followed – which should complement the DevTV sessions already posted on <a href="http://autodesk.com/developapps" target="_blank">the Autodesk Exchange Apps Developer Center</a>:</p>  <ul>   <li><a href="http://download.autodesk.com/media/adn/DevTV_Autodesk_Exchange_Apps_User_Experience/DevTV_Autodesk_Exchange_Apps_User_Experience.html" target="_blank">Autodesk Exchange Apps User Experience</a> </li>    <li><a href="http://download.autodesk.com/media/adn/DevTV_Autoloader_Format_Explained/DevTV_Autoloader_Format_Explained.html" target="_blank">Autoloader Format Explained</a> </li>    <li><a href="http://download.autodesk.com/media/adn/DevTV_Creating_a_Partial_CUI/DevTV_Creating_a_Partial_CUI.html" target="_blank">Creating a Partial CUI</a> </li> </ul>  <p>This initial post focuses on preparing an application for submission to Autodesk for posting. The follow-up post will step through the process of getting set up as a published and submitting the application to Autodesk.</p>  <h3>Step 1 – download an existing Autodesk app to copy.</h3>  <p>Download an Autodesk app from the Store: any of the previous Plugins of the Month will do. I tend to use Spiro as a basis for mine, for no particular reason.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2014e8b478025970d-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Downloading Spiro from Autodesk Exchange" border="0" alt="Downloading Spiro from Autodesk Exchange" src="/assets/image_721974.jpg" width="448" height="320" /></a></p>  <h3>Step 2 - navigate to where it is on your hard-drive and make a copy of it.</h3>  <p>I use this handy string, pasted into Windows Explorer, to get to the folder containing downloaded application bundles (i.e. the file structures containing the application modules, etc.):</p>  <p><em>%AppData%/Autodesk/ApplicationPlugins</em></p>  <p>Then you can simply right-click drag and select “Copy here”:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201539153b369970b-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Copy the bundle to work with" border="0" alt="Copy the bundle to work with" src="/assets/image_466480.jpg" width="479" height="255" /></a></p>  <p>Rename the bundle to have the name you want (prefixing it with your <a href="http://autodesk.com/symbolreg" target="_blank">Registered Developer Symbol</a>, to avoid name clashes with other applications).</p>  <p>I’d also suggest moving the bundle out of the autoloader folder – or at least renaming it so it doesn’t end in .bundle – as we will be launching AutoCAD before we’ve finished the complete migration (and don’t want it loaded until we’re ready).</p>  <p>The plugin’s source code is there for you to look at and extend, if you wish, but you should remove it from your copy of the application bundle (i.e. delete the entire “<em>Contents\Source</em>” sub-folder).</p>  <h3>Step 3 – copy your application modules to the new bundle’s “<em>Contents\Windows</em>” folder.</h3>  <p>Navigate down through your newly renamed bundle, to find the “<em>Contents\Windows</em>” sub-folder. This is the location to which you should copy your LISP, .NET, ObjectARX, etc. application modules.</p>  <h3>Step 4 – update the <em>PackageContents.xml</em> manifest.</h3>  <p>Here’s a sample manifest for Spiro, such as the one found in the root folder of the downloaded/copied bundle. I’ve also inserted a few additional attributes to comply with standards we’ll be starting to impose soon for newly submitted applications (as well as retro-fitting to the existing applications on the store).</p>  <p>For more details on the XML format, see <a href="http://exchange.autodesk.com/autocad/enu/online-help/browse#WS1a9193826455f5ff-4fe3487312ce8a8dba5-7feb.htm" target="_blank">this online help page</a>.</p>  <div style="font-family: courier new; This is also true of the .CUIX file, although you can simply delete the .MNR and the .HTM files, which will get generated automatically later (one when we edit the .CUIX file and the other when the application submission is processed by Autodesk).</p>  <p>Use an image editor – I’m a big fan of the free <a href="http://www.getpaint.net" target="_blank">Paint.NET</a> tool, as it supports layers and provides other features that makes life easier – to update (or create) images of the same dimensions as those in the bundle. Having the existing images renamed arguably makes it easier to get the filenames right as you overwrite them, although I can see how others may find this a little redundant.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201539153b374970b-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Editing images in Paint.NET" border="0" alt="Editing images in Paint.NET" src="/assets/image_645323.jpg" width="429" height="319" /></a></p>  <p>The background I’ve used for the various plugins I’ve posted is intended to make it clear the plugins are from Autodesk. You will clearly want to have your own, unique branding.</p>  <h3></h3>  <h3>Step 6 – update the CUIX file.</h3>  <p>All applications on the Exchange Store have a consistent UI for being launched, in that upon install they add a ribbon panel (containing one or more commands) to the “Plug-ins” ribbon tab inside AutoCAD. They do this by providing a partial CUIX file containing the panel definition that is also referenced from the XML manifest.</p>  <p>We’ll use AutoCAD to update the CUIX file to work for our application. Start by using the MENULOAD command to unload the partial CUIX file we’re basing ours on (just to avoid confusion, as the one we copied will have similar internal names and IDs).</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201543527320e970c-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Use MENULOAD to unload any partial CUIX with the same name" border="0" alt="Use MENULOAD to unload any partial CUIX with the same name" src="/assets/image_986308.jpg" width="368" height="319" /></a></p>  <p>You can then use the same dialog to browse to and load our copied/renamed CUIX file.</p>  <p>Now run the CUI command, selecting the partial CUIX to edit:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201539153c32b970b-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="Selecting our partial CUIX in the CUI dialog" border="0" alt="Selecting our partial CUIX in the CUI dialog" src="/assets/image_922944.jpg" width="474" height="409" /></a></p>  <p>We can see that the CUIX itself is pretty minimal: it really only contains our custom ribbon panel – and associated command – and an entry for it on the standard “Plug-ins” tab (whose internal name is “Add-Ins”).</p>  <p>A quick comment on this tab: when the CUIX file is brought in, an attempt is made to match the internal ID of the partial tab with those of the main CUI file. If a match is found, the contents of the tab in the partial file get added to the existing one. Otherwise a new tab is created with the name of the tab in the partial file.</p>  <p>So if the “Plug-ins” tab isn’t present in one of an AutoCAD-based product’s workspace, a new tab will be created. It’s recommended to use the <em>ID</em> of the “Plug-ins” tab for the tab containing your custom panel in the CUIX file, but you should use a <em>name</em> that makes sense if the standard “Plug-ins” tab isn’t present. If everyone just calls their tab “Plug-ins”, then potentially there could end up being one “Plug-ins” tab created for each product (it’s about the tabs in the main CUI file, not about those loaded in the current workspace).</p>  <p>Therefore, in this case, I’d suggest keeping the ID as “ID_ADDINSTAB” but changing the name of the ribbon tab to something like “Language” or “Translate”.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2014e8b479018970d-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="The contents of our partial CUIX" border="0" alt="The contents of our partial CUIX" src="/assets/image_828626.jpg" width="474" height="409" /></a></p>  <p>Start by editing the <em>Name</em>, <em>Display Text</em> and <em>Aliases</em> for the panel itself (in the Properties window you can see at the bottom right of the above image).</p>  <p>Next select the button image entry (the one named “Spiro” above, the first child of “Row 1”) and edit its various properties:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2014e8b479568970d-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="Our button image properties" border="0" alt="Our button image properties" src="/assets/image_956861.jpg" width="355" height="357" /></a></p>  <p>You should change the <em>Group Name, Command Name, Description, Command Display Name, Macro</em> and <em>Small </em>&amp;<em> Large image</em> properties. Don’t be concerned about the <em>Element ID</em> in the “Advanced” category: you can edit this by selecting the SPIRO command at the lower-left of the CUI dialog:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2014e8b479033970d-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="Our custom command properties" border="0" alt="Our custom command properties" src="/assets/image_509411.jpg" width="354" height="386" /></a></p>  <p>Now you should be able to save your changes and see the results on the “Plug-ins” tab inside AutoCAD.</p>  <p>There’s a quirk in the FCS (First Customer Shipment or the original, un-patched) version of AutoCAD 2012: if the Autoloader has already loaded your bundle – but then you remove and replace it – then the partial CUIX sometimes doesn’t get loaded. This should be addressed before too long, but in the meantime you can simply unload and reload your CUIX file using the MENULOAD or CUI commands.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201539153c395970b-pi" target="_blank"><img style="background-image: none; border-bottom: 0px; border-left: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top: 0px; border-right: 0px; padding-top: 0px" title="Our custom item on the plug-ins tab inside AutoCAD 2012" border="0" alt="Our custom item on the plug-ins tab inside AutoCAD 2012" src="/assets/image_966670.jpg" width="475" height="227" /></a></p>  <h3>Step 7 – test your bundle.</h3>  <p>Move your bundle back into the <em>“%AppData%\Autodesk\ApplicationPlugins</em>” folder (or rename it to have the .bundle suffix) and launch AutoCAD, to see how the application loads and performs. For instance, make sure the conditions for loading the various modules specified in the <em>PackageContents.xml</em> file are respected (such as loading the application on invocation of your custom commands).</p>  <p>In case you hit problems, try using the APPAUTOLOADER command with the RELOAD option to reload the various application bundles: this will report when your XML is incorrect or list the settings for each application, otherwise.</p>  <p>It’s really worth testing your application thoroughly before submitting.
