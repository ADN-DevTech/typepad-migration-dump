<p>Back in March, I received an email from Thomas Fitou suggesting an interesting blog topic:</p>  <blockquote>   <p><em>I was thinking about a cool feature in jigs:</em></p>    <ul>     <li><em>You invoke a command to enter an Mtext or text</em> </li>      <li><em>The editor is asking for some text</em> </li>      <li><em>You enter the text</em> </li>      <li><em>Then a jig is dragged asking for position</em> </li>      <li><em>But in the editor appear some options:[R]egular [B]old [I]talic [R]otate 90</em> </li>      <li><em>If the user hits &quot;B&quot; the text becomes bold</em> </li>      <li><em>If the user hits &quot;R&quot; the text is rotated 90 degrees</em> </li>      <li><em>If the user hits &quot;R&quot; again, another 90 degrees and so on...</em> </li>   </ul> </blockquote>  <p>It struck me as an interesting idea, but wasn’t something I managed to get to until now. It turned out to be a pretty interesting mini-project, too – so much so that I decided to split it across 3 separate posts:</p>  <ol>   <li>A basic jig to position, rotate and size a DBText object </li>    <li>An extended version that supports font-level formatting (bold, italic, etc.) </li>    <li>For extra points: interactive alignment adjustment (left, middle, right) </li> </ol>  <p>In today’s post, then, we’re going to see a basic jig implementation that allows the user to position, rotate and adjust the size of the created text.</p>  <p>A few notes on design decisions related to this code:</p>  <ul>   <li>I decided to stick with DBText rather than including MText </li>    <li>A few <a href="http://through-the-interface.typepad.com/through_the_interface/2009/03/jigging-an-autocad-solid-using-ironpython-and-net.html" target="_blank">previous</a> <a href="http://through-the-interface.typepad.com/through_the_interface/2009/03/jigging-an-autocad-block-with-attributes-using-net.html" target="_blank">implementations</a> on this blog have made sure geometry is database-resident before jigging it       <ul>       <li>While not needed for this initial post, it is needed later on, so I’ve adopted that same approach, here </li>     </ul>   </li>    <li>The proposed implementation suggested the use of single keystrokes to enable different modes (at least that’s how I read it)      <ul>       <li>The code uses the standard keyword mechanism, which means Enter is needed to submit the mode change </li>     </ul>   </li>    <li>I’ve placed the UI paths (i.e. the keywords) in the code for the latter parts – for now they don’t do anything, though      <ul>       <li>I dropped the “Regular” keyword, with the idea that “Bold” and “Italic” should really be toggles (and “R” is already fairly busy between “ROtate90” and “RIght”, for that matter) </li>     </ul>   </li> </ul>  <p>Here’s the C# code for this initial version:</p>  <div style="font-family: courier new;
