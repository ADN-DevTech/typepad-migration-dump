<p><em><strong>Important note:</strong> the code in this post – while potentially interesting, at a certain level – has been superceded by the code in <a href="http://through-the-interface.typepad.com/through_the_interface/2011/01/combining-autocad-blocks-in-separate-files-into-a-single-dwg-using-net-take-2.html" target="_blank">the following post</a>. Please go ahead and use that simpler, more complete implementation instead.</em></p>  <p>This question came in, over the holidays, that seemed like it was worth addressing:</p>  <blockquote>   <p><em>How do I get an external DWG into the blocks table as a block definition? </em></p> </blockquote>  <p>I started by quoting <a href="http://through-the-interface.typepad.com/through_the_interface/2006/08/import_blocks_f.html" target="_blank">this ancient (at least in terms of this blog) post</a>, but it turned out only to be of partial help: the problem is that the external DWG does not <em>contain</em> the block, it <em>is</em> the block. Which is pretty common – the WBLOCK command allows you to export blocks to individual DWG files, and INSERT clearly allows you to bring them back in.</p>  <p>So how would you go about combining a set of these externally-defined blocks into a single library?</p>  <p>Let’s think this through, a little. The blocks in these external drawings are actually not defined as blocks – as such – but have their contents in the modelspace. So one way to do this would be to adjust the approach in the old post (and it really does look old now, to me – which hopefully means I’ve learned something over the last four years ;-) to use WblockCloneObjects() to bring across the modelspace from each of the source drawings into the block table of our target database (which, for simplicity’s sake, we’ll probably just take as that of the active drawing).</p>  <p>A few things to consider:</p>  <ul>   <li>The blocks will be brought it with names such as “*Model_Space0”, “*Model_Space1”, etc., so we will need to remap them      <ul>       <li>Apart from anything else, leaving them like this will make them anonymous and harder to use </li>     </ul>   </li>    <li>We should set the new name to be that of the drawing file      <ul>       <li>This is the default filename used when WBLOCKing to a DWG </li>     </ul>   </li> </ul>  <p>Here’s the updated C# code to all the blocks defined in a particular folder into the active drawing:</p>  <div style="font-family: courier new;
