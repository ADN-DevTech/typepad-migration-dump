<p><em>[This post continues from <a href="http://through-the-interface.typepad.com/through_the_interface/2007/10/au-handouts-the.html">part 1</a> and <a href="http://through-the-interface.typepad.com/through_the_interface/2007/10/au-handouts-t-1.html">part 2</a> of this series.</em><em>]</em></p>  <p><strong>Building a full editing product using AutoCAD OEM</strong></p>  <p>Now we’re going to take our AutoCAD .NET module from the beginning of the session and build a complete CAD package around it.</p>  <p>AutoCAD OEM allows you to build a custom-branded version of AutoCAD, with the subset of AutoCAD’s own features and additional functionality you choose to build in. A key feature of AutoCAD OEM is its security layer, which will stop any module being loaded that has not been built in by design. By limiting its extensibility – and allowing a subset of standard functionality to be enabled – ISV (Independent Software Vendor) partners are able to sell products based on AutoCAD OEM at a lower price-point.</p>  <p>This is, for example, how we build AutoCAD LT and DWG TrueView: they are both “cut down” versions of AutoCAD built using the AutoCAD OEM engine.</p>  <p>So let’s take a look at how to create an AutoCAD OEM product. We want to build an application that only contains AutoCAD’s 3D editing &amp; visualization functionality, with our additional application logic added, to provide some in-built analysis of our models.</p>  <p>To get started, we launch the OEM Make Wizard. This tool depends on some Visual Studio environment variables to be set, so the simplest way is to launch it from a Visual Studio 2005 Command Prompt. I typically set the working directory for the command prompt to the root AutoCAD OEM folder (in my case “C:\Program Files\Autodesk\AutoCAD OEM 2008”) and then launch the tool by typing in “toolkit\oemmakewizard”.</p>  <p>On the first page of the Wizard we specify a name for our product. We’ll use “Solids”: this short name will be used for the executable name, for instance (i.e. Solids.exe, rather than acad.exe).</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_begin_session.png"><img title="Oem_make_wizard_begin_session" border="0" alt="Oem_make_wizard_begin_session" src="/assets/oem_make_wizard_begin_session.png" width="300" height="190" /></a> </p>  <p><em>Figure 10 – starting a project with the OEM Make Wizard</em></p>  <p>On the next page we specify some additional details, such as a longer product name and version numbers. You can get more information on this from the AutoCAD OEM documentation.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_project_information.png"><img title="Oem_make_wizard_project_information" border="0" alt="Oem_make_wizard_project_information" src="/assets/oem_make_wizard_project_information.png" width="300" height="190" /></a> </p>  <p><em>Figure 11 – entering project information in the OEM Make Wizard</em></p>  <p>Next we specify the standard commands we want enabled. There are different levels of command support – None, Internal Use, Full and Redefine. Internal Use means we will call it programmatically, but don’t want the command directly usable by the user, and Redefine means we want the command to be usable but we will provide our own implementation. It should be obvious what None and Full mean. :-)</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_autocad_commands.png"><img title="Oem_make_wizard_autocad_commands" border="0" alt="Oem_make_wizard_autocad_commands" src="/assets/oem_make_wizard_autocad_commands.png" width="300" height="190" /></a></p>  <p><em>Figure 12 – specifying the AutoCAD commands to expose with the OEM Make Wizard</em></p>  <p>Now we add our custom module. Application modules need to be bound to a particular OEM product, if they are to be loaded by it. This is essentially where we tell AutoCAD OEM’s security layer to allow specific modules to be loaded. This step also lets us specify a location to copy the application module to. .NET modules don’t need to be built specially, while ObjectARX modules need to be built against a special version of the ObjectARX SDK.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_your_modules_2.png"><img title="Oem_make_wizard_your_modules_2" border="0" alt="Oem_make_wizard_your_modules_2" src="/assets/oem_make_wizard_your_modules_2.png" width="300" height="190" /></a> </p>  <p><em>Figure 13 – adding your own modules with the OEM Make Wizard</em></p>  <p>Then we need to specify the commands that our module implements – in our case it’s SD and SD2.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_your_commands.png"><img title="Oem_make_wizard_your_commands" border="0" alt="Oem_make_wizard_your_commands" src="/assets/oem_make_wizard_your_commands.png" width="300" height="190" /></a> </p>  <p><em>Figure 14 – exposing your commands with the OEM Make Wizard</em></p>  <p>To enable Automation to work from outside our application – especially to embed it as an ActiveX control, which we will do later – we need to assign GUIDs for a number of interfaces. This is handled automatically by the tool.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_automation.png"><img title="Oem_make_wizard_automation" border="0" alt="Oem_make_wizard_automation" src="/assets/oem_make_wizard_automation.png" width="300" height="190" /></a> </p>  <p><em>Figure 15 – defining an automation interface with the OEM Make Wizard</em></p>  <p>Next we customize the look &amp; feel of our application, firstly by assigning a new splash screen, about box image and background image for our application frame.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_custom_images.png"><img title="Oem_make_wizard_custom_images" border="0" alt="Oem_make_wizard_custom_images" src="/assets/oem_make_wizard_custom_images.png" width="300" height="190" /></a> </p>  <p><em>Figure 16 – specifying custom splash screen and about box images with the OEM Make Wizard</em></p>  <p>We can also choose to disable certain user interface elements, such as toolbars and dashboards. This is clearly important to do if we’re disabling standard AutoCAD functionality.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_toolbar_and_dashboa.png"><img title="Oem_make_wizard_toolbar_and_dashboa" border="0" alt="Oem_make_wizard_toolbar_and_dashboa" src="/assets/oem_make_wizard_toolbar_and_dashboa.png" width="300" height="190" /></a></p>  <p><em>Figure 17 – turning off user interface components with the OEM Make Wizard</em></p>  <p>The same applies for AutoCAD features – we can disable various parts of AutoCAD’s standard functionality from this page.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_autocad_features.png"><img title="Oem_make_wizard_autocad_features" border="0" alt="Oem_make_wizard_autocad_features" src="/assets/oem_make_wizard_autocad_features.png" width="300" height="190" /></a> </p>  <p><em>Figure 18 – turning off AutoCAD features with the OEM Make Wizard</em></p>  <p>Certain file associations used by AutoCAD are not relevant for our AutoCAD OEM application. DWG is needed, of course, but many others are not.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_file_extensions.png"><img title="Oem_make_wizard_file_extensions" border="0" alt="Oem_make_wizard_file_extensions" src="/assets/oem_make_wizard_file_extensions.png" width="300" height="190" /></a> </p>  <p><em>Figure 19 – turning off file associations with the OEM Make Wizard</em></p>  <p>Finally – before we build – we can change the standard application icons (for the frame, each document, etc.) to be more representative of our application look &amp; feel.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=636,height=404,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;); return false" href="http://through-the-interface.typepad.com/.shared/image.html?/photos/uncategorized/2007/10/22/oem_make_wizard_change_icons.png"><img title="Oem_make_wizard_change_icons" border="0" alt="Oem_make_wizard_change_icons" src="/assets/oem_make_wizard_change_icons.png" width="300" height="190" /></a> </p>  <p><em>Figure 20 – changing your application’s icons with the OEM Make Wizard</em></p>  <p>Now let’s build the application. This can be time-consuming, but the good news is that many changes do not require a full rebuild. To make life easier we’re now going to implement demand-loading. The attached Registry file, when merged, will cause our AutoCAD OEM product to load our custom module when either the SD or the SD2 command is invoked by the user:</p>  <div style="font-size: 8pt; font-family: courier new; background: white; color: black">   <p style="font-size: 8pt; margin: 0px">Windows Registry Editor Version 5.00</p>    <br />    <p style="font-size: 8pt; margin: 0px">[HKEY_LOCAL_MACHINE\SOFTWARE\Autodesk\SolidsSystem\R11\SOLIDS-6001:409\Applications\SolidsAnalyzer]</p>    <p style="font-size: 8pt; margin: 0px">&quot;DESCRIPTION&quot;=&quot;SolidsAnalyzer&quot;</p>    <p style="font-size: 8pt; margin: 0px">&quot;LOADCTRLS&quot;=dword:0000000c</p>    <p style="font-size: 8pt; margin: 0px">&quot;LOADER&quot;=&quot;C:\\Program Files\\Autodesk\\AutoCAD OEM 2008\\oem\\Solids\\solidsanalyzer.dll&quot;</p>    <p style="font-size: 8pt; margin: 0px">&quot;MANAGED&quot;=dword:00000001</p>    <br />    <p style="font-size: 8pt; margin: 0px">[HKEY_LOCAL_MACHINE\SOFTWARE\Autodesk\SolidsSystem\R11\SOLIDS-6001:409\Applications\SolidsAnalyzer\Commands]</p>    <p style="font-size: 8pt; margin: 0px">&quot;sd&quot;=&quot;sd&quot;</p>    <p style="font-size: 8pt; margin: 0px">&quot;sd2&quot;=&quot;sd2&quot;</p> </div>  <p>The LOADCTRLS value above is “C” – hexadecimal for 12. This means that we’re saying the module will be loaded either “on load request” (8) or “on command invocation” (4). We could change this value to “2” (which is the same in decimal as in hexadecimal :-) if we wanted to load the module automatically “on startup”.</p>  <p>For more information on this topic, search for “demand loading” on the <em>Through the Interface</em> blog, or type in this URL:</p>  <p><a href="http://through-the-interface.typepad.com/through_the_interface/2006/09/automatic_loadi.html">http://through-the-interface.typepad.com/through_the_interface/2006/09/automatic_loadi.html</a></p>  <p><strong>Further streamlining the user interface by embedding our AutoCAD OEM product</strong></p>  <p>For our final demonstration in this session, we’re going to embed the AutoCAD OEM product we’ve just built into a .NET application.</p>  <p>Just as we added the DWG Thumbnail and DWG TrueView control to our Visual Studio Toolbox, we can now add the ActiveX control for our solids-editing product.</p>  <p><a onclick="window.open(this.href, &#39;_blank&#39;, &#39;width=571,height=403,scrollbars=no,resizable=no,toolbar=no,directories=no,location=no,menubar=no,status=no,left=0,top=0&#39;);
