<p>I mentioned last week that <a href="https://www.keanw.com/2021/04/dynamo-core-211-and-dynamo-210-and-generative-design-in-revit-2022.html" rel="noopener" target="_blank">Dynamo 2.11 allows package owners to add additional documentation for their nodes via the enhanced Documentation Browser</a>. I decided to give it a try, and then decided to test the possibilities around automating the process.</p>
<p>Here’s the idea: the Space Analysis package comes with 15 different samples showing how to use its various nodes. These are all hidden away in the <em>%appdata%\Dynamo\Dynamo [Core|Revit]\2.x\packages\SpaceAnalysis\extra</em> folder, so lots of people don’t know they exist.</p>
<p>What if the Documentation Browser told you about the samples that use a particular node when you pull up its help?</p>
<p>Here’s a quick GIF showing you what I mean. Here we are bringing up additional help for the code that creates a <em>SpaceLattice</em> (the data structure at the core of Space Analysis, so it comes as no surprise that it’s used in every single sample ;-).</p>
<p><a href="https://through-the-interface.typepad.com/.a/6a00d83452464869e20263e99d78da200b-pi" rel="noopener" target="_blank"><img alt="Additional SpaceAnalysis help" height="297" src="/assets/image_977559.jpg" style="margin: 30px auto; float: none; display: block;" title="Additional SpaceAnalysis help" width="500" /></a></p>
<p>The beauty of the problem of creating the Markdown files that hold this documentation is that it’s actually really easy to automate.</p>
<p>Firstly we need to get a list of the various methods in the package, which are luckily available in a handy XML file that’s in the package’s <em>bin</em> folder. (Visual Studio creates this manifest as a by-product of the .NET build process.)</p>
<p>Here’s an excerpt from this file – the full one can be found in the Space Analysis package.</p>

<p>Now for the really cool bit. Because .DYN files are just JSON-formatted text, we can search through the directory of sample files for each of the above methods: if a .DYN contains the method name as a “FunctionSignature” then it’s used in that sample. We can then just create a Markdown file (with the .md suffix) for each method listing the samples that were found to contain it. Super-easy and super-useful, too!</p>
<p>I’m not saying this is the best way to do it, but I will say it’s a reasonably compact way: I wrote a “simple” Unix shell script to do the work. I’m using this a Mac, so this was an accessible environment: the point was not that other people would necessarily follow the same approach – you could do the same thing with batch files, PowerShell or even C# – but I wanted to show people that it’s reasonably straightforward to do.</p>
<p>Here’s the Shell script:</p>
<div class="vscode" style="white-space: pre;">
<div><span style="color: #6a9955;">#!/bin/sh</span></div>
<div><span style="color: #6a9955;"># Shell script to generate .md files for each method in a Dynamo package,</span></div>
<div><span style="color: #6a9955;"># containing a list of sample graphs that make use of that particular method/node.</span></div>
<div><span style="color: #6a9955;"># Arguments: 1) package name, 2) sub-folder containing sample DYN files.</span></div>
<div><span style="color: #6a9955;"># Search the provided XML for a &quot;member name=?:&quot; (where ? is M or P) string, cutting the &quot;?:&quot;</span></div>
<div><span style="color: #6a9955;"># and stripping off anything after a &quot;(&quot;.</span></div>
<div><span style="color: #6a9955;"># The results get saved to methods.txt - as this is useful to have - but could also be just kept in memory.</span></div>
<div><span style="color: #d4d4d4;">awk -F </span><span style="color: #d7ba7d;">\&quot;</span><span style="color: #d4d4d4;">&#0160;</span><span style="color: #ce9178;">&#39;/member name=\&quot;[M|P]:/ {print $2}&#39;</span><span style="color: #d4d4d4;"> ./bin/</span><span style="color: #9cdcfe;">$1</span><span style="color: #d4d4d4;">.xml | cut -c 3- | awk -F </span><span style="color: #d7ba7d;">\(</span><span style="color: #d4d4d4;">&#0160;</span><span style="color: #ce9178;">&#39;{print $1}&#39;</span><span style="color: #d4d4d4;"> &gt; methods.txt</span></div>
<div><span style="color: #c586c0;">while</span><span style="color: #d4d4d4;">&#0160;</span><span style="color: #dcdcaa;">read</span><span style="color: #d4d4d4;"> m; </span><span style="color: #c586c0;">do</span></div>
<div><span style="color: #d4d4d4;">&#0160; </span><span style="color: #6a9955;"># Place a standard header in the Markdown file containing a title</span></div>
<div><span style="color: #d4d4d4;">&#0160; </span><span style="color: #dcdcaa;">echo</span><span style="color: #d4d4d4;">&#0160;</span><span style="color: #ce9178;">&quot;# Samples using this node:&quot;</span><span style="color: #d4d4d4;"> &gt; ./doc/</span><span style="color: #9cdcfe;">$m</span><span style="color: #d4d4d4;">.md</span></div>
