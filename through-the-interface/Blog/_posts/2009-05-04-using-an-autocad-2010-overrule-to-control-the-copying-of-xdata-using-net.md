<p>It’s quite common for AutoCAD developers to use Extended Entity Data (XData) to tag objects their application cares about. This is certainly the approach taken in the recent series showing <a href="http://through-the-interface.typepad.com/through_the_interface/2009/04/overruling-explode-in-autocad-2010-using-net.html">how to overrule the graphics display for an object</a> – we store a “pipe radius” in XData attached to the Line or Circle we want to have a 3D profile.</p>  <p>That’s fine, but what if we’re storing a unique identifier with an object in XData that we do not want copied with the object?
