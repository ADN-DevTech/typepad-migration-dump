<p>As mentioned, <a href="http://through-the-interface.typepad.com/through_the_interface/2011/09/no-source-available-when-debugging-an-autocad-plug-in.html" target="_blank">last week</a>, I’ve been working away to port my previous OpenNI/NITE AutoCAD-Kinect integration across to the official (although still in Beta) Microsoft Kinect SDK.</p>
<p>Today’s post presents a very basic implementation – essentially equivalent to the code in <a href="http://through-the-interface.typepad.com/through_the_interface/2011/04/previewing-and-capturing-kinect-sensor-data-directly-inside-autocad.html" target="_blank">this previous post</a> – which makes use of the Microsoft Kinect SDK to bring a colourised point cloud into AutoCAD. As in the previous post, the <a href="http://www.cs.unc.edu/~isenburg/lastools" target="_blank">txt2las</a> tool is still currently needed to bring the generated point cloud into AutoCAD.</p>
<p>It’s worth noting that the Microsoft SDK is a) much simpler to install/deploy and b) provides more reliable colourisation of the point cloud than I was able to get from OpenNI (at least at the time). The previous approach simply took pixels from each of the depth and RGB images and mapped the colour directly onto the 3D point. But as there are a few centimeters between the two cameras on the Kinect device, this was ultimately inadequate.</p>
<p>The Kinect SDK provides a function – NuiCamera.GetColorPixelCoordinatesFromDepthPixel() – which does a more reliable (although admittedly a bit slower) colour mapping.</p>
<p>It should also be noted that the Microsoft Kinect SDK provides more accurate mapping of depth data into 3D space (or “skeleton&quot; space”, in the parlance of this particular SDK). The points we get in AutoCAD are actually measured in metres (or fractions thereof). We’ll see how accurate this is, later on.</p>
<p>The big chunk of work, for this particular example, was to implement the equivalent of the GeneratePointCloud() method that Boris Scheiman had kindly implemented for me in his nKinect library. This new version uses the aforementioned SDK methods to generate an accurate – and accurately-colourised – point cloud.</p>
