<p>I wasn’t expecting to write a third part in this <a href="http://through-the-interface.typepad.com/through_the_interface/2013/12/moving-text-in-an-autocad-block-using-net-part-1.html" target="_blank">series</a> of <a href="http://through-the-interface.typepad.com/through_the_interface/2013/12/moving-text-in-an-autocad-block-using-net-part-2.html" target="_blank">posts</a>, but then Samir Bittar went and asked <a href="http://through-the-interface.typepad.com/through_the_interface/2013/12/moving-text-in-an-autocad-block-using-net-part-2.html?cid=6a00d83452464869e2019b02caa3e2970b#comment-6a00d83452464869e2019b02caa3e2970b" target="_blank">a follow-up question</a> that I felt obliged to look into. Thanks for the suggestion, Samir! :-)</p>  <p>Samir basically wanted to provide the user with more feedback as they’re selecting the nested entity – so that the sub-entity gets highlighted, rather than the full block reference.</p>  <p>This turned out to be quite a tricky scenario to address. The overall approach I used was to use a PointMonitor to perform a non-interactive, nested selection of the geometry beneath the cursor and <a href="http://through-the-interface.typepad.com/through_the_interface/2006/12/highlighting_an.html" target="_blank">then highlight the results</a>. The trick was how best to actually perform the selection itself. The options – with their pros and cons – came down to this:</p>  <ol>   <li>Use Editor.GetNestedEntity() to perform the selection.      <ul>       <li>Nice because you get the square selection cursor displayed. </li>        <li>Re-entering the function non-interactively from a PointMonitor causes strange behaviour with the dynamic input tooltip (possibly just one symptom of something that’s fundamentally unstable). </li>     </ul>   </li>    <li>Use Editor.GetEntity() to perform the selection.      <ul>       <li>Similar behaviour to 1, without the benefits of selecting the sub-entity. </li>     </ul>   </li>    <li>Use a basic jig and Editor.Drag() to acquire a point.      <ul>       <li>Also allows you to select the square selection cursor. </li>        <li>Stops the sub-entity highlighting from working at all. </li>     </ul>   </li>    <li>Use Editor.GetPoint() to get a raw point.      <ul>       <li>Works fine with non-interactive, nested entity selection. </li>        <li>You don’t get the nice, square selection cursor. </li>        <li>You need to disable object snap, to avoid further confusion. </li>     </ul>   </li> </ol>  <p>This list may well be missing some options – please do post a comment, if so.</p>  <p>After working through 1-3, I settled on 4. I didn’t like the fact you don’t get the selection cursor, but it seemed the lesser of the various evils.
