<p>In <a href="http://through-the-interface.typepad.com/through_the_interface/2012/04/exposing-a-restful-web-service-for-use-inside-autocad-using-the-aspnet-web-api-part-1.html" target="_blank">the last post</a>, we looked at the core technology we’re going to use to expose our web-service, along with the basic project set-up. In today’s post, we’re going to flesh out this project and see the web-service in action.</p>
<p>At this point, I should point out that I&#39;ve switched back across for VS2010 for the purposes of this post (and its immediate successors), as it really will simplify the eventual integration with Windows Azure.</p>
<p>Before we dive into the implementation of our web-service, here are a few words on what we’re after…</p>
<p>We want to implement two web-service APIs:</p>
<ol>
<li>One that will provide the circles representing an <a href="http://en.wikipedia.org/wiki/Apollonian_gasket" target="_blank">Apollonian gasket</a> (in 2D) </li>
<li>One that will provide the spheres representing an <a href="http://en.wikipedia.org/wiki/Apollonian_sphere_packing" target="_blank">Apollonian packing</a> (in 3D) </li>
</ol>
<p>Both of these services will take a radius (for the enclosing circle/sphere) and a “level” (i.e. the number of times we should recurse to create our fractal geometry).</p>
<p>They should be accessible via a simple URL – as we’re talking <a href="http://en.wikipedia.org/wiki/Representational_state_transfer" target="_blank">REST</a> rather than <a href="http://en.wikipedia.org/wiki/SOAP" target="_blank">SOAP</a> – for example, with a provisional domain name:</p>
<p><a href="http://myapollonianservice.com/circles/5/3">http://myapollonianservice.com/circles/5/3</a> should return the circles enclosed in an outer circle of radius 5, recursing through 3 levels.</p>
<p><a href="http://myapollonianservice.com/spheres/10/4">http://myapollonianservice.com/spheres/10/4</a> should return the spheres enclosed in an outer sphere of radius 10, recursing through 4 levels.</p>
<p>The results should be returned as <a href="http://en.wikipedia.org/wiki/Json" target="_blank">JSON</a> containing the circle/sphere definitions. These will simply be records containing fields for the center (X, Y, [Z] and radius (or curvature, for circles), and also the “level” of each circle/sphere, so we can colorise appropriately).</p>
<p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e2016303bb0914970d-pi" target="_blank"><img align="right" alt="Our Web API solution structure" border="0" height="222" src="/assets/image_963336.jpg" style="background-image: none; margin: 10px 0px 10px 10px; padding-left: 0px; padding-right: 0px; display: inline; float: right; padding-top: 0px; border-width: 0px;" title="Our Web API solution structure" width="162" /></a>Let’s take a look at the structure of our boilerplate solution…</p>

<p>It’s the code in this latter project that we care about, particularly that contained in two files: <em>Global.fs</em> and <em>ValuesControllers.fs</em> file.</p>
