<p>As <a href="http://through-the-interface.typepad.com/through_the_interface/2011/04/pinvoke-tools.html" target="_blank">promised, last week,</a> today we’re going to look at a technique pointed out to me by Viru Aithal for calling unexposed ObjectARX member functions via P/Invoke.</p>  <p>It sometimes happens that ObjectARX (unmanaged C++) methods fail to get exposed immediately via .NET, which is currently the case for MLeader.Scale (both getting and setting this property). The technique shown in this post allows you to work around this, prior to the managed API in AutoCAD being updated to include it. It’s worth noting that if you need a more complex exposure via .NET – as Dan Smith sometimes reminds me (thanks, Dan :-) – it’s preferably to go directly with C++/CLI rather than P/Invoke. This approach isn’t for everyone, of course – you should be comfortable with C++ to consider it – but I did want to just mention that (pre-emptively ;-).</p>  <p>There are a few steps involved in defining your P/Invoke statements, as covered in <a href="http://through-the-interface.typepad.com/through_the_interface/2006/07/calling_objecta.html" target="_blank">this old post of mine</a> and also <a href="http://www.quuxsoft.com/Programming/PInvoke.aspx" target="_blank">here</a> (a nice article which inspired me to include the <a href="http://msdn.microsoft.com/en-us/library/system.security.suppressunmanagedcodesecurityattribute.aspx" target="_blank">System.Security.SuppressUnmanagedCodeSecurity</a> attribute in our class).</p>  <p>I won’t go into that side of things, again, but will cut to the chase: when P/Invoking a method – which is really just a normal function with a “this” argument denoting the object for the method to work on – we simply need to pass in the UnmanagedObject property of our .NET object prior to the other arguments (making sure the method signature has been DllImported using <a href="http://msdn.microsoft.com/en-us/library/system.runtime.interopservices.callingconvention.aspx" target="_blank">the ThisCall calling convention</a>).</p>  <p>Here, for instance, we get the value of our MLeader’s scale property:</p>  <div style="font-family: courier new; background: white; color: black; font-size: 8pt">   <p style="margin: 0px"><span style="line-height: 140%; color: blue">double</span><span style="line-height: 140%"> mls = Scale(ent.UnmanagedObject);</span></p> </div> <!--EndFragment-->  <p>And here we set it to double the original value:</p>  <div style="font-family: courier new; background: white; color: black; font-size: 8pt">   <p style="margin: 0px"><span style="line-height: 140%">SetScale(ent.UnmanagedObject, mls * 2);</span></p> </div> <!--EndFragment-->  <p>Here’s the full C# code defining our DMLS (DoubleMLeaderScale) command:</p>  <div style="font-family: courier new;
