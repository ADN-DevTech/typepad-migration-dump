<p>So starts the next leg of my journey to become platform agnostic. After using a Mac as my primary system for the last year or so (albeit only doing a modest amount of Mac-specific coding, for now, and almost always having a Windows session alive inside Parallels Desktop), I now have my first <a href="http://en.wikipedia.org/wiki/Linux" target="_blank">Linux</a> system, in the form of an incredibly cool <a href="http://en.wikipedia.org/wiki/Raspberry_Pi" target="_blank">Raspberry Pi</a> device.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201761747b243970c-pi" target="_blank"><img style="background-image: none; border-right-width: 0px; margin: 20px auto; padding-left: 0px; padding-right: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="PC, Mac and Linux" border="0" alt="PC, Mac and Linux" src="/assets/image_6832.jpg" width="461" height="317" /></a></p>  <p>This is nearly the first time I’ve touched command-shell <a href="http://en.wikipedia.org/wiki/Unix" target="_blank">Unix</a> since graduating from University – I’ve opened a Terminal window a few times to do something trivial on OS X – but I’m happy to note I’ve managed to remember a fair bit (although I doubt I’ll ever re-learn the <a href="http://en.wikipedia.org/wiki/Vi" target="_blank">vi</a> shortcuts I used to know :-S).</p>  <p>It actually feels really good to be working so close to the metal again – I’ve missed the feeling of connecting with the OS and hardware at such a low level. I’m really having so much fun with this technology. :-)</p>  <p>And it was really very easy to get the device up and running, at least initially: I plugged in a keyboard via USB, a screen via HDMI and an RJ45 LAN cable, and then inserted an SD card containing the OS. To turn it on I simply plugged in a micro-USB phone charger for power, and it booted up right away.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e20177442e3d5b970d-pi" <p align="right"><em><font color="#666666">[Diagram by Paul Beech]</font></em></p>  <p>Over the last day or so, I have:</p>  <ul>   <li>Burned <a href="http://www.raspberrypi.org/downloads" target="_blank">a build of Raspbian “wheezy”</a> (a <a href="http://en.wikipedia.org/wiki/Debian" target="_blank">Debian</a> variant) onto a 2GB SD card using <a href="http://www.softpedia.com/get/CD-DVD-Tools/Data-CD-DVD-Burning/Win32-Disk-Imager.shtml" target="_blank">Win32DiskImager</a> </li>    <li>Booted the Raspberry Pi from it – which, as noted above, worked perfectly first time </li>    <li>Installed various secondary components using the excellent <a href="http://en.wikipedia.org/wiki/Advanced_Packaging_Tool" target="_blank">apt-get</a> (Debian’s equivalent of <a href="http://en.wikipedia.org/wiki/Nuget" target="_blank">NuGet</a>, which was almost certainly influenced by it) </li>    <li>Discovered I needed more space when downloading OpenCV </li>    <li>Saved my 2GB OS installation back to my PC – again using Win32DiskImager – and used it to write the enhanced OS image to a 4GB SD card </li>    <li>Expanded the OS partition to fill the 4GB SD card </li>    <li>Downloaded, extracted, built and installed OpenCV (by far the most time-consuming part) </li>    <li>Ran various OpenCV samples, to see them in action on the Pi </li> </ul>  <p>I’m having some trouble building/running <a href="http://www.shervinemami.info/faceRecognition.html" target="_blank">an OpenCV face detection implementation</a>, but I suspect that’s just through my ignorance.</p>  <p>To help with accessing and managing the device, I also performed some additional tasks:</p>  <ul>   <li>Set up <a href="http://www.simonthepiman.com/how_to_setup_remote_access.php" target="_blank">remote access via SSH and PuTTY</a> </li>    <li>Installed <a href="http://www.simonthepiman.com/how_to_setup_a_web_server_with_php_support.php" target="_blank">a basic PHP web server</a> (which I subsequently disabled – the main point was to get PHP installed and running) </li>    <li>Installed <a href="http://raspcontrol.com" target="_blank">RaspControl</a> (thanks for the tip, Alex!) </li> </ul>  <p>While waiting for help getting face detection via OpenCV working, I decided to “simply” attempt to <a href="http://www.unixmen.com/how-to-turn-your-webcam-into-a-motion-detecting-security-spy-camera-in-linux/" target="_blank">get motion detection working</a>, instead.</p>  <ul>   <li>Installed <a href="http://www.lavrsen.dk/foswiki/bin/view/Motion/WebHome" target="_blank">Motion</a> </li>    <li>Updated <a href="http://blog.pixelami.com/2012/06/raspberry-pi-firmware-update-for-debian-squeeze/" target="_blank">the Raspberry Pi firmware</a> (as I was struggling to get my webcam detected)</li>    <li>Installed <a href="http://blog.pixelami.com/2012/06/uvccapture-on-raspberry-pi-debian-squeeze/" target="_blank">uvccapture</a> to be able to grab stills from the webcam </li> </ul>  <p>So far I’ve managed to get a few stills out of the webcam, with differing lighting conditions:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201761747b271970c-pi"
