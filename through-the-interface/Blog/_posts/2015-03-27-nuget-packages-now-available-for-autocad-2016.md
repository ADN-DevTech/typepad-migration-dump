<p>Some time ago we posted <a href="http://through-the-interface.typepad.com/through_the_interface/2014/12/autocad-2015-apis-available-via-nuget.html" target="_blank">the NuGet packages for AutoCAD 2015’s .NET API</a>. The packages for AutoCAD 2016 are now live, too.</p>  <p>Here’s the report from the NuGet console (accessible in Visual Studio via Tools –&gt; NuGet Package Manager –&gt; Package Manager Console).</p>  <div style="font-size: 8pt; font-family: courier new; background: white; color: black; line-height: 140%">   <p style="margin: 0px">PM&gt; <strong>Get-Package -filter AutoCAD.NET -ListAvailable</strong></p>    <p style="margin: 0px">&#160;</p>    <p style="margin: 0px">Id&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160; Version&#160;&#160;&#160; Description/Release Notes</p>    <p style="margin: 0px">--&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160; -------&#160;&#160;&#160; -------------------------</p>    <p style="margin: 0px">AutoCAD.NET&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160; 20.1.0&#160;&#160;&#160;&#160; AutoCAD 2016 API</p>    <p style="margin: 0px">AutoCAD.NET.Core&#160;&#160;&#160;&#160; 20.1.0&#160;&#160;&#160;&#160; AutoCAD 2016 core object model API</p>    <p style="margin: 0px">AutoCAD.NET.Model&#160;&#160;&#160; 20.1.0&#160;&#160;&#160;&#160; AutoCAD 2016 drawing object model API</p> </div>  <p>To install the 2016 versions of the assemblies into your project, you can use the following command, once again in the NuGet Console.</p>  <div style="font-size: 8pt; font-family: courier new; background: white; color: black; line-height: 140%">   <p style="margin: 0px">PM&gt; <strong>Install-Package AutoCAD.NET -Version 20.1</strong></p> </div>  <p>And the 2015 are still available too, of course:</p>  <div style="font-size: 8pt; font-family: courier new; background: white; color: black; line-height: 140%">   <p style="margin: 0px">PM&gt; <strong>Install-Package AutoCAD.NET -Version 20.0</strong></p> </div>  <p>I actually just learned a nice trick: you can use tab to autocomplete the version number inside the NuGet console, as you can see below. Very handy!</p>  <br /><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201bb08105839970d-pi"><img title="NuGet package manager console with autocomplete" style="display: inline" alt="NuGet package manager console with autocomplete" src="/assets/image_458451.jpg" width="457" height="177" /></a>   <br />  <br />  <p>I hadn’t realised there was a 20.0.1 version posted – I’ll check in on that and report its significance.</p>  <p>As promised <a href="http://through-the-interface.typepad.com/through_the_interface/2015/03/autocad-2016-calling-commands-from-autocad-events-using-net.html" target="_blank">last time</a>, on Monday we’ll look at another possible use of DocumentCollection.ExecuteInCommandAsync() in AutoCAD 2016.</p>  <p><strong><em>Update:</em></strong></p>  <p>It turns out 20.0.1 was a very minor bugfix. There were some versioning issues with 20.0.0 and we’d also omitted the Brep API’s .NET assembly from that version.</p>
