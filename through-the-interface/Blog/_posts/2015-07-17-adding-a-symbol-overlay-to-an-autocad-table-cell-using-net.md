<p>While <a href="http://through-the-interface.typepad.com/through_the_interface/2015/07/adding-a-new-column-to-an-existing-autocad-table-using-net.html" target="_blank">my last post on tables</a> was apparently interesting, it turns out it didn’t actually address the question that inspired it. Oh well.</p>  <p>Here’s <a href="http://through-the-interface.typepad.com/through_the_interface/2007/06/creating_an_aut_1.html#comment-2139420011" target="_blank">a recent comment</a> that gave greater clarity on the requirement:</p>  <blockquote>   <p>Do you know those no smoking signs commonly seen in public places: a cigarette with a red cross struck through it. Can a similar thing be done programmatically in a table with the strike through happening through a number. A picture is worth a 1000 words: <a href="http://imgur.com/gallery/qPzEm1G">http://imgur.com/gallery/qPzEm...</a></p>    <p>In other words, can a block be inserted into a particular cell which already contains a number value? side by side or overlaid perhaps.</p> </blockquote>  <p>I found this to be a really interesting question… While I didn’t think there was a way to add an additional symbol, as such, to an individual cell, I did remember seeing the Cell.SetBlockAttributeValue() method. If we can apply a custom attribute value to an individual cell, then it should be a simple matter of creating a block with an attribute we can use to display our cell “contents”, right?</p>  <p>Anyway, armed with this hypothesis, I went ahead and created my initial “banned” symbol using CIRCLEs, LINEs, TRIM and a solid HATCH.</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201bb085412d7970d-pi"><img title="Creating our banned symbol" style="float: none; margin: 30px auto; display: block" alt="Creating our banned symbol" src="/assets/image_266528.jpg" width="240" height="240" /></a></p>  <p>I then added an attribute definition and BLOCKed the whole thing with the name “BANNED”.</p>  <p>I went ahead and updated the C# code we saw previously, adding a new CBA command (for CreateBlockWithAttributes). This command does something interesting: it adds the additional index column that we added via the MBT command but instead of using a simple row index as the cell’s contents it determines whether the row is odd or even: if it’s odd then we add our banned symbol as the cell contents and then the row index as the attribute value. To do this we need to first get the ObjectId of the BANNED block but also of the AttributeDefinition it contains (we just take the first one we find, for simplicity).</p>  <p>Here’s the table that gets created by the CBA command:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201b7c7affc57970b-pi" target="_blank"><img title="Our block table with symbolic overlays" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; float: none; padding-top: 0px; padding-left: 0px; margin: 20px auto; display: block; padding-right: 0px; border-top-width: 0px" border="0" alt="Our block table with symbolic overlays" src="/assets/image_234663.jpg" width="224" height="427" /></a></p>  <p>Here’s the updated C# code:</p>  <div style="font-size: 8pt;
