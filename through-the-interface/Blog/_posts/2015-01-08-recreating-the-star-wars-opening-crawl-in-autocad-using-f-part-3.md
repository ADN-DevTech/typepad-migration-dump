<p>Now that we’ve <a href="http://through-the-interface.typepad.com/through_the_interface/2015/01/recreating-the-star-wars-opening-crawl-in-autocad-using-f-part-1.html" target="_blank">introduced the series</a> and <a href="http://through-the-interface.typepad.com/through_the_interface/2015/01/recreating-the-star-wars-opening-crawl-in-autocad-using-f-part-2.html" target="_blank">seen how to add some introductory text, theme music and a field of stars</a>, it’s time for the Star Wars logo.</p>  <ol>   <li>The initial blue text </li>    <li>The theme music </li>    <li>The star field </li>    <li><strong>The disappearing Star Wars logo </strong></li>    <li>The crawling text </li> </ol>  <p>The logo came, once again, from the <a href="http://codepen.io/TimPietrusky/pen/eHGfj" target="_blank">HTML opening crawl</a>, which embeds some SVG content representing it. I copied this into a standalone SVG text file, ran it through a <a href="https://cloudconvert.com/svg-to-dxf" target="_blank">web-service</a> to translate it to DXF and then opened that inside AutoCAD.</p>  <p>The outlines of the letters (or groups of letters) were continuous polylines – which was great – but as there were double lines I really wanted to combine these into a single polyline with width. I measured the distance between the edges – it was 0.0833 – and so I offset the inner edge outwards by half this amount, erased the originals and applied a width of 0.0833 to each of the remaining lines.</p>  <p>As you can see in the below GIF of the process:</p>  <p><a href="http://through-the-interface.typepad.com/.a/6a00d83452464869e201b8d0ba82ca970c-pi"><img title="The Star Wars logo inside AutoCAD" style="display: inline" alt="The Star Wars logo inside AutoCAD" src="/assets/image_720843.jpg" width="450" height="222" /></a></p>  <p>You’ll notice that the logo only looks “right” when in a 3D visual style: one of the reasons we switch to a perspective view with “Realistic” for the star field onwards.</p>  <p>Now that we have the logo’s basic geometry, I wanted to capture it as code. I took the approach in <a href="http://through-the-interface.typepad.com/through_the_interface/2010/11/generating-c-code-for-selected-autocad-polylines-using-net.html" target="_blank">this post</a> as a basis, and modified the output code to be in F#. Once I knew it worked well, I went ahead and refactored the vertex definitions to be in a list format: a much cleaner approach for generating the polylines and adding them to the database, especially when working from a functional language that excels at this type of processing.</p>  <p>To simulate the movement of the logo, I decided to just scale it downwards – much simpler than moving it away. There’s definitely some display-dependent behaviour, here: on my retina screen I didn’t need a delay for the timing to work well, but I needed to add one when I changed resolution to record a video… and then needed to change again when I actually recorded (Camtasia seemed to be doing something to actually speed graphics throughput, interestingly). So this will definitely need tweaking on a per system basis.</p>  <p>I could also have included the logo in the supplied drawing as a block, of course, and then have scaled the block in the same way as I have the various polylines. That’s an avenue people may want to explore when fooling around with this code themselves.</p>  <p>Here’s the video of the updated crawl:</p>  <br /><iframe height="264" src="//www.youtube.com/embed/LpHm2D9xh24?rel=0&amp;showinfo=0" frameborder="0" width="470" allowfullscreen="allowfullscreen"></iframe>  <br />  <br />  <p>Here’s the updated F# code.
