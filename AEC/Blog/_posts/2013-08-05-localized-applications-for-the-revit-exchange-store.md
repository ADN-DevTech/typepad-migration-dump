<p>By <a href="http://adndevblog.typepad.com/aec/augusto-goncalves.html">Augusto Goncalves</a></p>  <p><strong><font size="3">What’s New</font></strong></p>  <p>Have you noticed the Autodesk Exchange Store in many more languages? The store is now available for a much bigger audience in many languages: i.e. Čeština, Deutsch, Français, Español, Magyar, Italiano, 日本語, 한국어, Polski, Português, Русский, 简体中文 and 繁體中文. The following image shows how the Revit Store looks like, for example, in Portuguese.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901ea1026f970b-pi"><img title="Autodesk Exchange Store in Portuguese" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Autodesk Exchange Store in Portuguese" src="/assets/image_754489.jpg" width="475" height="334" /></a></p>  <p>If you access the store from your browser, it will show according to the browser’s language settings. If the user clicks on the “X” symbol from the product’s UI, it will call the the browser and show it using the language of Revit that you are running. installation language. Below is an image with the “X” button and respective link for Revit 2014 Portuguese (note that language equals pt, mean that it is Portuguese).</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901ea1028d970b-pi"><img title="X_button" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="X_button" src="/assets/image_380342.jpg" width="406" height="207" /></a></p>  <p><u>Tip</u>: if you want to point to your application on the store (e.g. from your website), just include the language code in the link. If you don’t, it will open the page in the language following the browser settings. </p>  <p><strong><font size="3">Creating Multi-language Revit Applications</font></strong></p>  <p>First, if you plan to design a application that is on just one language, then you can skip to ‘<em>Preparing the .bundle folder</em>’ section. In such a case you just need to publish to the proper store. Your application can have the interface in one language. </p>  <p>Creating multi-language application means that will load the appropriate interface depending on the system configuration. And that’s what will be shown here.</p>  <p>The most basic approach consists in making the .addin file load the correct language string according to Revit language. This is described in this <a href="http://wikihelp.autodesk.com/Revit/enu/2014/Help/3665-Developers/0001-Introduc1/0018-Add-In_I18/0023-Localiza23">wiki help topic</a>. This is interesting, but will only affect the items under ‘External Commands’ menu – at the main ribbon. The <a href="http://usa.autodesk.com/adsk/servlet/item?siteID=123112&amp;id=20143032">Autodesk Exchange Apps - Information for Autodesk Revit Developers</a> states that, among other requirements, your app must have one Ribbon Button (at least), so the ‘External Commands’ does not apply here. We need the External Application definition.</p>  <p>Let’s start a project with the <a href="http://thebuildingcoder.typepad.com/blog/2013/05/add-in-wizards-for-revit-2014-1.html">Revit 2014 template</a> or an existing project. Create a new item of type ‘Resource File’ and, <strong>here is the trick:</strong> put the language code on the file name; for instance:</p>  <ul>   <li>MyAppResource.en-US.resx </li>    <li>MyAppResource.pt-BR.resx </li>    <li>any other you need to support, see a full <a href="http://msdn.microsoft.com/en-us/library/ms533052%28v=vs.85%29.aspx">list here</a>. </li> </ul>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901ea102a0970b-pi"><img title="new_resource" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="new_resource" src="/assets/image_652080.jpg" width="477" height="270" /></a></p>  <p>Now on each <strong><u>resource.LANGUAGE.resx</u></strong> file, add a list of resources strings, such as <strong>CommandName,</strong> <strong>CommandDescription,</strong> <strong>PanelName</strong> and <strong>HelpFile</strong> Below is a sample for Portuguese.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b0192ac606c36970d-pi"><img title="resource_values" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="resource_values" src="/assets/image_313556.jpg" width="488" height="171" /></a></p>  <p>Once you finish creating all the language resources, create a another file with no language specification, for this case, just <u>MyAppResource.resx</u>. Open it and choose ‘Access Modifier’ as ‘Internal’, as shown on the image below. This will make Visual Studio create a class to access the resources according to Revit language.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01910496f3a9970c-pi"><img title="resource_code" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="resource_code" src="/assets/image_821822.jpg" width="487" height="131" /></a></p>  <p><u>Important</u>: remember to deploy your application DLL and the language folders, in this sample en-US and pt-BR. These folders must be at the app DLL folder, otherwise Revit will not recognize them. </p>  <p>Now it’s time to write some code. </p>  <p>To create a panel with buttons there is no major changes, except that the String names will come from the resource manager, following the current execution language. This is a .NET feature, by the way.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901ea102f3970b-pi"><img title="revit_enu_ptb" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="revit_enu_ptb" src="/assets/image_787182.jpg" width="489" height="186" /></a></p>  <p><strong><font size="3">Preparing the .bundle Folder</font></strong></p>  <p>Now create a .bundle folder. For this sample let’s call the app as “<em>ADN Revit Global Product.bundle</em>”. We suggest you create a <u>Contents</u> subfolder, then <u>Win2014</u> subfolder (if you support more version and the app is not binary compatible, create a separated folder for each version). Copy the <u>\bin\Release</u> output content from Visual Studio content, as shown on the image below. The .addin file should be copied here. Finally the <u>Resource</u> subfolder will contain the Help files, and any other resource you might use.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01910496f3df970c-pi"><img title="folder_structure" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="folder_structure" src="/assets/image_25038.jpg" width="477" height="205" /></a></p>  <p>And the <u>PackageContents.xml</u> can be like shown below.
