<p>By <a href="http://adndevblog.typepad.com/aec/joe-ye.html">Joe Ye</a></p>  <p>Using the Dynamic Model Updater (DMU) mechanism, elements can be updated when other element is updated. After applied the DMU mechanism to a document, there will be a “Missing Third Party Updater” message box showing when reopening this Rvt file. The message text in the dialog is like:&#160; The file &lt;Rvt file name&gt; was modified by the third-party updater. Unknown: &lt;updater name&gt; which is not currently installed. If you continue to edit the file, data maintained by Unknown: &lt;updater name&gt; will not be updated properly. This may create problems when &lt;project name&gt; is later opened when Unknown: &lt;updater name&gt; is present.</p>  <p>The message dialog is like this.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b0168eb909ba6970c-pi"><img style="background-image: none; border-right-width: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px; padding-top: 0px" title="Missing Updater" border="0" alt="Missing Updater" src="/assets/image_782.jpg" width="244" height="181" /></a></p>  <p>&#160;</p>  <p><strong>Question</strong>: Sometime users don’t want to see and handle this message box. How to avoid this dialog when opening a document that contains updater?</p>  <p>&#160;</p>  <p><strong>Answer</strong>: There are two kinds of updater according to whether it can let Revit display the “Missing Third Party Updater” message: Optional updater and non-optional updater. The optional updaters don’t let Revit display this message dialog. The non-optional updaters force Revit to display the message box.&#160; You can change the updater type programmatically.</p>  <p>When register an updater, you can set it is optional or non-optional via the third argument.</p>  <div style="font-family: courier new;
