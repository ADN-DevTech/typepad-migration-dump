<p>By <a href="http://adndevblog.typepad.com/aec/augusto-goncalves.html">Augusto Goncalves</a></p>  <p>When we create new Ribbon items to execute a class that implements <strong>IExternalCommand</strong>, we need to specify this class name on the constructor of the&#160; item (e.g. <strong>PushButtonData</strong>). The thing is that parameter is a String and if we use a constant text, if the class name changes, then we need to remember to adjust this value.</p>  <p>So how can we make this more efficient? Simply using <a href="http://msdn.microsoft.com/en-us/library/tay4kywk.aspx" target="_blank">GetType</a> (VB.NET) or <a href="http://msdn.microsoft.com/en-us/library/58918ffs.aspx" target="_blank">typeof</a> (C#). The benefit is the value adjust automatically (Visual Studio renaming mechanism will update it properly) or at least we get and compiler error to help.
