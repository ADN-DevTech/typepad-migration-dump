<p>By <a href="http://adndevblog.typepad.com/aec/augusto-goncalves.html">Augusto Goncalves</a></p>  <p>With Revit 2014, there is a new application available at the Autodesk Exchange Store: <a href="http://apps.exchange.autodesk.com/RVT/en/Detail/Index?id=appstore.exchange.autodesk.com%3astructuralanalysisandcodecheckingtoolkit2014%3aen">Structural Analysis and Code Checking Toolkit 2014</a>. This new tools is available for free.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017eeae69dca970d-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="exchange_apps" border="0" alt="exchange_apps" src="/assets/image_365962.jpg" width="461" height="255" /></a></p>  <p>Once you download and install it, a couple new panels will appear under Analyze tab, as shown below, but it will not run due missing codes registered on the machine. </p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b019101df0897970c-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="codecheck_revit" border="0" alt="codecheck_revit" src="/assets/image_849042.jpg" width="461" height="305" /></a></p>  <p>The Code Check tool requires a code to check the model, but this code needs to be developed using .NET languages. Once developed, the codes can be downloaded and installed like any other Revit plugin using the Autodesk Exchange Store, see how to submit <a href="http://usa.autodesk.com/adsk/servlet/item?siteID=123112&amp;id=20143032">here</a>.</p>  <p>To create a new code, first download the Revit 2014 SDK, available at the <a href="http://www.autodesk.com/developrevit">Developer Center</a> (<a href="http://images.autodesk.com/adsk/files/Revit2014SDK_RTM0.exe">download</a>), or at the Revit install folder, under <u>\support\SDK\RevitSDK.exe</u></p>  <p>Inside the Revit SDK, the folder <u><strong>Structural Analysis SDK</strong></u> should contain all the supporting material. The very first file to look at is the Getting Started PDF at <u>\Revit 2014 SDK\Structural Analysis SDK\Documentation\<strong>Getting Started for Code&#160; Checking Framework SDK.pdf</strong></u>, which is a guide for new developers.</p>  <p>The SDK also contains the C# project templates, that work with Visual Studio. To install the templates, <strong>first close Visual Studio</strong>, then copy the following files:</p>  <ul>   <li>The VSCodeChecking Addin files are required to proper configure the project, so copy the files from folder <u>\Revit 2014 SDK\Structural Analysis SDK\VisualStudio\Addins\</u> to <u>My Documents\Visual Studio 2010\Addins\</u></li>    <li>Copy all 3 .zip files from <u>\Revit 2014 SDK\Structural Analysis SDK\VisualStudio\Templates\ProjectTemplates\Visual C#\Autodesk\Code Checking\</u> to <u>My Documents\Visual Studio 2010\Templates\ProjectTemplates\Visual C#\</u> folder. Do not unzip them.</li>    <li>Copy the <strong>Schema.zip</strong> file from <u>\Revit 2014 SDK\Structural Analysis SDK\VisualStudio\Templates\ItemTemplates\Visual C#\Autodesk\Code Checking\</u> to the folder <u>My Documents\Visual Studio 2010\Templates\ItemTemplates\Visual C#\</u>. Again, do not unzip the files.</li> </ul>  <p>Finally start Visual Studio, create a new project (menu File &gt;&gt; New &gt;&gt; Project) and the new options should appear under C# language. The templates are not available in VB.NET.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017eeae69dfc970d-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="new_project" border="0" alt="new_project" src="/assets/image_816823.jpg" width="461" height="328" /></a></p>  <p>The next steps are required to configure the project, follow the instructions from the wizard. The <strong>VSCodeChecking</strong> addin will update de DLL References path, so just wait a few moments. For your information, the required references are installed with the <a href="apps.exchange.autodesk.com/RVT/en/Detail/Index?id=appstore.exchange.autodesk.com%3astructuralanalysisandcodecheckingtoolkit2014%3aen">Code Checking</a> plug-in mentioned earlier, which is installed by default at <u>C:\ProgramData\Autodesk\ApplicationPlugins\Structural Toolkit for Revit 2014.bundle</u>, but don’t worry, the references paths will be updated automatically by the VSCodeChecking addin.</p>  <p>For a initial test, just Build the solution. The .addin files will be copied to <u>C:\Users\&lt;YourUserName&gt;\AppData\Roaming\Autodesk\Revit\Addins\2014\</u> folder. Finally start Revit or Revit Structure. Once you click on ‘Code Settings’ button, it will open and show the newly created code ‘CodeChecking1’ (which is the VS project name).</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901be90795970b-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="codecheck_registered" border="0" alt="codecheck_registered" src="/assets/image_195004.jpg" width="460" height="310" /></a></p>        <p>Of course this is just to demonstrate the overall structure of the project. There are more complete samples at <u>\Revit 2014 SDK\Structural Analysis SDK\Examples</u> folder. The <u>StructuralAssemblyResolver.exe</u> tool will help you update the references path so the samples open properly on Visual Studio. Just run the tool, select the folder where the sample folder and done.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b017eeae69e38970d-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="samples" border="0" alt="samples" src="/assets/image_224205.jpg" width="244" height="130" /></a></p>  <p>Once the sample projects are built, adjust the .addin files for each sample and copy to Revit Addins folder. The Code Checking toolkit will recognize and load them. Below is the Concrete sample loaded from <u>\Revit 2014 SDK\Structural Analysis SDK\Examples\Concrete\CodeCheckingConcreteExample</u> sample after the respective .addin files were copied to Revit Addins folder.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901be907e8970b-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="concrete_sample" border="0" alt="concrete_sample" src="/assets/image_999610.jpg" width="458" height="184" /></a></p>  <p>The complete documentation is available at the folder <u>\Revit 2014 SDK\Structural Analysis SDK\Documentation\</u>, including the Getting Started PDFs and the Reference Guide CHM, which contains a full list of classes.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01901be90806970b-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="docs" border="0" alt="docs" src="/assets/image_772938.jpg" width="356" height="130" /></a></p>  <p>Hope you enjoyed this first look at the Code Checking SDK. </p>
