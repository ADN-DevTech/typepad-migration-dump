<p>By <a href="http://adndevblog.typepad.com/aec/saikat-bhattacharya.html">Saikat Bhattacharya</a></p>  <p>The Translation parameter (which is of Autodesk.Revit.DB.XYZ type) in the ElementTransformUtils.CopyElement() is often mistaken to be the location at which any given instance is intended to be copied to. Some developers have used this parameter to specify the exact location where they wanted, say, a column, to be copied to and saw unexpected results.</p>  <p>This is translation parameter in the API method is infact the translation <strong>vector</strong> (and not the location). And so this parameter is the vector by which the new instance’s location will be changed to and copied at.
