Thanks to Arnošt Löbel for his help with the following question.</p>  <p><strong>Question</strong>: I'm creating a new note, and needing to add a leader to it. This is updating code from a 2015 plugin to 2016. Previously I was using (where 'note' is my TextNote.Create() object):</p>  <div class="code"><font style="background-color: #ffffff"><font face="Calibri"><font color="#006400">// Set the left and right leader attachments to Top          <br /></font><font color="#000000">note.get_Parameter(</font><font color="#808080">&quot;Left Attachment&quot;</font><font color="#000000">).Set(</font><font color="#800000">0</font><font color="#000000">)</font><font color="#0000ff">;</font></font></font><font style="background-color: #ffffff"><font face="Calibri"><font color="#808080">          <br /></font><font color="#000000">note.get_Parameter(</font><font color="#808080">&quot;Right Attachment&quot;</font><font color="#000000">).Set(</font><font color="#800000">0</font><font color="#000000">)</font><font color="#0000ff">;</font></font></font><font color="#808080">&#160;</font> </div>  <p>That has become obsolete and using the new method, I'm currently using:</p>  <div class="code"><font face="Calibri"><font color="#006400">// Set the left and right leader attachments to Top        <br /></font><font color="#000000">note.LeaderLeftAttachment.Equals(</font><font color="#808080">&quot;TopLine&quot;</font><font color="#000000">)</font><font color="#0000ff">;</font></font><font face="Calibri"><font color="#808080">        <br /></font><font color="#000000">note.LeaderRightAttachment.Equals(</font><font color="#808080">&quot;TopLine&quot;</font><font color="#000000">)</font><font color="#0000ff">;</font> </font></div>  <p>I pulled the &quot;TopLine&quot; from Revit Snoop. However, this is not making it into my text note when created in Revit. It is Top for the Left Attachment, but Bottom for Right Attachment. Is there a best time to pass these values (currently I do it after the leader's been created and before the command completes)?</p>  <p>Thanks in advance for any help.</p>  <p><strong>Answer</strong>: My sincere advice to you so to do less snooping. I understand that the Snoop is a popular tool and is occasionally necessary too, but at many times it does not do any good to you and your application.</p>  <p>As you can find in the RevitAPI.CHM file, text note has three methods related to Leaders. They somehow mirror the UI commands in Revit:    <br />- AddLeader     <br />- GetLeaders     <br />- RemoveLeaders     <br />In your case you will probably be using: <font color="#808080"></font></p>  <div class="code"><font face="Calibri"><font color="#000000">Leader</font><font color="#808080">&#160;</font><font color="#000000">leftLeader</font><font color="#808080">&#160;</font><font color="#0000ff">=</font><font color="#808080">&#160;</font><font color="#000000">myNote.AddLeader(TextNoetLeaderTypes.TNT_STRAIGHT_L)</font><font color="#0000ff">;</font><font color="#808080">&#160; </font></font></div>  <div class="code"><font face="Calibri"><font color="#808080">//and        <br /></font><font color="#000000">Leader</font><font color="#808080">&#160;</font><font color="#000000">rightLeader</font><font color="#808080">&#160;</font><font color="#0000ff">=</font><font color="#808080">&#160;</font><font color="#000000">myNote.AddLeader(TextNoetLeaderTypes.TNT_STRAIGHT_R)</font><font color="#0000ff">;</font></font> </div>  <p>The Add method returns you the newly created Leader object. You will notice, that just like in the UI, the above method add leaders with some default shape and position. However, you can use the returned leader object and using its method a properties adjust the appearance and position of each leader.    <br />For example: </p>  <p><font face="Calibri"><font color="#000000">leftLeader.End</font><font color="#808080">&#160;</font><font color="#0000ff">=</font><font color="#808080">&#160;</font><font color="#000000">xyzPointSomewhereOnTheModel</font><font color="#0000ff">;</font></font><font color="#808080">&#160;</font> </p>  <p>I hope that makes it clear.</p>  <p>Thank you Arnošt for the detailed answer to the Revit API community forum.</p>  <p>Thanks for reading and until next time. </p>
