<p><strong>Update:</strong>&#0160; <a href="http://justonesandzeros.typepad.com/blog/2014/05/deco-2015.html" target="_self">DECO 2015</a> is now available</p>
<p>Time to start updating my apps.&#0160; The first on the list is DECO (Design Editor for Custom Objects).&#0160; Not only is this a must-have when using custom objects, but it gave me a chance to play with the new grid control in the Vault SDK.&#0160; If you are unfamiliar with DECO, have a look at the page for the <a href="http://justonesandzeros.typepad.com/blog/2012/07/deco-custom-ui-without-custom-code.html">2013 version</a>.</p>
<p>I added one cool new feature for DECO 2014: a grid containing all the linked children.&#0160; You can now view/add/delete links using UI created with DECO.</p>

<ul>
<li>Vault Professional 2014</li>

<p><strong>Issues / Limitations:</strong></p>
<ul>
<li>Custom Objects only.&#0160; No support for files, folders, items, and so on.</li>
<li>The grid control only works with links to child entities.&#0160; There is no way to show “where used” data in a DECO control.</li>
<li>Visual Studio 2010 has problems editing a XAML file with a WindowsFormsHost control.&#0160; I don’t know about Visual Studio 2012.</li>
<li>The XAML files and Settings.xml need to be distributed to all clients in order for them to see the custom UI.&#0160; This issue will be addressed in the next release of Project Thunderdome.</li>
<li>When adding a link, the browse dialog only lets you browse items available through Project Explorer.&#0160; In other words, only files, folders and linked entities.</li>
</ul>
