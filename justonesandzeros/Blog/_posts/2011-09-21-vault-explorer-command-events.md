<p><img alt="" src="/assets/Concepts3.png" /></p>
<p>Vault 2012 was a very eventful release for the API in the sense that we added a 3 event engines for you to hook to in your programs.&#0160; I&#39;ve already talked about <a href="http://justonesandzeros.typepad.com/blog/2011/04/faq-events.html">Web Service Command Events</a>, which is the most complex of the 3.&#0160; Now let me switch to the most simple type, Vault Explorer Command Events.</p>
<p>Whenever any UI command is invoked in Vault Explorer, it will fire a pre and a post to anyone that subscribes.&#0160; The UI command is basically everything in the main menu, toolbars, and context menus.&#0160; In your handler, you get passed in the key string for that command.&#0160; And that&#39;s it.&#0160; You don&#39;t get the selection context and you don&#39;t get any information about what happened during the command.</p>
<p>It told you it was simple.&#0160;&#0160; But don&#39;t worry, I can stretch this out to a full article.&#0160; Just because a feature is <em>simple</em> doesn&#39;t mean it&#39;s <em>useless</em>.</p>
<hr noshade="noshade" style="color: #5acb04;" />
<p>Here are a couple of uses to get things started...</p>
<p><strong>Use #1:&#0160; Cache clearing</strong></p>
<p>Sometimes I find myself building up a cache in my custom command.&#0160; For example, I&#39;ll need to read all the file Property Definitions.&#0160; I don&#39;t want to read these objects each time my command is invoked, so I save the data in memory.&#0160; But what if I want to keep my cache up-to-date?&#0160; I just need to check for the Refresh or Admin Settings command and clear the cache at that point.</p>
<p><strong>Use #2:&#0160; Event combo</strong></p>
<p>Just like in video games, you make use of combos to get the best results.    <br />Web Service Command events are good at knowing what work is being done but only in isolation.&#0160; They don&#39;t know what overall command is being done.&#0160; Vault Explorer Command events are just the opposite.&#0160; If you combine the two, you have a fairly complete picture.</p>
<p>Here is the timeline of a Copy Design operation that creates 3 files.&#0160; The top blue lines are the Vault Explorer events, the bottom blue lines are the web service events, and the red lines are the web service function calls.&#0160;</p>
<p><img alt="" src="/assets/Timeline.png" /></p>
<p>We can make good use of all these events.&#0160; For example, you can send out an email report after a copy design is completed.&#0160; Here is how you would do it:</p>
<ol>
<li>In the Vault Explorer pre event for Copy Design:&#0160; Create a new List. </li>
<li>In the DocumentService AddFile post event:&#0160; Add the File to your List. </li>
<li>In the Vault Explorer post event for Copy Design:&#0160; Use the entries in the List to send out the email report explaining which files were added via Copy Design. </li>
</ol>
<p>Now you&#39;re playing with power!&#0160;&#0160;</p>
