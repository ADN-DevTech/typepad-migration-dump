<p><img alt="" src="/assets/SampleApp3.png" /></p>
<p>Change Orders have a feature where you can have a discussion threads on each object.&#0160; I thought it would be neat if you could do the same thing with files and items.&#0160; I also promised a sample app using Custom Entities, so here it is.</p>
<p><img alt="" src="/assets/discussonTab.png" /></p>
<hr noshade="noshade" style="color: #013181;" />
<p><strong>Features:</strong></p>
<ul>
<li>Add discussion threads to existing items and files. </li>
<li>Files and items can have multiple threads on a single object. </li>
<li>Respond to postings and see the whole discussion history. </li>
<li>Works even if the items and files are released. </li>
<li>App will configure Vault when the admin first runs it.&#0160; No need to manually create the custom object definition, UDP, etc. </li>
</ul>
<p><img alt="" src="/assets/config.png" /></p>
<hr noshade="noshade" style="color: #013181;" />
<p><strong>Requirements:</strong></p>
<ul>
<li>Vault Professional 2013 </li>
</ul>

<hr noshade="noshade" style="color: #013181;" />
<p><strong>To Use:</strong></p>
<p>Run the install and start Vault Explorer as an administrator.&#0160; You should be prompted to run a configuration.&#0160; After the configuration is complete, restart Vault Explorer.&#0160; You should now have the Discussion Thread custom object.</p>
<p>All users who want to make use of this app, should have the Custom Object Editor (Level 1) and Custom Object Manager (Level 1) roles.</p>
<p>When you want to add a discussion to a file or item, select the object and right click.&#0160; Select the New Discussion Thread command to create the new discussion.</p>
<p><img alt="" src="/assets/FileContext.png" />&#0160; <br />To respond to the thread, use the Discussion tab.&#0160; Select the posting you want to respond to, and hit the Respond button in the tab.</p>
