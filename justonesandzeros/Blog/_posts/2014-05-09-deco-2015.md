<p>I was hoping that I could retire DECO now that <a href="http://help.autodesk.com/view/VAULT/2015/ENU/?guid=GUID-3025E6BA-F3CB-4F6B-81B4-D2E0B8FD47AA" target="_blank">Vault Data Standard has support for custom objects</a>.&#0160; Unfortunately, DECO still has one feature that Data Standard does not.&#0160; DECO overrides the “new” command with custom content.&#0160; Data Standard lets you create another new command, but the default new is unchanged.&#0160; It’s a subtle difference, but enough for me to continue supporting DECO.</p>
<p>It’s probably worthwhile for me to compare and contract both products.&#0160; I should also mention that they are not mutually exclusive.&#0160; You can have both Data Standard and DECO installed side by side if you want.</p>
<p>Another cool thing is that both products use the same framework for editing UI.&#0160; They both use XAML files, which can be edited in the free version of Visual Studio.&#0160; So learning one product will give you experience on the other one.&#0160; Likewise, it should be very easy to port your DECO content to Data Standard.</p>

<hr noshade="noshade" style="color: #013181;" />
<p>DECO (Design Editor for Custom Objects) mainly specializes in “property page” functionality.&#0160; It’s great for displaying and editing properties on a single custom object.&#0160; If that’s is all the functionality you need, then you will probably find DECO a bit easier to use than Data Standard.</p>
<p><img alt="" src="/assets/edit-scaled.png" /></p>
<p><img alt="" src="/assets/run.png" /></p>
<hr noshade="noshade" style="color: #013181;" />
<p>Data Standard can be used to create New, Edit and View operations for custom objects.&#0160; However, Data Standard offers many more features.&#0160; You can create custom commands and tabs for whatever you want.&#0160; You are not limited to New, Edit and View.&#0160; You are also not limited to custom objects.&#0160; Files, folders, items and change orders are also customizable.</p>
<p>The extra features make Data Standard a bit more complex.&#0160; So it’s a few extra steps to get New, Edit and View working when compared to DECO.</p>
