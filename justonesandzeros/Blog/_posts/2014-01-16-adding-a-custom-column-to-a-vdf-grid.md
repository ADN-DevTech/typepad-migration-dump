<p><img alt="" src="/assets/Vault4.png" /> <br /><img alt="" src="/assets/TipsAndTricks4.png" /></p>
<p>As I mentioned a <a href="http://justonesandzeros.typepad.com/blog/2014/01/vdf-client-properties.html" target="_blank">few articles ago</a>, the VDF has the concept of client properties.&#0160; On top of that, the VDF gives you the ability to add your own.&#0160; The main advantage here is that it gives you a column of your very own in the VDF grid.&#0160; Beyond that, most of the work is up to you.&#0160; You need to set the value.&#0160; If you want to persist the data, that’s also up to you.</p>
<hr noshade="noshade" style="color: #d09219;" />
<p>Perhaps it’s best if I show a quick example.&#0160; Let’s create a property called “my prop def”, and set values based on a counter that I increment.</p>
<p><img alt="" src="/assets/screenshot.png" /></p>
<p>&#0160;</p>
<p><strong>Step 1: Define a class the implements IPropertyExtensionProvider.</strong></p>
<p>IPropertyExtensionProvider can do a lot of things.&#0160; Here I’m using it to supply a client property definition to the VDF.&#0160; I’m also using it to provide the values to that property when the VDF asks for them.</p>

<p><strong>Step 3: Enjoy. </strong></p>
<p>You can now use the column in any of your UI that uses the VDF.&#0160; Even if a VDF grid doesn’t explicitly set the column, the property shows up in the list when the user goes to customize the columns.</p>
<p>If your app is a Vault Explorer 2014 plug-in, your custom property is <strong>not</strong> available in the main grid or any of the default tabs.&#0160; This is because those grids are not true VDF grids.&#0160; (Fun fact: The file browser on the Attach command <em>does</em> used the VDF grid, so you can see your custom properties there)</p>
