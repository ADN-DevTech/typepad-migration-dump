<p>Keep in mind I don&#39;t know <em>when</em> I&#39;ll be upgrading any of these apps.&#0160; Sometime before the next Vault release is as specific as I can get right now.</p>
<p><strong>My proposed solution</strong></p>
<p>One of the main reasons that I posted the source code to all the apps is so that I wouldn&#39;t be solely responsible for things like bug fixes, new features and upgrades.&#0160; I&#39;m perfectly fine with other people making the needed updates.&#0160;</p>
<p>Here are the basic steps:</p>
<ol>
<li>Update the app. </li>
<li>Remove all the Autodesk copyright/legal stuff.&#0160; Replace with your own copyright/legal stuff if you want. </li>
<li>Remove the OnesAndZeros icon: <img alt="" src="/assets/OnesAndZeros.png" /> </li>
<li>Increase the version number. </li>
<li>Update the readme. </li>
<li>Post the app somewhere. </li>
<li>Tell me where you posted the app, and I&#39;ll link to it from this blog.&#0160; The app itself, will not be hosted here. </li>
</ol>
<p>The basic idea is that when you update the app, it becomes <em>yours</em>, not <em>mine</em>.&#0160;</p>
<p><strong>Rejected solutions:</strong></p>
<p>A couple of people suggested using online development sites like SourceForge or CodePlex.&#0160; Those sites are fine, but I don&#39;t want to be the one managing it.&#0160; If I did, that would be yet one more site that I would be running, and I don&#39;t have the time.&#0160; Feel free to set up a site on your own, and I will be happy to link to it.&#0160; I just don&#39;t want to <em>run</em> the site.</p>
<p>Submitting an updated project to me also won&#39;t work.&#0160; I don&#39;t have time to review other people&#39;s changes.&#0160; In many ways, reviewing code takes longer than just doing the update myself.&#0160; At least when I write the update, I know exactly what changed.</p>
<p><strong>Other ideas:</strong></p>
<p>If you have any other suggestions, let me know in the comments section below.&#0160;</p>
