<p>In writing <a href="http://justonesandzeros.typepad.com/blog/2013/09/vault-web-view-2014.html" target="_blank">Vault Web View 2014</a>, I came across a problem.&#0160; I wanted to store a common configuration for some custom tab settings.&#0160; The standard way to do this is with a Vault Option.&#0160; The problem is that custom tabs are registered <em>before</em> Vault Explorer logs in to Vault.&#0160; So how do I set up the custom tabs or custom commands if I can’t read the Vault Option?</p>
<p>A couple other people have run into this same problem, so I’d like to go over my solution.&#0160; It’s nothing fancy or anything.&#0160; It’s just a message telling the user to restart Vault Explorer.&#0160; This technique works for custom commands, tabs and custom entity handlers.</p>
<p>Vault Web View deals with 2 copies of the config settings.&#0160; One copy is kept locally as an XML file in the same folder as the extension DLL.&#0160; The other copy is XML data stored in a Vault Option.&#0160; When Vault Web View sets up the tabs, it does it based off of the local settings file.&#0160; </p>
