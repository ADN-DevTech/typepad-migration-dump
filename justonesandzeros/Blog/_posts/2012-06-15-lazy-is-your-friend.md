<p><img alt="" src="/assets/TipsAndTricks3.png" /></p>
<p>The Lazy class is a nice little tool that was added in .NET 4.&#0160; Like the name implies, it helps with <a class="zem_slink" href="http://en.wikipedia.org/wiki/Lazy_initialization" rel="wikipedia" target="_blank" title="Lazy initialization">lazy initialization</a> of data.&#0160; That way the data is initialized when you ask for it instead of when you class is constructed.</p>
<p>This is useful when programming with Vault because there are many times you need to look up server data and store it.&#0160; For example, let’s say you have a custom command that makes use of a specific property definition.&#0160; At some point you need to get the PropDef object from Vault server, but when is the best time?&#0160; You don’t want to do it during OnLogOn because it costs time to look up the PropDef, and the user may not run your command.&#0160; The best time to get the PropDef is the first time your command is run.&#0160; Then you store the object so that subsequent calls don’t have to make the same server request again.</p>
<p>Here is how you would look up the Author PropDef with Lazy:</p>
