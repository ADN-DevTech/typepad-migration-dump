<p>Update:&#0160; The <a href="http://justonesandzeros.typepad.com/blog/2013/10/effective-permissions-2014.html" target="_blank">2014 version</a> is now available.</p>
<p>I’m currently in an amazingly interesting DB training class.&#0160; So I decided to do some Vault app programming so that my heart wouldn&#39;t explode from all the excitement.&#0160; The result is Effective Permissions 2013.&#0160; It’s an update of my <a href="http://justonesandzeros.typepad.com/blog/2010/04/effective-folder-permissions-20.html" target="_blank">Effective Folder Permissions 2.0</a>.&#0160; The ‘Folder’ part of the name has been dropped since it now does more than just folder permissions.&#0160; Files and custom objects can also be managed.</p>
<p>Check out the video:</p>
<iframe allowfullscreen="allowfullscreen" frameborder="0" height="315" src="http://www.youtube.com/embed/ol53Wch8rB4" width="420"></iframe>
<p>The main new addition is a view/edit window for object permissions.&#0160; You can now edit an Access Control List and see the effect before you commit the change.&#0160; It’s also smart enough to determine the source of the ACL and make the 
appropriate updates.</p>
<p><img alt="" src="/assets/PermissionsWindow_scaled.png" /></p>
<p>All the old features are still there.&#0160; So you can still see the Administrators List, and you still have the Permissions Overview, which shows the user-based permissions for a set of folders.</p>
<p><img alt="" src="/assets/PermissionsOverview.png" /></p>
