<p>I decided it was time to bring back Vault Web View.&#0160; The concept is pretty simple.&#0160; The app provides a tab with a web control in it.&#0160; The web page shown is based on the selected object.</p>
<p><a href="http://justonesandzeros.typepad.com/images/2013/VaultWebView/fullscreen.png" target="_blank"><img alt="" src="/assets/scaled.png" /></a>     <br /><span style="color: #666666;">(click image for full view)</span></p>
<p>New for Vault 2014.&#0160; You can now have multiple tabs.&#0160; And you can determine which tabs show up on which entity types.&#0160; Lastly, you can use any Vault property or properties to determine the URL.&#0160; </p>
<p><img alt="" src="/assets/Configure.png" /></p>
<p>How it works is the admin sets up the URL for the tab.&#0160; Anything in { } gets replaced by the property value in the selected object.&#0160; So you can build up URLs based on Vault properties.</p>
<p>So, if a tab is set to “http://www.{sitename}.com” and you select an object where the <strong>sitename</strong> property is <strong>autodesk</strong>, you will see <a href="http://www.autodesk.com">http://www.autodesk.com</a> in the tab view.</p>

<p><strong>Notes:</strong></p>
<ul>
<li>The resulting URL value must start with &quot;<strong>http://</strong>&quot;.&#0160; Otherwise you won&#39;t see anything in the tab view. </li>
<li>If an object is in a folder, the folder properties can also be used in the URL.&#0160; How it works is the selected object properties are applied first to the URL.&#0160; Next, the parent folder properties are applied to the URL.&#0160; This way if a property is empty in the selected object, the folder can provide the value. </li>
<li>If there are empty property values required by the URL, then nothing will show up in the tab view. </li>
</ul>
