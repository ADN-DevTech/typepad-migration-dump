<p><img alt="" src="https://justonesandzeros.typepad.com/images/2011/4apps/4 apps-title2.png" /></p>
<p>If you are like me, and I know I am, you would prefer to work with a visual representation of file lifecycle states.&#0160; Thankfully <a href="http://justonesandzeros.typepad.com/blog/weipeng-hou.html" target="_blank">Weipeng Hou</a>, <a href="http://justonesandzeros.typepad.com/blog/frank-ran.html" target="_blank">Frank Ran</a> and <a href="http://justonesandzeros.typepad.com/blog/yuyue-wang.html" target="_blank">Yuyue Wang</a> came up with Visual Lifecycle Status.</p>
<p><a href="http://justonesandzeros.typepad.com/images/2011/VisualLifecycleStatus/overview.png" target="_blank"><img alt="" border="0" src="/assets/overview_scaled.png" style="border-width: 0px;" /></a> <br /><span style="font-size: xx-small;">(click image for full view)</span></p>
<p><img alt="" src="https://justonesandzeros.typepad.com/images/2011/4apps/4 apps-hr2.png" /> <br /><strong>Features:</strong></p>
<p>For a selected file, the app shows all the states for the lifecycle, along with the allowed transitions.&#0160; Select a transition to see more information about it, such as revision bumps or transition rules.    <br />&#0160;<img alt="" src="/assets/TransitionInfo.png" /></p>
<p>A red line is shown if you select a transition that is not allowed.<img alt="" src="/assets/TransitionBlock.png" /></p>
<p>Double click on a state to move the file to that state.    <br /><img alt="" src="/assets/TransitionMove.png" /></p>
<p><strong><img alt="" src="https://justonesandzeros.typepad.com/images/2011/4apps/4 apps-hr2.png" /> <br />Requirements:</strong></p>
<ul>
<li>Vault Workgroup 2012, Vault Collaboration 2012 or Vault Professional 2012 </li>
</ul>

<p><strong><img alt="" src="https://justonesandzeros.typepad.com/images/2011/4apps/4 apps-hr2.png" /> <br />To Use:</strong></p>
<ol>
<li>Download the application. </li>
<li>Unzip and run the installer. </li>
<li>Run Vault Explorer. </li>
<li>Select a file with lifecycle state information. </li>
<li>Go to the Lifecycle Status tab </li>
</ol>
