<p>12月16日～18日に開催予定の&#0160;Accelerator Workshop について、いくつかご質問をいただいていますので、それらを FAQ としてまとめてみました。参加をご検討いただく際の参考にしていただければと思います。</p>
<ol>
<li>&#0160;<strong>Accelerator Workshop とは何ですか ?</strong><br /><br />オートデスクが提供する Web サービス API を使ってサービスやアプリを開発していただく開発イベントです。ハッカソンとは異なり、開発内容やアイデアを競うのではなく、実際のソリューションとして提供したいサービスやアプリのプロトタイプを開発するのが目的です。<br /><br /></li>
<li><strong>対象となる Web サービス API は何ですか ?</strong><br /><br />Accelerator Workshop でサポートされる API は、次のとおりです。AutoCAD や Revit、Inventor などのデスクトップ製品 API のみの使用は対象にはなりませんが、それらと下記の Web サービス API を組み合わせてプロトタイプを開発するアイデアでも参加いただくことが可能です。<br /><br />・ Data and View API<br />・ AutoCAD I/O<br />・ ReCap Photo API<br />・ BIM 360 Glue API<br />・ BIM 360 Field API<br />・ Fusion 360 API<br />・ Infraworks 360 API（予定）<br /><br /></li>
<li><strong>開発者ではありませんが、参加することは出来ますか ?</strong><br /><br />実際に提供したいソリューションを開発するなかで、必要な情報提供やお手伝いをするイベントです。可能であれば、開発知識をお持ちの方にご参加いただくようお願いしています。<br /><br /></li>
<li><strong>3 日間の開催期間中すべてに参加しなければなりませんか ?</strong><br /><br />年末のお忙しい時期でもありますので、みなさまのスケジュールにお任せいたします。3 日間で開発を完了させるのは難しいと思いますが、プロトタイプ開発に目途を立てる意味でも、通しでご参加いただくことをお勧めいたします。全日ご参加いただけない場合でも、管理上ご参加いただく方の人数を把握する必要がございますので、必ず事前お申込みをお願いします。お申込み詳細は、<a href="http://adndevblog.typepad.com/technology_perspective/2015/10/developer-week-2015.html#workshop" target="_blank"><strong>こちら</strong></a>&#0160;をご参照ください。<br /><br /></li>
<li><strong>Accelerator Workshop に参加費用がかかりますか ?</strong><br /><br />Accelerator Workshop には無償でご参加いただけます。ただし、ADN にご加入いただいていない方には、NDA（秘密保持契約）を Web フォームでご記入いただく必要がございます。NDA 締結のご案内は、ADN 事務局に参加申し込みされた時点でご案内させていただきます。お申込みの方法は、<a href="http://adndevblog.typepad.com/technology_perspective/2015/10/developer-week-2015.html#workshop" target="_blank"><strong>こちら</strong></a> をご参照ください。<br /><br /></li>
<li><strong>Web サービス API の学習のために参加することは出来ますか ?</strong><br /><br />特定の目的を持って開発作業していただくことをお願いしています。ワークショップ形式でのトレーニングではありませんので、その点をご留意ください。<br /><br /></li>
<li><strong>オートデスクの特定の Web サービス API を利用する予定ですが、詳細を把握出来ていません。このような状態でも参加することが出来ますか ?</strong><br /><br />期間中、1日目と2日目には、可能な範囲で 15 ～ 30 分程度の説明セッションをご用意します。お申込みいただく際に、お使いになりたい対象 API をご申告ください。<br /><br /></li>
<li><strong>開発用のPCを持ち込みますが、どのような開発環境をインストールしておけばいいですか ?</strong><br /><br />Web ブラウザで稼働させることを目的としたオートデスク Web サービス API の多くは、JavaScript や REST API を使用します。例えば、View and Data API の場合は、<a href="http://adndevblog.typepad.com/technology_perspective/2015/08/setup-development-environment-view-and-data-api.html" target="_blank"><strong>こちら</strong></a> にあるような環境などを考えることが出来ます。このため、特に必須となる開発環境やツールは指定していません。<br />Windows など、特定のプラットフォームで動作するネイティブアプリを開発される場合や、AutoCAD などのデスクトップ製品アドインを開発して Web サービス API と連携するような場合には、Visual Studio や Xcode をご用意いただく必要があります。これは、Android や iOS についても同様です。ただし、個別の開発環境の利用方法までは、把握しかねる場合があります。<br />また、Fusion 360 API や BIM 360 API、Infraworks 360 API など、ホストとなる特定のクラウド サービスの API をお使いいただく場合には、あらかじめそれらをインストールしていただく必要があります。<br /><br /></li>
<li><strong>最終日の発表は必須ですか ?</strong><br /><br />可能な範囲で結構です。実際のデモがあればベストですが、プレゼンテーション データで実装（予定）内容の概要をご紹介いただいたり、意向をご説明いただいても結構です。機密情報のようなノウハウをご披露いただく必要はありません。<br /><br /></li>
<li><strong>他国の事例を見ることは出来ますか ?</strong><br />
