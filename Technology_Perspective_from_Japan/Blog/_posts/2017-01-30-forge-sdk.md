<p><strong><a href="http://adndevblog.typepad.com/technology_perspective/2017/01/various-forge-samples.html" rel="noopener noreferrer" target="_blank">多様な Forge サンプル</a></strong> のブログ記事では、デベロッパ ポータルに説明付きで記載されたサンプルについてご案内しました。ここでご紹介した <strong><a href="https://autodesk-forge.github.io/" rel="noopener noreferrer" target="_blank">https://autodesk-forge.github.io/</a></strong>&#0160;のタイトル キャプションには、「SDKs」 の言葉が挿入されている点にお気づきの方もいらっしゃると思います。</p>
<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb09720016970d-pi" style="display: inline;"><img alt="Forge_samples_and_sdks" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01bb09720016970d image-full img-responsive" src="/assets/image_283697.jpg" title="Forge_samples_and_sdks" /></a></p>
<p><strong><a href="https://ja.wikipedia.org/wiki/%E3%82%BD%E3%83%95%E3%83%88%E3%82%A6%E3%82%A7%E3%82%A2%E9%96%8B%E7%99%BA%E3%82%AD%E3%83%83%E3%83%88" rel="noopener noreferrer" target="_blank">SDK</a></strong> とは、Software Development Kit の略で、オートデスクでは、デスクトップ製品のアドインを開発するための ライブラリやユーティリティ、サンプル、ドキュメントを 1 つにまとめたパッケージを指しています。具体的には、AutoCAD の <a href="http://www.autodesk.com/objectarx" rel="noopener noreferrer" target="_blank"><strong>ObjectARX SDK</strong> </a>や Revit の <strong><a href="http://help.autodesk.com/view/RVT/2017/JPN/?guid=GUID-D7E8694D-7DB3-41CA-A0F3-AF64DC2DA015" rel="noopener noreferrer" target="_blank">Revit SDK</a></strong> などを、製品とは別に独立した SDK として提供しています。</p>
<p>一方、Forge の場合には、Authentication API(OAuth API）や Data Management API、Model Derivative API など、<strong><a href="https://ja.wikipedia.org/wiki/REST" rel="noopener noreferrer" target="_blank">RESTful API</a></strong>&#0160;が主体であるため、Web サービス API や Web サービス といった表現を使って説明しているため、「SDK」という言葉には馴染まない印象があります。</p>
<p>実は、この <strong><a href="https://autodesk-forge.github.io/" rel="noopener noreferrer" target="_blank">https://autodesk-forge.github.io/</a></strong>&#0160;&#0160;ページには、Forge サンプルのほかに、Forge の開発する SDK と呼ぶ Node.js 用の<a href="https://ja.wikipedia.org/wiki/%E3%83%9F%E3%83%89%E3%83%AB%E3%82%A6%E3%82%A7%E3%82%A2" rel="noopener noreferrer" target="_blank">ミドルウェア</a>&#0160;をパッケージ化して記載しています。このため、「Forge Samples and <strong>SDKs</strong>」のような表記をしています。</p>
<p>現時点では、次のドルウェア パッケージが用意されています。</p>
<p><img alt="Forge_nodejs_sdk" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01bb0971ffc5970d image-full img-responsive" src="/assets/image_205060.jpg" title="Forge_nodejs_sdk" /></p>
<ul>
<li><strong><a href="https://github.com/Autodesk-Forge/forge.oauth2-js" rel="noopener noreferrer" target="_blank">forge.oauth2-js</a></strong></li>
</ul>
<p style="padding-left: 60px;">Authentication API(OAuth API）の RESTFul API をまとめた Node.js パッケージです 。<a href="https://www.npmjs.com/" rel="noopener noreferrer" target="_blank">npm サイト</a>上の <a href="https://www.npmjs.com/package/forge-oauth2" rel="noopener noreferrer" target="_blank">https://www.npmjs.com/package/forge-oauth2</a>&#0160;からも参照することが出来ます。</p>
<ul>
<li><strong><a href="https://github.com/Autodesk-Forge/forge.oss-js" rel="noopener noreferrer" target="_blank">forge.oss-js</a></strong></li>
</ul>
<p style="padding-left: 60px;">2-legged 認証で得たアクセス トークンを使って OSS（Object Storage Service）を利用する&#0160;Data Management API 部分のNode.js パッケージです。<a href="https://www.npmjs.com/" rel="noopener noreferrer" target="_blank">npm サイト</a>上の <a href="https://www.npmjs.com/package/forge-oss" rel="noopener noreferrer" target="_blank">https://www.npmjs.com/package/forge-oss</a>&#0160;からも参照することが出来ます。</p>
<ul>
<li><strong><a href="https://github.com/Autodesk-Forge/forge.data.management-js" rel="noopener noreferrer" target="_blank">forge.data.management-js</a></strong></li>
</ul>
<p style="padding-left: 60px;">3-legged 認証で得たアクセス トークンを使って A360 などのオートデスク ストレージのユーザ領域を利用する Data Management API 部分の&#0160;Node.js パッケージ です。<a href="https://www.npmjs.com/" rel="noopener noreferrer" target="_blank">npm サイト</a>上の <a href="https://www.npmjs.com/package/forge-data-management" rel="noopener noreferrer" target="_blank">https://www.npmjs.com/package/forge-data-management</a>&#0160;からも参照することが出来ます。</p>
<ul>
<li><strong><a href="https://github.com/Autodesk-Forge/forge.model.derivative-js" rel="noopener noreferrer" target="_blank">forge.model.derivative-js</a></strong></li>
</ul>
<p style="padding-left: 60px;">デザインデータを変換する Model Derivative API をまとめた &#0160;Node.js パッケージです。<a href="https://www.npmjs.com/" rel="noopener noreferrer" target="_blank">npm サイト</a>上の <a href="https://www.npmjs.com/package/forge-model-derivative" rel="noopener noreferrer" target="_blank">https://www.npmjs.com/package/forge-model-derivative</a>&#0160;からも参照することが出来ます。</p>
<ul>
<li><strong><a href="https://github.com/Autodesk-Forge/forge-api-nodejs-client" rel="noopener noreferrer" target="_blank">forge-api-nodejs-client</a></strong></li>
</ul>
<p style="padding-left: 60px;">Authentication API(OAuth API）、Data Management API、Model Derivative API、Design Automation API をまとめて 1 つにまとめた Node.js パッケージです。<a href="https://www.npmjs.com/" rel="noopener noreferrer" target="_blank">npm サイト</a>上の <a href="https://www.npmjs.com/package/forge-apis" rel="noopener noreferrer" target="_blank">https://www.npmjs.com/package/forge-apis</a>&#0160;からも参照することが出来ます。</p>
<p>これら SDK を Node.js で使うことで得られる利点ですが、現時点は次のようなものになるはずです。</p>
<ul>
<li>Forge Platform API が提供する RESTful API の endpoint を直接呼び出すコードを実装する必要がなく、実装コードを簡素化することが出来る。</li>
<li>ある程度、コーディング スタイルを統一し易くなる。</li>
<li>今後、予想される endpoint やパラメータの変更、基本的なセキュリティ対策を SDK 側が吸収してくれるため（してくれるはず）、既存のコードをなるべく維持して、変更等に対する細かい対応が不要になる。</li>
<li>npm コマンドでインターネット上からパッケージをインストールすることで出来るため、比較的容易に開発環境を構築することが出来る。</li>
</ul>
<p>上記 Node.js 用ミドルウェア パッケージ の場合には、パッケージ自体が更に別のパッケージを数多く参照しているため、ご自身ですべて実装するよりも、かなりコードが簡素化されていくはずです。</p>
<p>一方、参照しているミドルウェア パッケージにも細かいバージョンが存在し、一定のタイミングでバージョンアップされています。これら、ミドルウェア パッケージのバージョンアップで SDK の互換性が崩れないよう、メンテナンスし続ける必要があるのは言うまでもありません。Forge SDK に限らず、Node.js のミドルウェア パッケージの利用には、その点に注意を払う必要があります。幸い、オートデスクが提供する Forge SDK は、Forge の開発チームが直接メインテナンスに携わっているため、この点は安心です。</p>
<p>Forge SDK を利用するか否かは、Forge をお使いになる開発者の一存に依ります。これら Forge SDK は、前述の ObjectARX SDK &#0160;のように、利用しなければ開発が出来ない、といった類のものではありませんのご注意ください。&#0160;</p>
<p>なお、Node.js ではなく、Java 環境用に Authentication API(OAuth API）、Data Management API、Model Derivative API、Design Automation API &#0160;をまとめた&#0160;<strong><a href="https://github.com/Autodesk-Forge/forge-api-java-client" rel="noopener noreferrer" target="_blank">forge-api-java-client</a></strong> も用意されています。</p>
