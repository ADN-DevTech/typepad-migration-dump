<p style="text-align: left;">Autodesk Exchange Apps ストアの実績について、<a href="http://adndevblog.typepad.com/technology_perspective/2015/07/autodesk-exchange-apps-store-surpasses-1-million-downloads.html" target="_blank">以前のブログ記事</a>でご紹介しました。この Autodesk Exchange Apps ストアには、どなたでもアプリやコンテンツを無償で記載していただくことが出来ます。現在、実際に記載されているコンテンツには、次のようなものがあります。</p>
<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb085533c3970d-pi" style="float: right;"><img alt="Exchange_apps" class="asset  asset-image at-xid-6a0167607c2431970b01bb085533c3970d img-responsive" src="/assets/image_915528.jpg" style="width: 180px; margin: 0px 0px 5px 5px;" title="Exchange_apps" /></a></p>
<ul>
<li>アドイン/プラグイン アプリケーション</li>
<li>スタンドアロン アプリケーション</li>
<li>トレーニング教材</li>
<li>電子書籍</li>
<li>コンテンツ</li>
<li>Web サービス</li>
<li>その他</li>
</ul>
<p>ここで言うスタンドアロン アプリケーションとは、オートデスク製品を指すのではなく、&#0160;オートデスク製品データを変換したり、特殊な編集を加えるようなコンパニオン アプリケーションを指しています。また、コンテンツとは、AutoCAD ブロック ライブラリ、Revit ファミリ ライブラリ、Inventor パーツ ライブラリなどの製品内で利用されるコンテンツが主流です。</p>
<p>また、Web サービスでは、3rd Party デベロッパが開発、販売しているサービスで、オートデスク製品データを扱ったり、操作するための Web ブラウザを用いたサービスを指しています。もちろん、スタンドアロン アプリケーションでも、Web サービスで、ビューワー的な位置づけのものも存在しています。</p>
<p>このように、既存のオートデスク製品でカバー仕切れていない機能やコンテンツなどを補助するアプリやコンテンツが豊富に記載されています。</p>
<p>さて、アプリやコンテンツの記載方法ですが、特定のオートデスク製品用に作成を公開を希望される場合には、一定のルールに沿ってアプリやコンテンツを作成したり、利用できるようにしていただく必要があります。</p>
<p>各製品毎のルールは、次の各リンクから詳細を把握していただくことが出来ます。</p>
<ul>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=21691866">AutoCAD と AutoCAD 業種別製品デベロッパ向けの情報</a></div>
</li>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=21691942">Revit デベロッパ向けの情報</a></div>
</li>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=21691918">Inventor デベロッパ向けの情報</a></div>
</li>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=22889343">3ds Max デベロッパ向けの情報</a></div>
</li>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=23080340">Maya デベロッパ向けの情報</a></div>
</li>
</ul>
<p>&#0160;※ ここに記載の製品については、個別に appsinfo@autodesk.com へメールまでお問い合わせください。</p>
<p>さて、Web サービスを含むアプリやコンテンツの公開方法の手順や概要は、</p>
<p style="text-align: center;"><span style="font-size: 18pt;"><a href="http://www.autodesk.co.jp/app-publish" target="_blank">http://www.autodesk.co.jp/app-publish</a></span>&#0160;</p>
<p>に記載されていますので、ぜひ、ご確認ください。アプリ提出前に準備すべき内容については、<a href="http://images.autodesk.com/apac_japan_main/files/app_submission_-_required_information_j.doc">アプリ提出フォーム– マーケティング テクニカル情報の準備</a>&#0160;をご確認ください。&#0160;</p>
<p style="text-align: center;"><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b8d13ba652970c-pi" style="display: inline;"><img alt="Any_apps" class="asset  asset-image at-xid-6a0167607c2431970b01b8d13ba652970c img-responsive" src="/assets/image_686560.jpg" style="width: 420px;" title="Any_apps" /></a></p>
<p>もし、有償版のアプリやサービスを公開される場合には、エンドユーザの購入操作が発生した際に、どの決済システムを利用するか決定しておく必要があります。前述の<a href="http://adndevblog.typepad.com/technology_perspective/2015/07/autodesk-exchange-apps-store-surpasses-1-million-downloads.html" target="_blank">ブログ記事</a>でも触れていますが、現在、決済には、<a href="http://home.bluesnap.com/" target="_blank"><strong>BlueSnap</strong></a>&#0160;と <strong><a href="http://www.paypal.jp/" target="_blank">PayPal</a>&#0160;</strong>を利用することが可能です。残念ながら、有償版の公開時に指定できるのは&#0160;<strong>US ドル</strong>&#0160;での価格設定のみだけですが、BlueSnap を選択した場合、エンドユーザの日本国内での購入時には、円換算された状態で決済することが出来ます。<a href="http://adndevblog.typepad.com/technology_perspective/2015/07/autodesk-exchange-apps-store-surpasses-1-million-downloads.html" target="_blank"><br /></a></p>
<p>Autodesk Exchange Apps ストアへ有償版アプリの提出前には、事前に BlueSnap、ないし、PayPal のアカウント設定を完了しておく必要があります。それぞれの作成手順については、次のリンクをご参照ください。</p>
<ul>
<li><a href="https://apps.exchange.autodesk.com/ja/Public/BlueSnapGuide" target="_blank">BlueSnap</a>&#0160;（推奨）</li>
<li>
<div><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=21692080">PayPal</a></div>
</li>
</ul>
<p style="text-align: left;">&#0160;</p>
<p style="text-align: center;"><a class="asset-img-link" href="http://www.autodesk.co.jp/subscription/desktop" style="display: inline;" target="_blank"><img alt="Desktop-subscription-lockup-stacked-no-year" class="asset  asset-image at-xid-6a0167607c2431970b01bb0856579d970d img-responsive" src="/assets/image_297137.jpg" style="width: 420px;" title="Desktop-subscription-lockup-stacked-no-year" /></a><br />&#0160;</p>
<p>さて、 オートデスクのデスクトップ製品は、従来の永久ライセンス販売から、Desktop Subscription と呼ばれるレンタル ライセンス（期間限定ライセンス）へ、その販売形態を移行していきます。詳細は過去の<a href="http://adndevblog.typepad.com/technology_perspective/2015/05/about-desktop-subscription-products.html" target="_blank">ブログ記事</a>に譲りますが、Desktop Subscription の期間限定のデスクトップ製品で動作するアドインやプラグイン アプリケーションも、 場合によって、同じように一定期間のみ動作するよう、アプリ内の仕組みを変えていく必要があります。</p>
<p>また、エンドユーザが有償版のアプリを利用する場合には、不正利用を防止するため、ユーザがアプリを購入した実績があるか、チェックする必要も発生するはずです。オートデスクでは、主要なオートデスク製品では、製品利用中にサインインしているのエンドユーザの Autodesk ID を利用して、その Autodesk ID で特定のアプリの購入実績をチェックする &#0160;Entitlement API（権限 API） と呼ばれる&#0160;API を用意しています。</p>
<p>Entitlement API の利用方法と、期間限定ライセンスを実現するためのシナリオについては、次のリンクをご確認ください。</p>
<ul>
<li><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=24534621">アプリでのコピー保護の実装</a></li>
<li><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=24534506">デスクトップ アプリ用の Entitlement API（権限 API）</a></li>
<li><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=24534714">IPN 通知フォーマット</a></li>
<li><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=24534807">月額 Subscription のデスクトップ アプリ販売</a></li>
<li><a href="http://www.autodesk.co.jp/adsk/servlet/item?siteID=1169823&amp;id=24535572">月額 Subscription の Web サービス販売</a></li>
</ul>
<p>繰り返しになりますが、Autodesk Exchange Apps ストアへは、どなたでもアプリやコンテンツを公開することが出来ます。クラウドや期間限定ライセンスへの対応も可能です。この機会に、お手持ちのアプリ、コンテンツやノウハウの公開をご検討いただければと思います。</p>
<p>なお、ここまでご紹介した内容も含めて、Developer Center には、アプリ公開ををまとめたページも用意されています。こちらも、併せてご確認ください。</p>
