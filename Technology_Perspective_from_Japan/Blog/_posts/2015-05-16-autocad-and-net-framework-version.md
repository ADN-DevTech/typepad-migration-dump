<p style="text-align: center;"><span style="background-color: #ffff00;">&lt;2020年4月19日更新&gt;</span><a href="http://adndevblog.typepad.com/technology_perspective/2013/12/benefit-by-autocad-dotnet-api.html" rel="noopener noreferrer" target="_blank"><strong>AutoCAD .NET API の利点</strong></a> で紹介したとおり、AutoCAD 2005 以降のバージョンでは、AutoCAD 自身が Microsoft .NET Framework を利用して機能を実装しています。AutoCAD .NET API でアドイン アプリケーションを開発する場合には、.NET Framework のバージョンを意識する必要があります。もちろん、該当する .NET Framework バージョンを対象としてアセンブリを出力できる Visual Studio が必要です。</p>
<p>AutoCAD 2016 もリリースされていますので、改めて、AutoCAD と AutoCAD が利用している .NET Framework バージョン、開発環境としてサポートされる Visual Studio バージョンを列挙しておきたいと思います。</p>

<p style="padding-left: 30px;">※1 .NET Framework 4 Update 1 は、<a href="http://support.microsoft.com/kb/2533523/ja" rel="noopener noreferrer" target="_blank">KB2533523</a> として入手することが出来ます。</p>
<p style="padding-left: 30px;">※2 AutoCAD 2017 は.NET Framework 4.6 をインストールしますが、アドインは<a href="http://help.autodesk.com/view/ACD/2017/JPN/?guid=GUID-A6C680F2-DE2E-418A-A182-E4884073338A" rel="noopener noreferrer" target="_blank">.NET Framework 4.5 互換を許容</a>します。</p>
<p style="padding-left: 30px;">※3 AutoCAD 2025 からは、従来製品が採用してきた Windows 専用の&#0160; .NET Framework に代わって、クロスプラットフォームで利用可能な .NET を採用しています。詳しくは、<a href="https://adndevblog.typepad.com/technology_perspective/2023/08/net-framework-and-net-core.html" rel="noopener" target="_blank">.NET Framework と .NET（.NET Core）</a>をご参照ください。</p>
<p>Visual Studio バージョンについては、ObjectARX アプリケーションのサポート環境として利用される場合を想定しているため、Visual Studio バージョンが出力可能な .NET Framework バージョンとは異なり、Visual Studio バージョンに含まれる Visual C++ バージョンを基準として設定している点に注意してください。</p>
