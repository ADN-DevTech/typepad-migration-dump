<p>引き続き、<strong>オートデスク 3D ハッカソン</strong> の参加者を募集中です。ご参加いただいた方には、当日、T シャツをお配りしますので、指向性のあった方同士でチームを作って、作成したゴールに取り組んでいただくことになります。T シャツはこのイベント専用なので、ちょっとしたレアものです。</p>
<p><img alt="T_shirts" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01b7c6dae6a3970b image-full img-responsive" src="/assets/image_4346.jpg" style="display: block; margin-left: auto; margin-right: auto;" title="T_shirts" /></p>
<p>お申込みは、次の URL にて受け付けています。なお、<span style="background-color: #ffff00;">参加人数を把握する必要がありますので、必ず事前登録をお願いいたします。</span></p>
<p style="text-align: center;"><span style="font-size: 18pt;"><a href="http://autodesk.expoline.jp/3dhackathon/" target="_blank"><strong>http://autodesk.expoline.jp/3dhackathon/</strong></a></span></p>
<p><strong><a href="http://adndevblog.typepad.com/files/%E3%82%AA%E3%83%BC%E3%83%88%E3%83%87%E3%82%B9%E3%82%AF-3d-%E3%83%8F%E3%83%83%E3%82%AB%E3%82%BD%E3%83%B3-%E3%83%81%E3%83%A9%E3%82%B7-3.pdf">オートデスク 3D ハッカソン チラシ</a>&#0160;</strong>も用意しましたので、お仲間を誘っていただく際にお使いください。また、ハッカソンで利用いただけるテクノロジは、<a href="http://adndevblog.typepad.com/technology_perspective/2014/07/technologies-on-autodesk-3d-hachathon.html" target="_blank"><strong>こちら</strong></a>&#0160;で詳しくご案内しています。</p>
<p>さて、このハッカソン イベントについては、いろいろと興味を持っていただいているようで、さまざまなご質問を頂戴しています。今日は、そのいくつかにお答えしておきたいと思います。</p>
<p><strong>なぜ、2日間の日程で開催されるのですか?</strong></p>
<p style="padding-left: 30px;">初日は、お使いいただくことが出来るテクノロジの説明後、参加者個人のアイデアに基づいて、チームを作っていただきます。チーム作成後には、チームとしてのゴールや作業分担するようなディスカッションも必要です。この過程はアイデアソンと呼ばれますが、開発作業前に必ず必要なものです。</p>
<p style="padding-left: 30px;">2日目は、実際の開発作業になります。もちろん、各チームの発表や投票、授賞式なども考えると、全行程で2日間は必要というわけです。</p>
<p><strong>どちらか1日の参加でも大丈夫ですか?</strong>&#0160;</p>
<p style="padding-left: 30px;">チーム ビルディングを考えると2日間の参加をお願いしたいところです。どうしても無理という方は、オートデスク&#0160;ハッカソン イベント事務局までご相談ください。 事務局の連絡先は、<a href="http://autodesk.expoline.jp/3dhackathon/" target="_blank">http://autodesk.expoline.jp/3dhackathon/</a>&#0160;に明記されています。</p>
<p style="padding-left: 30px;">繰り返しになりますが、達成感や異業種コミュニケーション、社会経験を期待される方には、 2日間を通した参加をお勧めしたいところです。&#0160;</p>
<p><strong>コンピュータやネットワーク環境は用意されていますか?</strong></p>
<p style="padding-left: 30px;">ハッカソンで利用するコンピュータやモバイル機器は、各自必要に応じてご持参ください。同様に、エディタ、コンパイラ等の開発環境や、画像編集ソフトや CAD/CG ツールなども、日頃お使いのものをインストールしてご持参ください。どのようなツールを利用するかも自由なので、特に規定は用意していません。</p>
<p style="padding-left: 30px;">3D モデルの作成には、オートデスク製品の体験版もお使いいただけると思います。日本語版を含む各種体験版は、<a href="http://www.autodesk.com/free-trials" target="_self">http://www.autodesk.com/free-trials</a> から無償で入手可能です（30日間の無償利用期間で義利用ください）。</p>
<p style="padding-left: 30px;">WiFi ネットワークは用意されていますので、WiFi 接続できる機器なら、インターネットに接続することが出来ます。モバイル ルーターをお持ちの場合は、もちろん、そちらをお使いいただくことも出来ます。</p>
<p><strong><strong>食事は用意されているのでしょうか</strong>?</strong></p>
<p style="padding-left: 30px;">作業を継続しながら適宜食事出来るように、軽食（昼食）とお飲物、スナックをご用意しています。</p>
<p><strong>サンプルなどは提供されますか?</strong></p>
<p style="padding-left: 30px;">コードサンプルは提供されます。シナリオにあった 3D モデルは提供されません（各種製品のサンプルは利用可能です）。</p>
<p><strong>チームで参加することはできますか?</strong></p>
<p style="padding-left: 30px;">はい、可能です。チームを作成する際には、ご要望に応じてチーム構成を考慮いたします。</p>
<p><strong>提供されるテクノロジ以外の利用は出来ますか?</strong></p>
<p style="padding-left: 30px;">はい、可能です。View &amp; Data サービス API や ReCap Photo API 以外にも、他の Web サービス API やデスクトップ製品、プラグイン、ライブラリ、モバイル デバイスなどを組み合わせて、自由に&#0160;<strong><a href="http://ja.wikipedia.org/wiki/%E3%83%9E%E3%83%83%E3%82%B7%E3%83%A5%E3%82%A2%E3%83%83%E3%83%97_(Web%E3%83%97%E3%83%AD%E3%82%B0%E3%83%A9%E3%83%9F%E3%83%B3%E3%82%B0)" target="_blank">マッシュアップ</a>&#0160;</strong>いただくことが出来ます。なお、社外秘にあたるような未公開のテクノロジやサービスなど、特定の企業や団体、個人に悪影響を与えるものは、責任を負いかねますので利用はお控えください。</p>
<p><strong>Web グラフィック関係の仕事をしていますが、全く新しくテクノロジを習得する必要がありますか?</strong>&#0160;</p>
<p style="padding-left: 30px; text-align: left;">ハッカソンでお使いいただく View &amp; Data サービス API は、WebGL と、WebGL ベースの JavaScript ライブラリである Three.js をベースに構築された Web サービス API です。この意味では、一部、Three.js で定義されたクラスなどを利用するので、既存の知識を生かすこも出来ます。ただし、View &amp; Data サービス APIの最も大きな違いとして、JavaScript コードで 3D モデルやシーン、マテリアル、光源などを定義していくのではなく、あらかじめオーサリング ツールで作成した 3D モデルを表示するサービスであるという点をご認識ください。</p>
<p style="padding-left: 30px; text-align: left;"><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b8d064defc970c-pi" style="display: inline;"><img alt="Base_technologies" class="asset  asset-image at-xid-6a0167607c2431970b01b8d064defc970c img-responsive" src="/assets/image_131092.jpg" style="width: 420px; display: block; margin-left: auto; margin-right: auto;" title="Base_technologies" /></a></p>
<p style="padding-left: 30px; text-align: left;">ここでいうオーサリング ツールとは、CAD や CG ツールということになりますが、これらはオートデスク製品である必要はありません。例えば、<a href="http://www.sketchup.com/ja" target="_blank"><strong>Trimble SketchUp</strong></a> や <a href="http://apps.microsoft.com/windows/ja-jp/app/3d-builder/75f3f766-13b3-45e9-a62f-29590d5781f2" target="_blank"><strong>Microsoft 3D Builder</strong></a>&#0160;などでも View &amp; Data サービス API がサポートするファイル形式を保存できるもなら&#0160;OK です。サポートされるファイル形式の詳細は、<a href="http://adndevblog.typepad.com/files/autodesk-360-view-data-service-%E3%83%81%E3%83%A9%E3%82%B7.pdf"><strong>Autodesk 360 View &amp; Data Service チラシ</strong></a>&#0160;をご参照ください。</p>
<p style="padding-left: 30px; text-align: left;">ファイル形式によっては、 マテリアル情報を表現できないものもありますが、<a href="http://www.autodesk.co.jp/products/navisworks/free-trial" target="_blank"><strong>Autodesk&#0160;Navisworks の体験版</strong></a>でお金をかけずにマテリアルを適用していただくようなことも出来ます。&#0160;</p>
<p style="padding-left: 30px; text-align: left;">もちろん、<a href="http://adndevblog.typepad.com/technology_perspective/2013/10/recap-photo-cloud-service.html" target="_blank"><strong>ReCap Photo</strong></a> で写真から 3D モデルを生成する、といったことも可能ですし、処理を&#0160;<strong><a href="http://adndevblog.typepad.com/files/autodesk-360-recap-photo-service-%E3%83%81%E3%83%A9%E3%82%B7-1.pdf">Autodesk 360 ReCap Photo Service</a>&#0160;</strong>で自動化することも出来ます。</p>
<p style="text-align: left;"><strong>プログラム知識は必須ですか?</strong></p>
<p style="padding-left: 30px;">いいえ。ハッカソンはチームで取り組んでいただくイベントです。チームには 3D モデリングなどに長けた方や、Web ページをデザインされる方も必要になります。もちろん、それ以前にアイデアが重要になります。このため、アイデアを出していただく&#0160;方も必要ですし、チームをリードしてくリーダーも必要です。</p>
<p style="padding-left: 30px;">みなさんの知識や能力を生かしていただけるので、プログラミング知識が必須というわけではありません。</p>
<p><strong>オートデスク製品の使用経験がないのですが、参加できますか?</strong></p>
<p style="padding-left: 30px;">もちろんご参加いただけます。 オートデスク製品の知識や経験は必須ではありません。</p>
<p><strong>CAD ツールを作る必要があるのでしょうか?</strong></p>
<p style="padding-left: 30px;">決して CAD ユーザのツールだけを対象にしているわけではありません。View &amp; Data サービスは、ブログあソーシャルメディアなど、各種 Web ページに埋め込んでお使いいただけますし、ReCap Photo サービスは写真が取れれば 3D モデルを生成することが出来ます。</p>
<p style="padding-left: 30px;">なので、利用用途はみなさんのアイデア次第です。当然、CAD ツールとしてのアイデアも大歓迎です。</p>
<p>これ以外にも、ご不明な点がありましたら、オートデスク&#0160;ハッカソン イベント事務局までお問い合わせください。</p>
<p>みなさんの参加をお待ちしています。</p>
