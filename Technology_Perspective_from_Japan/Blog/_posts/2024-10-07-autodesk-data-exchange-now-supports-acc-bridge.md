<p><a href="https://construction.autodesk.com/tools/bridge/" rel="noopener" target="_blank">ACC Bridge</a> と Autodesk Data Exchange が連携し、多職種間コラボレーションをまったく新しいレベルに引き上げたことを発表できることを嬉しく思います。</p>
<h3 style="font-weight: 800;">Data Exchange（データ交換）とは？</h3>
<p style="padding-left: 40px;">今日の AEC プロジェクトでは、重要なデザイン データが大量の異なるファイル形式を持つファイルに閉じ込められているため、コラボレーションの円滑さを阻害、遅延を招く結果になっています。<a href="https://aps.autodesk.com/data-exchange-cover-page" rel="noopener" target="_blank">Autodesk Data Exchange</a> は、この問題点に対処します。</p>
<p style="padding-left: 40px;">Data Exchange を使用すると、ファイル全体ではなく、BIM モデル内の照明器具など、モデルデータのサブセットを共有することが出来るようになります。この共有サブセットは、オートデスクのエコシステム（Revit、AutoCAD、Inventor など）だけでなく、Rhino、Grasshopper といったオートデスク外のソリューションでも使用することが可能です。</p>
<p style="padding-left: 40px;">Data Exchange には、PDF、IFC、RVT といｙたファイルを使ったデータ交換よりも優れた点があります。つまり、静的アーカイブ形式となるファイル全体を共有する必要がなくなるため、意図しない知的財産（IP、Intellectual Property）のコピー/共有を低減することが出来る、という点です。Data Exchange を使ったデータ交換では、データがアプリの境界を越え、組織の境界も越えることが出来るようにすることで、多くの相互運用性ワークフローを解き放ちます。</p>
<h3 style="font-weight: 800;">Bridge（ブリッジ）とは？</h3>
<p style="padding-left: 40px;">Bridge は、互いに協力する企業に永続的なデータ所有権を提供することを目指しています。Hub（ハブ）/Account（アカウント）は、特定の企業にマッピングされることが多いため、Bridge によってHub/Account 間でデータを共有し、作業範囲が完了した後も各 Hub/Account が所有権を維持できるようにする方法を提供します。また、企業が独自のプロジェクト、独自の権限、独自の基準を設定する自律性を提供します。</p>
<p style="padding-left: 40px;">Bridge を利用しないと、ACC をファイル リポジトリとして使用している企業はすべて、互いに「共有」するために 1 つの ACC プロジェクト内に存在しなければなりません。同じスペースで数十の企業が働いているかもしれませんが、最終的には Hub 所有者だけがそのデータの真の所有権を持っています。これは多くのお客様にとっては十分に機能しますが、他のお客様にとってはきな問題になる場合があります。</p>
<p style="padding-left: 40px;">Bridge には、ACC シート ツール、ACC ファイル ツール、Design Collaboration、[As-Builts（ACC および PlanGrid）の機能があります。ACC の「製品」に関しては、Autodesk Docs、Build、または Design Collaboration で Bridge を使用できます。</p>
<h3 style="font-weight: 800;">App（アプリ）、Hub（ハブ）、Project（プロジェクト）間でサブセットを共有</h3>
<p style="text-align: center;"><iframe allowfullscreen="" frameborder="0" height="270" src="//www.youtube.com/embed/-1ERMOuJPX0" width="480"></iframe></p>
<p style="padding-left: 40px;">今回、Bridge は、ACC の Hub と Project 間でのデータ交換の共有をサポートするようになりました。ユーザーは Autodesk Docs Viewer でデータ交換を作成できます。Exchange が作成されると、ACC内 の別の Hub または Project にブリッジ出来るようになります。基になる Revit ファイルに公開された更新は、Exchange で更新をトリガーし、その更新をブリッジされたHub と Project に自動的に同期します。</p>
<p style="padding-left: 40px;">これは、ユーザーが初めて、デザイン ファイル全体ではなく、デザイン データのサブセットを、それが作成された ACC プロジェクトを超えて共有できることを意味します。この機能は、データ共有を簡素化してデータ セキュリティを向上させるだけでなく、ACC プロジェクトやオーナーシップ スペース全体で収集されたコラボレーションを促進する重要な一歩となるものです。</p>
<p style="padding-left: 40px;">Autodesk University 2024 では、<a href="https://conferences.autodesk.com/flow/autodesk/au2024/sessioncatalog/page/inperson/session/1714141794330001ItYU" rel="noopener" target="_blank"><strong>CS2353 | The Art of Bridge: SSOE Group’s Secret Weapon for Long-Term Success</strong></a> クラスで、SSOE の Luis Angel-Garcia がブ Bridge をテーマにしたクラスで講演をおこないます。<a href="https://conferences.autodesk.com/flow/autodesk/au2024/sessioncatalog/page/inperson/session/1714141794330001ItYU" rel="noopener" target="_blank"></a>この新しい Data Exchange 統合の早期テスターとして、多大な協力をいただきました。</p>
<p style="padding-left: 40px;"><strong><em>「Data Exchange の革新と Bridge を組み合わせることで、AEC 業界に新たな可能性が開かれています。これらの革新的なツールは、チームがクラウドで収集されたデータを安全に共有する方法を再構築し、設計と調整だけでなく、プロジェクトのライフサイクルの全フェーズでデータ利用を最適化しながら、コントロールとプライバシーを強化します。この可能性が明らかになるにつれて、業界はより効率的なワークフロー、より強固な信頼、そしてシームレスなコラボレーションの新時代を迎える準備がで可能になるでしょう。」</em></strong></p>
<p style="text-align: right;"><strong>- Luis Angel-Garcia 氏 (VDC テクニカル リード、SSOE)</strong></p>
<h3 style="font-weight: 800;">どう機能するのか？</h3>
<ol>
<li>Data Exchange（データ交換）を任意の ACC プロジェクトの一部として作成します。<br />
<ol>
<li>まず、パブリッシュ セットに 3D ビューが含まれているパブリッシュされた Revit ファイルから&#0160;<a href="https://help.autodesk.com/view/DOCS/JPN/?guid=Create_Manage_Data_Exchanges" rel="noopener" target="_blank">ACC でエクスチェンジを作成します</a>。</li>
<li>Exchange 作成の一環として、Exchange を保存する ACC 内のフォルダを指定するように求められます。</li>
</ol>
</li>
<li>プロジェクトフォルダと別のプロジェクトまたはハブとの間にブリッジを確立します。<br />
<ol>
<li>Exchange が保存されているフォルダを共有することを選択します</li>
<li>[Share with Another project]（別のプロジェクトと共有） を選択し、Exchange を同期する宛先を選択します。</li>
<li>Bridge は、自動同期を確保するためにフォルダ レベルで設定する必要があります。ファイル レベルでブリッジを設定しても、変更は自動的に同期されません。</li>
</ol>
</li>
<li>更新が発生すると、データは自動的に同期されます<br />
<ol>
<li>ソースとなっている Revit ファイルが更新され、RVT が再度 Docs にパブリッシュされると、Data Exchange によってバージョンも自動的に更新されます。</li>
<li>Data Exchange の更新が完了すると、宛先プロジェクトまたはハブのブリッジされたフォルダーでも更新が踏襲されます。</li>
</ol>
</li>
</ol>
<h3 style="font-weight: 800;">この新しい統合にはどのような利点があるのか?</h3>
<ul>
<li>知的財産を保護し、ファイル全体の共有ではなくきめ細かなデータ共有を可能にする、プロジェクト間の交換ワークフローを実現します。</li>
<li>Power BI を使ったモデル品質チェックや、Revit、Rhino、Inventor、Tekla などのモデル コーディレーション ワークフローのためのダウンストリーム&#0160; ワークフローで共有設計データにアクセスできます。</li>
</ul>
<ul>
<li>自動化されたワークフローにより、共同プロジェクトをより安全に、効率的に、予測どおりに実行できます。</li>
</ul>
