<p>1月22日（火）東京、1月25日（金）大阪で BIM &amp; Forge セミナーを開催し、合計で約 100 名の皆様にご参加いただきました。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b022ad3918609200c-pi" style="display: inline;"><img alt="Venues" border="0" class="asset  asset-image at-xid-6a0167607c2431970b022ad3918609200c image-full img-responsive" src="/assets/image_410860.jpg" title="Venues" /></a></p>
<p>当日ご紹介したプレゼンテーション資料と動画（一部除く）を下記に公開いたしますので、ご確認ください。</p>
<hr />
<div style="padding-left: 40px;"><strong>13:00～14:00&#0160; BIM 360 ファミリ製品とデータ管理&#0160;－ アナビ、森</strong></div>
<div style="padding-left: 40px;">
<p style="padding-left: 40px;">Revit&#0160; など、BIM を標ぼうする CAD ソフトウェアで作成した BIM データをクラウド上に保存し、関係者間でコラボレーションするプラットフォームに BIM 360 Docs があります。ここでは、BIM 360 Docs が持つ効果的な機能や Revit との連携に加え、<strong><a href="https://adndevblog.typepad.com/technology_perspective/2018/05/nextgen-bim-360.html" rel="noopener" target="_blank">次世代 BIM 360</a></strong> と言われている BIM 360 の製品体系と役割を整理しました。</p>
<p style="padding-left: 40px;"><strong><span class="asset  asset-generic at-xid-6a0167607c2431970b022ad3b78fc5200d img-responsive"><a href="https://adndevblog.typepad.com/files/1.bim360%E8%A3%BD%E5%93%81%E7%B4%B9%E4%BB%8B.pdf">プレゼンテーション資料をダウンロード</a></span></strong></p>
<p><strong>14:00～14:10&#0160; &#0160; &#0160;&lt;休憩&gt;</strong></p>
<p><strong>14:10～15:00 Forge 概要と BIM 360 の関係&#0160;－ 伊勢崎</strong></p>
<p style="padding-left: 40px;">Forge Platform API の 1 つに BIM 360 API があります。このセッションでは、Revit API など、デスクトップ製品 API と Forge の考え方や違いを示し、Forge が現在提供するソリューションと BIM 360 の関係を明示的ご案内しました。</p>

<p><strong>15:00～15:15&#0160; &#0160; &#0160;&lt;休憩&gt;</strong></p>
<p><strong>15:15～16:05 Forge による BIM 360 Docs 統合&#0160;－ 伊勢崎</strong></p>
<p style="padding-left: 40px;">オートデスクがクラウドを使って実現しようとしている方向性をご紹介した後に、Forge を使って BIM 360 に統合する手順と方法、また、統合によって得られる効果をコストも含めてご説明しました。</p>

<p><strong>16:05～16:20 &#0160; &#0160; &lt;休憩&gt;</strong></p>
<p><strong>16:20～17:20&#0160;&#0160;Forge Design Automation API for Revit での自動化&#0160;－ 小笠原</strong></p>
<p style="padding-left: 40px;">Design Automation API を利用すれば、Revit がデスクトップの環境になくても、クラウド上で Revit API を使用したカスタム処理を実行して、 Revit のモデルを作成・編集したり、データを抽出したりすることができます。このセッションでは、Revit アドインの開発経験のある方や Forge にご興味のある方を対象に、Design Automation API でサポートされる機能の概要と、また機能的な制約、そして活用方法をご紹介いたしました。</p>
