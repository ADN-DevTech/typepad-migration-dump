<p><a href="https://openai.com/blog/chatgpt/" rel="noopener" target="_blank">ChatGPT</a> をご存じでしょうか？</p>
<p>ChatGPT は <a href="https://ja.wikipedia.org/wiki/OpenAI" rel="noopener" target="_blank">OpenAI</a> が開発、公開している自然対話を目的にした<a href="https://ja.wikipedia.org/wiki/%E4%BA%BA%E5%B7%A5%E7%84%A1%E8%84%B3" rel="noopener" target="_blank">チャットボット</a>（chatbot）です。今のところ利用者からのフィードバックを得る目的で「調査プレビュー」として無償提供されていますが、将来変更される可能性はあります。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02af14ae5e73200b-pi" style="display: inline;"><img alt="Chatgtp" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02af14ae5e73200b image-full img-responsive" src="/assets/image_422499.jpg" style="width: 757.37px;" title="Chatgtp" /></a></p>
<p>ちなみに、以前、オートデスクのチャットボットである Autodesk Virtual Agent、改め、Autodesk Virtual Assistant、通称 AVA について<a href="https://adndevblog.typepad.com/technology_perspective/2018/05/autodesk-virtual-agent-ava.html" rel="noopener" target="_blank">触れた</a>ことがあるかと思います。AVA は、オートデスク製品の購入やライセンス、アクティベーションについてガイドする運用で利用されています。また、あいにく、対話は英語に限定されてしまっています。</p>
<p>一方、ChatGPT&#0160; は日本語に対応しているので、日本語を使った対話を楽しむことが出来ます。この ChatGPT、オートデスクとは直接関係がないのですが、昨年11月末の公開以来、かなり話題になっているようなので少し試してみました。ChatGPT を利用するためのアカウントの作成方法や利用方法は、ネット上にさまざまな記事が見受けられますので、ここでは割愛させてください。</p>
<p>まずは、日本語でオートデスクに特化した内容を次のように聞いてみました。</p>
<blockquote>
<p><strong>質問：<br /></strong>Autodesk Platform Services の API について教えてください。どのような種類の API がありますか？</p>
</blockquote>
<p>これに対しての応答は次のとおりです。</p>

<blockquote>
<p><strong>応答：<br /></strong>Autodesk Platform Services の API には、様々なタイプの API があります。</p>
<ul>
<li>
<p>Data Management API: Autodesk のクラウド・データ・ストレージ・サービスを利用して、プロジェクト・データを管理するための API です。この API を使用することで、プロジェクト・データをアップロード、ダウンロード、検索、および管理することができます。</p>
</li>
<li>
<p>Model Derivative API: Autodesk のモデル・デリバティブ・サービスを利用して、3D モデルを変換、抽出、および可視化するための API です。この API を使用することで、3D モデルをスキャン、STL、OBJ、FBX、glTFなどのフォーマットに変換することができます。</p>
</li>
<li>
<p>Design Automation API: Autodesk のデザイン・オートメーション・サービスを利用して、自動化されたワークフローを構築するための API です。この API を使用することで、CAD ツールを利用して設計作業を自動化することができます。</p>
</li>
<li>
<p>Reality Capture API: Autodesk のリアリティ・キャプチャ・サービスを利用して、3D モデルを生成するための APIです。このAPIを使用することで、スキャンデータから3Dモデルを生成することができます。</p>
</li>
</ul>
<p>これらの API のほかにも、Viewer、Forge BIM360、Forge WebhooksなどのAPIがあります。</p>
</blockquote>

<p style="padding-left: 40px;"><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02af1c981236200d-pi" style="display: inline;"><img alt="Aps_api" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02af1c981236200d image-full img-responsive" src="/assets/image_783293.jpg" title="Aps_api" /></a></p>
<p>一部、表現に補足が必要な箇所があるように感じますが、さも、一般的な話題のように違和感のない応答があることに驚きます。応答内容は都度変化するようなので、「参考」として見るべきと思いますが、使われている言葉の言い回しが自然で機械的な応答には見えません。単なるチャットボット以上の可能性を感じます。</p>
<p>ChatGPT が話題になっているのは、プログラム コードについても応答を得ることが出来るためです。そこで、少し意地悪ですが、AutoCAD 固有の AutoLISP について少しマニアックな質問をしてみました。</p>
<blockquote>
<p><strong>質問：<br /></strong>AutoCADにAutoLISPというAPIがありますが、(entmake)関数を使って、”基準線”という画層に指定した始点と終点を持つ線分を作図するを使ったコードが欲しいのですが、わかりますか？</p>
</blockquote>
<p>こういった内容に対する応答も画一的な応答ではなく、毎回内容が変わります。今回は次の応答がありました。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02af1c9812f9200d-pi" style="display: inline;"><img alt="Chat1" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02af1c9812f9200d image-full img-responsive" src="/assets/image_732703.jpg" title="Chat1" /></a></p>
<p>コマンド定義にあたるコードは割愛されているので、このままカスタム コマンドとして実行することは出来ませんが、コマンド定義内の実装コードとして見ると、概ね期待した内容になっているように思います。質問内容にヒントになるキーワードや文言を入れていないにもかかわらず、適切な DXF グループコードやリスト操作をしていて、AI の進化を直じかに感じます。</p>
<p>次に、先ほどと同じ処理について別言語の質問をしてみました。もちろん、世間一般ではニッチな部類の内容と思います。</p>
<blockquote>
<p><strong>質問：</strong><br />AutoCAD .NET APIで”基準線”という画層に指定した始点と終点を持つ線分を作図するを使ったC#コードが欲しいのですが？</p>
</blockquote>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02af1c981317200d-pi" style="display: inline;"><img alt="Chat2" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02af1c981317200d image-full img-responsive" src="/assets/image_413208.jpg" title="Chat2" /></a></p>
<p>AutoCAD .NET API をご存じの方にはご理解いただけると思いますが、ここまでくると、少し怖い気もします。応答スピードも十分と感じます。</p>
<p>厳密にいうと、ここで返された応答内容にはコマンドを定義する属性記述がないので、このままカスタム コマンドとして動作はしません。ただ、先の AutoLISP の応答も含め、すでに同 API の知識があれば、適切にコードを改修して使用出来るようにすることも可能と思います。</p>
<p>さて、なぜ ChatGPT について記事を書いたかというと、オートデスク製品に関連して、ChatGPT が応答した内容やコードに対する動画や問い合わせが散見されるようになってきたからです。</p>
<p>繰り返しになりますが、オートデスクが直接関与しているわけではないので、応答されたコードや内容に改修や補足、補正が必要な場合があります。その点はご理解いただきたいと思います。</p>
<p>もちろん 、OpenAI は制限事項や免責事項を記した<a href="https://openai.com/terms/" rel="noopener" target="_blank">利用規約（英語）</a>を用意していますので、評価前によく確認しておくべきでしょう。</p>
<p>いずれにしても、近未来を垣間見るような興味深い体験です。</p>
