<p>諸藩の事情により、日本でのご案内が遅れておりましたが、準備が整いましたので、改めてご案内させていただきます。</p>
<p>非常に大きなモデルで Forge Viewer を使用している場合は、パフォーマンスとスケーラビリティの問題が発生している可能性があります。例えば、ビューアの読み込みに時間がかかりすぎていたり、長い待ち時間の後にブラウザがクラッシュしたりする可能性があります。</p>
<p>SVF2 と呼ばれる新しいパフォーマンス強化バージョンの SVF をお使いいただくと、これらのタイプの問題の多くを解消出来る可能性があります。昨年の Forge DevCon Las Vegas での 基調講演をお見逃しの場合には、<strong><a href="https://www.youtube.com/watch?time_continue=1&amp;v=c8-AxaoHDlk&amp;t=22m5s" rel="noopener" target="_blank">こちら</a></strong>から収録動画にアクセスして、当時 OTG と呼ばれていた 内容をご確認いただくことが出来ます。SVF2 形式と更新された Forge Viewer の使用によって、モデル内に同じ形状のコンポーネントが含まれている状況でも、SVF に比較してパフォーマンスが向上します。基調講演では、鉄筋のパフォーマンスが大幅に向上している点をご理解いただけるはずです。SVF2 に効果を期待出来るのは、非常に多くのコンポーネントを含む建築モデルや製造モデルです。</p>
<p>このブログ記事は、SVF2 評価を目的としたプライベートベータとしての最初のステップです。大規模データでベータ版をテストしてフィードバックをご提供ください。もじ、前提条件を満たし、評価使用に同意、定期的にフィードバックをチームに提供できる場合は、ぜひご連絡ください。</p>
<p><strong>前提条件：</strong></p>
<ul>
<li>Forge Viewer および Model Derivative API に精通している方</li>
<li>大規模データをお持ちでパフォーマンスを向上させたい方</li>
</ul>
<p>上記の前提条件を満たしていて、この機会に興味をお持ちいただける場合には、メールタイトルに「Interested in SVF2 beta Testing」とご記入の上、次の情報を記載して <a href="mailto:forge.svf2.testers@autodesk.com">forge.svf2.testers@autodesk.com</a> 宛にメールでお問い合わせください（英語）。</p>
<ul>
<li>お名前</li>
<li>メールアドレス</li>
<li>会社名</li>
<li>会社の URL</li>
<li>業種（製造、建設など）</li>
<li>国（またはタイムゾーン-電話でお話させていただく必要がある場合）：例） Japan、+81-3-XXXX-YYYY</li>

</ul>
