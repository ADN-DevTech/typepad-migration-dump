<p>このブログ記事には、使用する RESTful API の&#0160;endpoint 毎に、Postman のコレクションを記載していますので、順次、ダウンロードして Postman にインポートしてお使いください。コレクションのインポートは、Postman ウィンドウ左上の [Import] ボタンから実行することが出来ます。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b0282e15358b3200b-pi" style="display: inline;"><img alt="Import_collection" border="0" class="asset  asset-image at-xid-6a0167607c2431970b0282e15358b3200b image-full img-responsive" src="/assets/image_560645.jpg" title="Import_collection" /></a></p>
<p>なお、コレクション内部に指定されているアクセス トークン（Access Token）や Bucket 名（Bucket Key）、アップロードするファイル（Object Key）などは、適宜、ご自身の環境でお使いになるものに書き換えてください。&#0160;</p>
<p>実際に RESTful API で endpoint 毎の処理をテストする手順は次のとおりです。</p>
<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b022ad3738c42200c-pi" style="display: inline;"><img alt="Viewer_steps" border="0" class="asset  asset-image at-xid-6a0167607c2431970b022ad3738c42200c image-full img-responsive" src="/assets/image_70990.jpg" title="Viewer_steps" /></a></p>
<p>それでは、順に endpoint と処理内容を見ていきます。</p>
<p><strong>1. <a href="https://forge.autodesk.com/" rel="noopener noreferrer" target="_blank">forge.autodesk.com</a>&#0160;からアカウント登録とアプリの作成 （手動）</strong></p>
<p style="padding-left: 30px;">Forge&#0160; ポータルでアプリを登録して Forge Platform API の利用に必要な Client ID（Consumer Key）と Client Secret（Consymer Secret）を取得します。この処理は、手動でおこなう必要があります。具体的な手順は、過去のブログ記事 <strong><a href="http://adndevblog.typepad.com/technology_perspective/2016/07/app-registration-and-getting-keys-for-forge-api.html" rel="noopener noreferrer" target="_blank">Forge API を利用するアプリの登録とキーの取得</a></strong>&#0160;を参考にしてください。</p>
<p><strong>2.&#0160;アクセス トークンの取得</strong></p>
<p style="padding-left: 30px;">Forge Platform API で RESTful API を使用する場合には、1. で取得した Client ID（Consumer Key）と Client Secret（Consymer Secret）から Access Token を取得して呼び出しに利用する必要があります。この処理には、<strong><a href="https://developer.autodesk.com/en/docs/oauth/v2/overview/" rel="noopener noreferrer" target="_blank">Authentication API</a></strong>&#0160;を利用します。</p>
<p style="padding-left: 30px;"><span style="text-decoration: line-through;">今回は、2-legged 認証で Forge が提供する OSS（Object Storage Service）の領域にアクセスするので、使用する endpoint は<strong> <a href="https://developer.api.autodesk.com/authentication/v1/authenticate" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/authentication/v1/authenticate</a></strong>、メソッドは <strong>POST</strong>&#0160;となります。</span></p>
<ul>
<li><span style="background-color: #ffff00;">（2023年7月追記）</span>Authentication API（OAuth API）は v1 から v2 に移行しています。詳細は、<strong><a href="https://adndevblog.typepad.com/technology_perspective/2023/05/migration-guide-oauth2-v1-to-v2.html" rel="noopener" target="_blank">OAuth2 v1 から v2 への移行ガイド</a></strong>&#0160;の記事を確認してください。</li>
</ul>
<p style="padding-left: 30px;">開発中のアプリ（同一の Client ID/Consumer Keyと Client Secret/Consymer Secret）で作成したすべての Bucket を列挙するには、endpoint <strong><a href="https://developer.autodesk.com/en/docs/data/v2/reference/http/buckets-GET/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/oss/v2/buckets</a></strong>、メソッド<strong><strong> GET&#0160;</strong></strong>が利用可能です。</p>
<p style="padding-left: 60px;"><span style="text-decoration: line-through;"><strong> <span class="asset  asset-generic at-xid-6a0167607c2431970b01b8d222ac4b970c img-responsive"><a href="http://adndevblog.typepad.com/files/2.-%E3%82%A2%E3%82%AF%E3%82%BB%E3%82%B9-%E3%83%88%E3%83%BC%E3%82%AF%E3%83%B3%E3%81%AE%E5%8F%96%E5%BE%97.postman_collection-1.json">2. アクセス トークンの取得.postman_collectionをダウンロード</a></span></strong>&#0160;</span></p>
<p style="padding-left: 30px;">Authentication API（OAuth API）v2 Postman コレクションは、<strong><a href="https://adndevblog.typepad.com/technology_perspective/2023/07/2lo-v2-on-postman.html" rel="noopener" target="_blank">Postman を使った 2-legged 認証(OAuth v2)</a></strong>&#0160;からダウンロードすることが出来ます。&#0160;</p>
<p style="padding-left: 30px;">ダウンロードした Postman コレクションを使った Access Token の取得の手順を次の動画で確認することが出来ます。</p>
<p><strong>3.&#0160;バケットの作成</strong></p>
<p style="padding-left: 30px;">2-Legged 認証では OSS 上に一意な名前で Bucket を作成して、ファイルをアップロード、変換を経て、Viewer への表示を準備します。Bucket の作成とデザイン ファイルのアップロードには、<strong><a href="https://developer.autodesk.com/en/docs/data/v2" rel="noopener noreferrer" target="_blank">Data Management API</a> </strong>を使用します。Bucket &#0160;作成時には、名称とともに Bucket Policy を指定する必要があります。詳細は、過去のブログ記事 <strong><a href="http://adndevblog.typepad.com/technology_perspective/2016/07/summary-about-bucket.html" rel="noopener noreferrer" target="_blank">Bucket に関してのサマリー</a></strong> をご確認ください。</p>
<p style="padding-left: 30px;">Bucket の作成に使用する endpoint は <strong><a href="https://developer.autodesk.com/en/docs/data/v2/reference/http/buckets-POST/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/oss/v2/buckets</a></strong>、メソッドは <strong>POST</strong> となります。</p>
<p style="padding-left: 30px;">既に登録されている Bucket について、特定の Bucket が存在するかを知るためには、endpoint <strong><a href="https://developer.autodesk.com/en/docs/data/v2/reference/http/buckets-:bucketKey-details-GET/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/oss/v2/buckets/:bucketKey/details</a></strong>、メソッド<strong> GET</strong>&#0160;を利用することが出来ます。上記 <a href="https://ja.wikipedia.org/wiki/Uniform_Resource_Identifier" rel="noopener noreferrer" target="_blank">URI</a> 中の <strong>:bucketKey</strong> 部は照会したい Bucket 名で置き換えてください。この endpoint は、Bucket 内のデザイン ファイルの列挙にも利用することが可能です。</p>

<p style="padding-left: 30px;"><span class="asset  asset-generic at-xid-6a0167607c2431970b01bb093c4631970d img-responsive">ダウンロードした Postman コレクションを使って Bucket を作成したり、有無をチェックしたりする方法を次の動画で確認することが出来ます。</span></p>

<p><strong>4.&#0160;デザイン ファイルのアップロード</strong></p>
<p style="padding-left: 30px;">Bucket を新規作成、または、既存 Bucket の情報が取得出来たら、Viewer で表示させたいデザイン ファイルをアップロードします。<span style="text-decoration: line-through;">アップロードに使用する endpoint は<a href="https://developer.autodesk.com/en/docs/data/v2/reference/http/buckets-:bucketKey-objects-:objectName-PUT/" rel="noopener noreferrer" target="_blank">&#0160;<strong>https://developer.api.autodesk.com/oss/v2/buckets/:bucketKey/objects/:objectName</strong></a><strong>&#0160;</strong>で、メソッドは <strong>PUT</strong> です。上記 <a href="https://ja.wikipedia.org/wiki/Uniform_Resource_Identifier" rel="noopener noreferrer" target="_blank">URI</a> 中の <strong>:bucketKey</strong> と<strong> :objectName </strong>部分は、それぞれアップロード先となる Bucket 名とアップロードするファイル名を <strong><a href="https://www.urlencoder.org/" rel="noopener noreferrer" target="_blank">URL エンコード</a></strong> された値に置き換えてください。</span></p>
<ul>
<li class="asset-video">2019年2月末から、リクエスト ヘッダーに &#0160;<strong>Content-Length</strong> 情報を含めてアップロードするファイルのサイズを明示化することが必須になります。Postman は自動的に &#0160;<strong>Content-Length</strong> ヘッダー情報を付加するので、ここでは割愛していますが、実際にコーディングする際には注意が必要です。詳細は、<strong><a href="https://adndevblog.typepad.com/technology_perspective/2019/02/forge-api-enforcing-content-length-header.html" rel="noopener" target="_blank">Content-length ヘッダー指定要件の必須化</a></strong> 記事をご確認ください<span style="background-color: #ffff00;">（2019年4月追記）</span>。</li>
<li class="asset-video">ファイル アップロードは、新しい endpoint を使った AWS S3 への直接アップロードが2022年3月ににアナウンスされ、使用が推奨されています。詳細は、<a href="https://adndevblog.typepad.com/technology_perspective/2022/03/data-management-oss-object-storage-service-migrating-to-direct-to-s3-approach.html" rel="noopener" target="_blank"><strong>Data Management OSS (Object Storage Service) の Direct-to-S3 アプローチへの移行について</strong></a> をご確認ください。<span style="background-color: #ffff00;">（2022年4月追記）</span>。</li>
</ul>
<p style="padding-left: 30px;">Postman でファイルをアップロード指定する際には、 Header タブで指定する&#0160;Content-Type パラメータに&#0160;application/octet-stream を指定してください。また、Body タブでは、binary チェックボックスを有効にすると [ファイル選択] ボタンが表示されるので、アップロードするファイルを表示されるファイル ダイアログで選択出来ます。</p>
<p style="padding-left: 30px;"><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b8d249079a970c-pi" style="display: inline;"><img alt="Specify_a_file" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01b8d249079a970c image-full img-responsive" src="/assets/image_87716.jpg" title="Specify_a_file" /></a></p>
<p style="padding-left: 30px;"><span class="asset  asset-generic at-xid-6a0167607c2431970b02a308d7ab31200c img-responsive"><a href="https://adndevblog.typepad.com/files/4.-%E3%83%87%E3%82%B6%E3%82%A4%E3%83%B3-%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%81%AE%E3%82%A2%E3%83%83%E3%83%97%E3%83%AD%E3%83%BC%E3%83%89-%E3%83%80%E3%82%A6%E3%83%B3%E3%83%AD%E3%83%BC%E3%83%89.postman_collection-1.json"><strong>4. デザイン ファイルのアップロード-（ダウンロード）.postman_collectionをダウンロード</strong></a></span></p>

<p><strong>5.&#0160;変換リクエスト</strong></p>
<p style="padding-left: 30px;">Bucket にデザイン ファイルがアップロードが完了したら、Viewer でストリーミング表示出来るように ファイルを変換します。変換に利用するのは<strong> <a href="https://developer.autodesk.com/en/docs/model-derivative/v2" rel="noopener noreferrer" target="_blank">Model Derivative API</a></strong> です。</p>
<p style="padding-left: 30px;">Bucket 内のデザイン ファイルの変換指示に使用するのは、endpoint <strong><a href="https://developer.autodesk.com/en/docs/model-derivative/v2/reference/http/job-POST/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/modelderivative/v2/designdata/job</a></strong>、メソッド <strong>POST</strong> です。</p>
<p style="padding-left: 30px;">なお、この endpoint を利用する際には、変換対象のデザイン ファイルは <strong><a href="https://ja.wikipedia.org/wiki/Uniform_Resource_Name" rel="noopener noreferrer" target="_blank">URN</a></strong> と呼ばれるドキュメント ID を <strong><a href="https://ja.wikipedia.org/wiki/Base64" rel="noopener noreferrer" target="_blank" title="https://ja.wikipedia.org/wiki/Base64">BASE64&#0160;エンコード</a></strong> して指定することが義務付けられています。Postman 利用時には、ファイル アップロード時に返された JSON レスポンスから <strong>objectId</strong> 部分を <strong><a href="https://www.base64encode.org/" rel="noopener noreferrer" target="_blank">https://www.base64encode.org/</a></strong>&#0160;サイト等で変換して変換リクエストに利用してください。</p>
<p style="padding-left: 30px;">例えば、fpd-japan-sample-bucket 名の Bucket に&#0160;Bike frame.f3d （エンコード時&#0160;Bike%20frame.f3d）をアップロードした際には、次のような JSON レスポンス Body を得ることが出来ます。</p>
<p style="padding-left: 30px;">{<br />&quot;bucketKey&quot;: &quot;fpd-japan-sample-bucket&quot;,<br />&quot;objectId&quot;: &quot;<strong>urn:adsk.objects:os.object:fpd-japan-sample-bucket/Bike%20frame.f3d</strong>&quot;,<br />&quot;objectKey&quot;: &quot;Bike frame.f3d&quot;,<br />&quot;sha1&quot;: &quot;aff3baa7e6ddd2053f669632ee69412bc70399fa&quot;,<br />&quot;size&quot;: 9916386,<br />&quot;contentType&quot;: &quot;application/octet-stream&quot;,<br />&quot;location&quot;: &quot;https://developer.api.autodesk.com/oss/v2/buckets/fpd-japan-sample-bucket/objects/Bike%20frame.f3d&quot;<br />}</p>
<p style="padding-left: 30px;">ここでは、<strong>urn:adsk.objects:os.object:fpd-japan-sample-bucket/Bike%20frame.f3d</strong> が URN となるので、Base64 に変換して、<strong>dXJuOmFkc2sub2JqZWN0czpvcy5vYmplY3Q6ZnBkLWphcGFuLXNhbXBsZS1idWNrZXQvQmlrZSUyMGZyYW1lLmYzZA==</strong>&#0160;で変換指定をおこなうことになります。&#0160;</p>
<p style="padding-left: 30px;">URN を含め、SVF ファイルへの変換指定は、クライアントからの JSON 指定となります。Postman では Body タブで row チェックボックスを選択して直接 JSON を貼り付けて実行します。</p>
<p style="padding-left: 30px;"><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01bb093c51ff970d-pi" style="display: inline;"><img alt="Convert_a_file" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01bb093c51ff970d image-full img-responsive" src="/assets/image_351997.jpg" title="Convert_a_file" /></a></p>
<p style="padding-left: 30px;">この endpoint を実行した際の JSON レスポンス内の result パラメータに&#0160;success が返れば、変換処理が正常に起動できたことになります。</p>
<p style="padding-left: 30px;">なお、変換処理はすぐに完了するものではありません。<strong><a href="https://ja.wikipedia.org/wiki/%E3%83%9D%E3%83%BC%E3%83%AA%E3%83%B3%E3%82%B0_(%E6%83%85%E5%A0%B1)" rel="noopener noreferrer" target="_blank">ポーリング</a>&#0160;</strong>を実施して、変換処理が完了するかチェックする必要があります。ポーリングに利用するのは、endpoint <strong><a href="https://developer.autodesk.com/en/docs/model-derivative/v2/reference/http/urn-manifest-GET/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/modelderivative/v2/designdata/:urn/manifest</a></strong>、メソッド <strong>GET</strong> です。JSON レスポンスの status パラメータ値で変換状況を、progress パラメータ値で進捗状況を検出することが出来ます。</p>
<p style="padding-left: 30px;"><img alt="Converted_result" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01b7c898d8a5970b image-full img-responsive" src="/assets/image_845099.jpg" title="Converted_result" /></p>
<p style="padding-left: 30px;">ストリーミング配信用に利用される内部ファイル形式は、SVF ファイルです。アップロードするファイル形式を SVF ファイル形式に変換可能か否かは、endpoint&#0160; <strong><a href="https://developer.autodesk.com/en/docs/model-derivative/v2/reference/http/formats-GET/" rel="noopener noreferrer" target="_blank">https://developer.api.autodesk.com/modelderivative/v2/designdata/formats</a></strong>、メソッド <strong>GET</strong> を利用します。JSON で返された SVF スコープ内に、該当するファイル拡張子があれば、そのデザイン ファイルを変換して Viewer に表示出来ることになります。</p>
<p style="padding-left: 60px;"><strong><span class="asset  asset-generic at-xid-6a0167607c2431970b01b8d2298eeb970c img-responsive"> <span class="asset  asset-generic at-xid-6a0167607c2431970b0278807ac84b200d img-responsive"><a href="https://adndevblog.typepad.com/files/5.-%E5%A4%89%E6%8F%9B%E3%83%AA%E3%82%AF%E3%82%A8%E3%82%B9%E3%83%88.postman_collection-2.json">5. 変換リクエスト.postman_collectionをダウンロード</a></span></span></strong></p>
<p style="padding-left: 30px;"><span class="asset  asset-generic at-xid-6a0167607c2431970b01b8d2298eeb970c img-responsive">ダウンロードした Poatman コレクションを使ってデザイン ファイルを変換したり、変換状況をチェックをリストする手順を次の動画で確認することが出来ます。</span></p>

<p><strong>6.&#0160;クライアントからのアクセス</strong></p>
<p style="padding-left: 30px;">ここまでの手順で Viewer に表示する準備が整ったことになります。Viewer は JavaScript API でるため、Postman ではなく、表示に必要な処理を埋め込んだ HTML を用意する必要があります。</p>
