<p>Autodesk Developer Networkの工藤　暁です。今回は3ds Maxテクニカルブログが開設されましたので御紹介します。</p>
<p>このブログは<a href="http://getcoreinterface.typepad.com/">GetCOREInterface</a>と題しＡＤＮエンジニアのKevin Vandecarにより運営されるものです。既に幾つかの記事がアップされておりますが、.NET APIに関する記事とサンプルコードが存在しますので、是非お立ち寄り下さい。</p>
<p>09/20/2013 &quot;Capturing viewport input using the .NET API&quot;</p>
<p>09/17/2013 &quot;Some .NET API Samples&quot;</p>
<p>既に、.NET APIをお試しの方もいるかと存じますが、.NET APIの使用には注意事項が御座いますので御紹介させて下さい。それは3ds Max SDK Reference &gt; What’s<br />New: 3ds 2013 SDK &gt; <a href="http://docs.autodesk.com/3DSMAX/16/ENU/3ds-Max-SDK-Programmer-Guide/index.html?url=files/GUID-B4DA0A04-F442-4E57-9165-30D3827090F7.htm,topicNumber=d30e7283">3ds Max.NET SDK Changes</a> に以下の様に記述されております。</p>
<p><em>Other changes: </em></p>
<ul>
<li><em>We recommend that SDK users do not write plug-ins that derive from classes in the </em><em>Autodesk.Max.Plugins</em><em> namespace. We support custom user actions and<br />tools or utilities loaded by the .NET assembly loader. </em></li>
</ul>
<p>&#0160;これは名前空間Autodesk.Max.Pluginsのクラスを継承したプラグインを作成する事を推奨せず、.NET assemblyによるユーティリティーツールに使用する様お願いするのです。 </p>
<p>将来この制限が無くなった際は、此方のブログでもお知らせさせて頂きます。</p>
<p>&#0160;今後とも<a href="http://getcoreinterface.typepad.com/">GetCOREInterface</a>共々宜しくお願い致します。</p>
