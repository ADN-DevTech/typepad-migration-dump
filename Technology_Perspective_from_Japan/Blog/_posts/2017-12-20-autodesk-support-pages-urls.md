<p><a class="asset-img-link" href="http://adndevblog.typepad.com/.a/6a0167607c2431970b01b7c93c8f04970b-pi" style="float: right;"><img alt="Web_search" class="asset  asset-image at-xid-6a0167607c2431970b01b7c93c8f04970b img-responsive" src="/assets/image_282773.jpg" style="margin: 0px 0px 5px 5px;" title="Web_search" /></a>オートデスクの Web ページは&#0160;<strong><a href="http://www.autodesk.co.jp/" rel="noopener noreferrer" target="_blank">http://www.autodesk.co.jp</a></strong>&#0160;の URL で検索することが出来ますが、一部のページに移動していくと、URL がページ ID（英数字で構成された分かり難いもの）に変化してしまうものがあります。</p>
<p>例えば、デスクトップ製品をまとめた&#0160;<strong>オートデスク デベロッパーネットワーク（ADN）</strong>&#0160;&gt;&gt;&#0160;<strong>プラットフォームテクノロジー</strong>&#0160;のページを表示させると、URL が&#0160;<strong><a href="http://www.autodesk.co.jp/adsk/servlet/index?id=8084359&amp;siteID=1169823" rel="noopener noreferrer" target="_blank">http://www.autodesk.co.jp/adsk/servlet/index?id=8084359&amp;siteID=1169823</a></strong>&#0160;と表示されるはずです。Web 管理システムが自動的に URL を生成してしまうのが理由ですが、このような URL を第三者に伝達しようとする場合、とても不便です。</p>
<p>今後、利用する Web 管理システムの変更にともない、このような URL 生成はなくなっていくはずですが、まだ少し時間がかかるかと思います。</p>
<p>そこで、余談ではありますが、今回はサポートや開発にかかわる URL や検索についてご案内します。</p>
<p>オートデスク Web ページには、ページ ID に代わって人間が記憶しやすい URL に置き換える仕組みが導入されています。先の プラットフォームテクノロジー ページは、<a href="http://www.autodesk.co.jp/developerplatforms" rel="noopener noreferrer" target="_blank"><strong>http://www.autodesk.co.jp/developerplatforms</strong></a>&#0160;で表示させることも出来ます。後者は、オートデスクが意図的に設定した&#0160;<strong>Marketing URL</strong>、通称、<strong>MURL</strong>&#0160;と呼ばれています。</p>
<p>サポートや開発にかかわるページは、あまり話題になりくい場合もありますが、まずは覚えておくと便利な MURL を記しておきたいと思います。</p>

<p>なお、中には、既に新システムで作成されているページも存在します。AutoCAD のトップページのように、<strong><a href="http://www.autodesk.co.jp/autocad" rel="noopener noreferrer" target="_blank">http://www.autodesk.co.jp/autocad</a></strong>&#0160;と MURL を入力すると、<a href="http://www.autodesk.co.jp/products/autocad/overview" rel="noopener noreferrer" target="_blank"><strong>http://www.autodesk.co.jp/products/autocad/overview</strong></a>&#0160;のように、判別しやすい URL が構成されはずです。</p>
<p><strong><span style="background-color: #ffff00;">さて、AKN の記事やこのブログ記事は、Google や Microsoft Bing など、一般の Web 検索エンジンを使ったキーワード検索が出来るように調整されています。普通にキーワードを組み合わせて検索すれば、対象をはるかに見つけやすいはずです。URL を直接伝えて検索してもらう方法もいいですが、キーワード検索もお忘れなく。</span></strong></p>
<p>意図的にキーワード検索を意識させるようなサービスも存在します。<strong><a href="http://lmgtfy.com/" rel="noopener noreferrer" target="_blank">http://lmgtfy.com/</a></strong>&#0160;です。例えば、このサービスを使って、Forge についてのブログ記事を検索する場合には、キーワード検索を模倣する URL をリンクとして利用することが出来ます。次のリンクをクリックして、何が起こるか確認してみてください。</p>
<p><strong><a href="http://lmgtfy.com/?q=autodesk+perspective+forge#" rel="noopener noreferrer" target="_blank">http://lmgtfy.com/?q=autodesk+perspective+forge#</a>&#0160;</strong></p>
<p>AutoCAD のライセンスエラーについて、相手に AKN 記事を含む内容を検索させたい場合には、</p>
<p><strong><a href="http://lmgtfy.com/?q=autocad+ライセンスエラー#" rel="noopener noreferrer" target="_blank">http://lmgtfy.com/?q=autocad+ライセンスエラー#</a></strong></p>
<p>のリンクをメールに貼り付けて送ってあげると、メールを受け取った人が直接検索結果を見ることが出来るばかりでなく、以後 Web 検索を積極的に使ってくれるようになるかも知れません。無償なので不要な広告が表示されてしまいますが、Web ページや Email 内からキーワード検索へ誘導するには便利です。</p>
