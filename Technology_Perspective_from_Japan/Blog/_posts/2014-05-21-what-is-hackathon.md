<p><strong>ハッカソン</strong> というイベントをご存じでしょうか？</p>
<p>英語で表記すると <strong>Hackathon</strong>&#0160;となります。この単語は、Hacking（ハッキング）の Hack と Marathon（マラソン）の thon を組み合わせた造語で、決められた時間内に、与えられた新しい API やテクノロジを使ってアイデアを出し合い、実際にプログラムを完成させる、といった開発イベントとして知られています。</p>
<p>もともと米国で発祥したイベントですが、日本では、まだあまり馴染みのないイベントかもしれません。それでも、ネットで「ハッカソン」または「Hackathon」で検索すれば、沢山のイベント レポートが表示されるはずです。</p>
<p><strong>イベントの流れ</strong></p>
<p>一般的なハッカソン イベントでは、まず、Web やFacebook などのソーシャルメディア上のイベント告知で集まった参加者が、主催者やスポンサーが提供する API を使って何が出来るか、簡単な説明を受けます。次に、イベントによって様々なようですが、参加者同士でいくつかのチームを作り、アイデア出しをおこいます。続いて、チームとしてどういったプログラムを作成するか、アイデアを整理、調整していきます。このようなプロセスは、<strong>アイデアソン</strong>&#0160;は呼ばれています。もちろん、この言葉も Idea ＋ Marathon ＝ <strong>Ideathon</strong>&#0160;で、造語です。</p>
<p>アイデアソンが終了したら、プログラムを記述する人、アイコン画像などの作成も含めてユーザインタフェースのデザインをする人、プレゼンを作る人、といった具合にチーム内で分担を決めて作業に取り掛かります。当然、チームリーダーを決めておいたほうがいいでしょう。</p>
<p>時間的に制約のある中でアイデアとプリグラム作成をしなければならないので、結構、しんどい部分もあるかも知れませんが、初めて知り合った者同士、団結して目的に慢心することが出来るユニークなイベントだと思います。</p>
<p>作業に必要となるコンピュータは参加者が慣れ親しんだノートPCやモバイル デバイスなどを持ち込み、開発環境も特に指定がないかぎり、自由に選択することが出来ます、逆に言えば、事前に「〇〇&#0160;をインストールしておいてください」とか「△△が必須です」的なもにがないのが一般的です。もちろん、WiFi などインターネットの接続環境や、発表に必要なプロジェクターなどは主催者が用意します。</p>
<p>また、イベント開催中は飲食もゆっくりしていられないので、主催者がドリンクやスナック、お弁当を用意してくれます。つまり、作業をしながらの飲食が可能なわけです。</p>
<p>欧米では、まるまる24時間、一昼夜をかけて実施されるのが一般的なようですが、さずがに日本ではアイデアソンとハッカソンを別の日に分けて、合計2日間、昼間の時間帯だけで開催される事が多いようです。</p>
<p><strong>誰でも参加できます！</strong></p>
<p>さて、開発イベントといっても、提供される API で何ができるのか、そのアイデアが重要です。それに、短い時間で見た目の美しい、洗練されたプログラムが開発できれば達成感も半端ではありません。実際、アイデアソンを主体とする、成果物がプログラムではない、そんなハッカソンもあるようです。</p>
<p>そうです。必ずしも、プログラマーが参加しなければならないイベントではありません。もちろん、プログラムが何を指すのか程度は知っておく必要がありますが、重要なのはチーム内の分担なのです。</p>
<p><strong>なぜ、このブログでハッカソンを説明しているか？</strong></p>
<p>オートデスクは、世界で、そして、日本でも、近いうちにハッカソン イベントを開催する予定です。対象は、既存のデスクトップ製品や、その API ではなく、比較的新しいテクノロジやAPI になるはずです。オートデスクのハッカソンですから、3D モデリングが得意な方に参加いただいてもいいかもしれません。</p>
