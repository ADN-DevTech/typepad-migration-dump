<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b022ad3865bda200c-pi" style="float: right;"><img alt="Tokenflex api" border="0" class="asset  asset-image at-xid-6a0167607c2431970b022ad3865bda200c img-responsive" src="/assets/image_451478.jpg" style="margin: 0px 0px 5px 5px;" title="Tokenflex api" /></a>オートデスクとの間に EBA 契約を締結されているお客様向けに、新しく <strong><a href="https://forge.autodesk.com/en/docs/tokenflex/v1/developers_guide/overview/" rel="noopener" target="_blank">Token Flex API</a></strong> が提供されています。EBA 契約とは Enterprise Business Agreement の略で、大企業ユーザ様向けの包括契約を指しています。EBA 契約を締結したお客様にはオートデスクの製品やクラウドサービス、テクノロジを自由に切り替えてお使いいただける Token Flex と呼ばれるライセンスが提供されることになります。</p>
<p>詳細は、<a href="https://www.autodesk.co.jp/campaigns/premium-plan" rel="noopener" target="_blank">こちらのページ</a>に譲りますが、今回提供を開始した Token Flex API を用いると、契約毎に Token Flex ライセンスで使用されている製品やクラウドサービスの利用状況を照会することが可能となります。<a href="https://knowledge.autodesk.com/ja/customer-service/account-management/account-profile/enterprise-accounts/enterprise-accounts-token-flex-licensing" rel="noopener" target="_blank"></a></p>
<p>Token Flex API は EBA 契約ありきの API なのですが、Forge ポータル上で<strong><a href="https://forge.autodesk.com/developer/documentation" rel="noopener" target="_blank">ドキュメント</a></strong>が公開されているので、念のためご案内しておきたいと思います。</p>
<p>なお、Forge サンプルを記載した <strong><a href="https://github.com/Autodesk-Forge/forge-tokenflex-export" rel="noopener" target="_blank">GitHub リポジトリ</a></strong>上には、当該 API のサンプルも既に記載されていますので、何が可能かご理解いただけるものと思います。</p>
