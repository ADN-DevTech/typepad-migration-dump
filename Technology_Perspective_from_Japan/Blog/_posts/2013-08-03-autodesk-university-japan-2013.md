<p>9月13日（金曜日）に <strong>Autodesk University Japan&#0160;</strong>が開催されます。</p>
<p>
<a class="asset-img-link" href="http://www.autodesk.co.jp/auj" style="display: inline;" target="_blank"><img alt="Autodesk-university-japan-2013-logo-2-line-color-ppt-small" border="0" class="asset  asset-image at-xid-6a0167607c2431970b01910490dc31970c image-full" src="/assets/image_29389.jpg" title="Autodesk-university-japan-2013-logo-2-line-color-ppt-small" /></a></p>
<p>基調講演も含めて、「学びと交流の場」としてたくさんのセッションが用意されています。詳細な情報は専用 Web サイト&#0160;<strong><a href="http://www.autodesk.co.jp/auj" target="_blank">http://www.autodesk.co.jp/auj</a>&#0160;</strong>や、次のリーフレットをご参照ください。</p>
<p>
<strong><span class="asset  asset-generic at-xid-6a0167607c2431970b01910490fdd1970c"><a href="http://adndevblog.typepad.com/files/auj2013-digital-leaflet.pdf">AUJ2013デジタルリーフレット をダウンロード</a></span></strong></p>
<p>ここでは、カスタマイズに関するセッションについてご案内したいと思います。今回の AUJ では、カスタマイズについての情報収集や、はじめて API や カスタマイズを習得される方を対象にして <strong><a href="https://reg34.smp.ne.jp/regist/is?SMPFORM=nerd-ohmep-1a344c30fd27675ed49051128358b3ea" target="_blank">カスタマイズ トラック</a></strong>&#0160;（C トラック）として<strong>計 4 セッション</strong>をご用意しています。</p>
<p style="padding-left: 30px;"><strong>C-1 Inventor カスタマイズとデータ管理のコツ<br /></strong>Inventor API を使ったカスタマイズの初めの一歩をご紹介します。VBA だけでなく、アドインを使ったカスタマイズの概要をご紹介し、適切なデータ管理のカスタマイズのためにVaultとVault API のコツをご紹介します。クラウド時代に向けた新しい環境として、Fusion 360 や PLM 360についても機能とカスタマイズの方向性に言及します。</p>
<p style="padding-left: 30px;"><strong>C-1 Revit マクロから始める Revit カスタマイズ<br /></strong>Revit にマクロ作成機能があることをご存じない方が多くいらっしゃります。せっかくの機能を使いこなすため、Revit 2014と共にインストールされる SharpDevelp でマクロを学んでみませんか?マクロが把握できれば、アドインの作成にカスタマイズの領域を広げていただくこともできます。このセッションでは、クラウドやモバイルを使った BIM モデルの活用についても、デモでご紹介します。クラウド時代に向けた新しいカスタマイズのヒントを見出していただけます。</p>
<p style="padding-left: 30px;"><strong>初めてのAutoCAD アドオンアプリケーション作成<br /></strong>これから AutoCAD をカスタマイズしてアドオン アプリケーションを作成しようとする方に、AutoCAD API の概要と適切なAPI の選定方法をご紹介します。また、移植性にすぐれたAutoCAD .NET API を例にとって、実際の開発方法と手順をデモでお見せし、クラウド時代に向けた要素として AutoCAD 360 や JavaScript API にも言及します。</p>
<p style="padding-left: 30px;"><strong>あなたのコンテンツを Exchange Apps ストアで公開しませんか?<br /></strong>アドイン アプリケーションや電子コンテンツなどを公開、ダウンロードできる Autodesk Exchange Apps ストアに、アプリケーションやブロック/ファミリ ライブラリ、電子書籍などを公開する方法と手順をご紹介します。公開するコンテンツは、無償でも、有償でもかまいません。新しいオートデスクブランドのマーケットプレイスをフル活用するためのコツをお知らせします。</p>
<p>また、Autodesk Developer Network にご加入いただいている開発パートナー様の専用ブースも&#0160;<strong><a href="http://au.autodesk.co.jp/exhibition.html" target="_blank">出展ブース</a></strong>でご覧いただけるようになっています。数多くのカスタム ソリューションが展示されますので、オートデスク製品の特長であるカスタマイズ機能でどんなことが出来るのか、実際に目で確かめて、直接ご相談いただく機会になるはずです。ご参加いただいた際には、ぜひお立ち寄りいただきたいブースです。</p>
<p>さて、<a href="http://adndevblog.typepad.com/technology_perspective/2013/03/autodesk%E3%81%AE%E3%83%96%E3%83%A9%E3%83%B3%E3%83%89%E3%81%8C%E6%96%B0%E3%81%97%E3%81%8F%E3%81%AA%E3%82%8A%E3%81%BE%E3%81%97%E3%81%9F.html" target="_blank">オートデスクのブランディングの変更</a>に合わせて、AU ロゴも一新されています。3D プリント用に <strong><a href="http://ja.wikipedia.org/wiki/Standard_Triangulated_Language" target="_blank">STL ファイル</a></strong>が用意されているようなので、今話題の 3D プリンタで出力できるはずです。当日、出力されたモデルが見れるのでは、と期待しています。</p>
