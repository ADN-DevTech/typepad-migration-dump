<p><a href="https://adndevblog.typepad.com/technology_perspective/2023/10/aps-viewer-shown-not-updated.html" rel="noopener" target="_blank">APS Viewer：以前の内容が表示されてしまう</a>&#0160;で触れた問題は、<a href="https://adndevblog.typepad.com/technology_perspective/2023/03/aps-vs-code-extension.html" rel="noopener" target="_blank">Autodesk Platform Services VSCode Extension</a> を使って <a href="https://adndevblog.typepad.com/technology_perspective/2020/09/viewer-workflow-on-vs-code-forge-extension.html" rel="noopener" target="_blank">Viewer ワークフロー</a>を繰り返す中でも遭遇しがちな問題です。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02c8d3a19c44200d-pi" style="display: inline;"><img alt="Not_updated" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02c8d3a19c44200d image-full img-responsive" src="/assets/image_41073.jpg" title="Not_updated" /></a></p>
<p>この場合もキャッシュが原因ですが、あいにく、VS Code 起動中に Autodesk Platform Services VSCode Extension だけが使用するキャッシュのみを削除する方法はありません。</p>
<p>一旦、VS Code を終了する必要がありますが、VS Code のキャッシュを削除することで、Viewer 表示を更新された状態にすることが出来ます。ただし、VS Code の他の機能が使用しているキャッシュも削除してしまうことになるので、その点は事前に理解しておく必要があります。</p>
<p>Windows の場合、<strong>C:\Users\<em>username</em>\AppData\Roaming\Code\Cache </strong>を Windows Exprorer で開いてすべてのファイルを削除後、</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02c8d3a19c94200d-pi" style="display: inline;"><img alt="Cache" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02c8d3a19c94200d image-full img-responsive" src="/assets/image_83882.jpg" title="Cache" /></a></p>
<p>VS Code を起動すれば、シード ファイルの変更が Viewer に反映されるはずです。</p>
