<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02b751ac549d200c-pi" style="display: inline;"><img alt="MFG Data Model API" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02b751ac549d200c image-full img-responsive" src="/assets/image_395093.jpg" title="MFG Data Model API" /></a></p>
<p><a href="https://adndevblog.typepad.com/technology_perspective/2023/06/fusion-data-sample.html" rel="noopener" target="_blank">Fusion Data サンプル</a> でもご案内した Fusion Data API の名称が変更されて、「<strong>Manufacturing Data Model API</strong>」となります。オートデスクは Design and Make（デザインと創造)） プラットフォームを通じて、コラボレーション、データ管理、製品ワークフロー機能の強化を提供していきます。 オートデスク プラットフォームは、Autodesk Data Model を使用して、Design and Make データを管理・整理し、インダストリー クラウド間を接続することで、ビジネスに新たなワークフローと可能性をもたらします。</p>
<p><strong>Manufacturing Data Model</strong>（旧称 Fusion Data）は <strong>Autodesk Data Model</strong> 上に構築されており、製造業向けインダストリー クラウド（Autodesk Fusion）からのデータへのアクセスと共有のための一貫した方法を提供します。この変更は、データ、自動化、インサイト（洞察）を通じてエンドツーエンドのプロセスとワークフローを接続できるプラットフォームを提供するという継続的なコミットメントを反映したものです。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02c1b25ad070200d-pi" style="display: inline;"><img alt="Industry_cloud" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02c1b25ad070200d image-full img-responsive" src="/assets/image_455391.jpg" title="Industry_cloud" /></a></p>
<p>名称が変更されても、現在ご利用のエンドポイント、GraphQL クエリー/ミューテーションに影響はありませんのでご安心ください。Manufacturing Data Model と API の詳細については、<a href="https://aps.autodesk.com/manufacturing-data-model-api">Manufacturing Data Model API</a> をご確認ださい。</p>
<p>By Toshiaki Isezaki</p>
