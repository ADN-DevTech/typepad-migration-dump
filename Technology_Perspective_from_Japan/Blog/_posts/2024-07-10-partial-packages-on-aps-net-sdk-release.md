<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02c8d3b8ea01200b-pi" style="display: inline;"><img alt="Dotnet_sdk" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02c8d3b8ea01200b image-full img-responsive" src="/assets/image_358311.jpg" title="Dotnet_sdk" /></a></p>
<p><a href="https://adndevblog.typepad.com/technology_perspective/2024/06/migrating-to-the-new-aps-net-sdk.html" rel="noopener" target="_blank">新しい APS .NET SDK への移行</a>&#0160;でご案内した新しい<a href="https://www.nuget.org/profiles/AutodeskPlatformServices.SDK" rel="noopener" target="_blank">&#0160;.NET SDK</a>&#0160;について、Authentication API（OAuth API）v2 と Model Derivative API v2 をラップするパッケージが正式にリリースされています。この SDK で特筆すべきところは、それぞれにドキュメントが用意されている点です。</p>
<p>Authentocation API については、エンドポイントのリファレンスと同列に、SDK のリファレンス ドキュメントが用意されいて、名前空間毎にクラス、メソッドの利用方法を把握出来るようになっています。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02dad0c90c8d200d-pi" style="display: inline;"><img alt="Authentication" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02dad0c90c8d200d image-full img-responsive" src="/assets/image_236908.jpg" title="Authentication" /></a></p>
<p>&#0160;Model Derivative API パッケージについても同様です。</p>
<p><a class="asset-img-link" href="https://adndevblog.typepad.com/.a/6a0167607c2431970b02c8d3b8ea3f200b-pi" style="display: inline;"><img alt="Model_derivative" border="0" class="asset  asset-image at-xid-6a0167607c2431970b02c8d3b8ea3f200b image-full img-responsive" src="/assets/image_162160.jpg" title="Model_derivative" /></a></p>
<p>なお、SDK の利用は必須ではありませんのでご注意ください。REST API のエンドポイントを直接呼び出す方法も、もちろん引き続きサポートされています。</p>
<p>By Toshiaki Isezaki</p>
