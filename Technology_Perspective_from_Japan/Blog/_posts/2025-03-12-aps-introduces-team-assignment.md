<p>Autodesk Platform Services (APS）では、新機能であるチーム割り当てを APS ポータル（<a href="https://aps.autodesk.com/" rel="noopener" target="_blank">https://aps.autodesk.com/</a>） に導入し、ポータルで所有する各 APS アプリをチームに割り当てることが出来るようになりました。APS の課金対象 API（プレミアム API）を使用するユーザーの場合、アプリを割り当てるチームにそのアプリに対する課金がおこなわれるようになります。複数のチームのメンバーである APS ユーザーの場合、アプリを正しいチームに割り当てて、アプリに対する課金が正しいチームにすることが重要です。この機能は <strong>3月12日まで&#0160;</strong>にアクセス可能な状態になっているはずです。</p>
<p><strong>必要な対応&#0160;</strong>- Autodesk Platform Services（APS）ユーザーで、複数のチームのメンバーである場合、または APS プレミアム API を使用しているユーザーの場合には、下記の情報をご一読の上、必要な手順の実行をお願いします。APS は、所有するすべてのアプリの既定のチーム割り当てを完了していますが、<strong>2025年3月25日までに</strong>、アプリが正しいチームに割り当てられていることを確認していただく必要があります。また、アプリに対して間違ったチームが請求される可能性があるため、必要に応じてアプリを適切なチームに再割り当てていただくことを強くお勧めします。</p>
<h3 style="font-weight: 800;">所有している各アプリケーションのチーム割り当てを確認するには：</h3>
<ul>
<li>プロファイル メニューの <strong>My applications</strong>&#0160;ページをクリックします。</li>
</ul>

<ul>
<li>Applications ページで所有しているアプリをクリックします。</li>
</ul>

<h2 style="font-weight: 800;">追加情報</h2>
<p><strong>チームとは?</strong></p>
<p><strong><a href="https://www.autodesk.com/jp/support/account/admin/users/manage-teams" rel="noopener" target="_blank">チーム</a></strong>とは、Autodesk Account を通じて管理されるオートデスク全体の概念です。既定では、オートデスク アカウントには、すべてのユーザーとオートデスク サブスクリプションを含むチームが 1 つ関連付けられています。使用用途に応じて複数のチームに追加することもできます。<strong><a href="https://www.autodesk.com/jp/support/account/admin/users/manage-teams" rel="noopener" target="_blank">詳しくはこちらをご覧ください</a></strong>。</p>
<p><strong>チームはどこで確認できますか?</strong></p>
<ul>
<li>ユーザープロファイルの&#0160;<strong>[アカウント]</strong> をクリック後に&#0160;<strong>[設定]</strong> へ移動して [<strong>チームの設定]</strong> を確認します。</li>
<li>または、<a href="https://manage.autodesk.com/home" rel="noopener" target="_blank"><strong>https://manage.autodesk.com/home</strong></a>&#0160;に移動して、左側のナビゲーションから <strong>[サブスクリプションと契約]</strong> をクリックして、所属しているチームを表示します。</li>
</ul>
<p><strong>*&#0160;</strong>サブスクリプションを購入していない場合、または、チームのプライマリ管理者またはセカンダリ管理者ではない場合には、アカウントにチームが表示されない可能性があります。その場合は、メインの管理者にお問い合わせください。詳細については、下記をご覧ください。</p>
<p><strong>アプリをチームに関連付ける利点は何ですか？</strong></p>
<p>APS の複数のチームの一部である場合、特定のチームに異なるアプリを割り当てることがより詳細にコントロール出来ます。これによって、APS の課金対象 API（プレミアム API）を使用している場合に、アプリに請求される適切なチームを選択することが出来ます。&#0160;</p>
<p><strong>既定の割り当てはおこなわれていますか？ また、すべてのアプリにチームを割り当てる必要がありますか？</strong></p>
<p>チームに基づく既定の割り当ては完了しています。複数のチームのメンバーである場合は、上記のようにチームの選択を確認する必要があります。2025年3月26日以降、APS のプレミアム API を使用する場合、アプリに割り当てられたチームが課金されます。</p>
<p><strong>3月25日の締め切り後に、応募に割り当てられたチームを再割り当てすることはできますか?</strong></p>
<p>はい、締め切り後も、いつでもチームの割り当てを変更できます。3月26日以降にアプリケーションに割り当てられたチームが請求がおこなわれます。&#0160;</p>
<p><strong>APS で無料の API のみを使用していますが、この場合でもチームの割り当てを確認する必要がありますか?</strong><br />無料の API を使用している場合にはチームに課金されることはありませんが、すべてのアプリのチームの割り当てを確認することは依然として重要です。チーム割り当て機能を確認して、各アプリの責任者を特定しておくことを強くお勧めします。これにより、さまざまなアプリのチーム割り当てを可視化できます。</p>
<p><strong>チームはチームの割り当て機能自体に表示されますが、manage.autodesk.com でアカウントを確認すると表示されません。なぜこのようなことが起こるのですか？</strong></p>
<p>ここでは、2 つのシナリオが考えられます。</p>
<ul>
<li>APS の無料 API を使用しており、サブスクリプションを購入していないユーザーの場合、manage.autodesk.com にはチームは表示されませんが、チームの割り当て機能には表示されます。これは想定される状態であり、APS の使用には影響しません。</li>
<li>Autodesk Flex、Token Flex サブスクリプション（EBA 契約）、またはクラウド クレジット契約のユーザーで、かつ、プライマリ/セカンダリ管理者でない場合、アカウントにチーム名を表示することは出来ません。ただし、ユーザーとしてチームの一員であることに変わりはありません。プライマリ/セカンダリ管理者に連絡して、チームの名前をご確認ください。</li>
</ul>
<p><strong>APS Developer ハブに存在するアプリはどうなりますか?</strong></p>
<p><a href="https://aps.autodesk.com/blog/aps-developer-hubs-introducing-developer-admin-role-autodesk-platform-services" rel="noopener" target="_blank">APS Developer ハブ</a> に存在するアプリは、ハブを所有するチームに自動的に割り当てられます。メインのチーム管理者に連絡して、ハブを所有しているチームを確認してください。ハブ内のアプリを再割り当てすることは出来ません。</p>
