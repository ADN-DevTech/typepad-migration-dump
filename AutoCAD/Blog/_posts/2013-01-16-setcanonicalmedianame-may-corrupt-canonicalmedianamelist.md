<p>By <a href="http://adndevblog.typepad.com/autocad/gopinath-taget.html">Gopinath Taget</a></p>  <p>The string array (in form of AcArray) returned by the AcDbPlotSettingsValidator::canonicalMediaNameList() method gets reset by the call to the AcDbPlotSettingsValidator::setCanonicalMediaName(). Therefore after the first call﻿﻿﻿, if you try to use the string values from the original string array you will find the values are corrupted.</p>  <p>The solution to this problem ﻿﻿﻿is to simply create your own copy of media names out of the AcArray returned by the canonicalMediaNameList().</p>  <p>Also, you can use the ActiveX interfaces. The code below will iterate through all of the canonicalMediaNames and set each one current.</p>  <p>Note: We have found that the PageSizes are corrupted if RefreshPlotDeviceIno is called and &quot;None&quot; is the current plotter. Another issue is the canonicalMediaNames are not returned when the current device is &quot;None&quot;. This example returns if &quot;None&quot; is the current setting for the device.
