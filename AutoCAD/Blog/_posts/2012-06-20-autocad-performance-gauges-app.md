<p>by <a href="http://adndevblog.typepad.com/autocad/fenton-webb.html">Fenton Webb</a></p>
<p>I wrote this App to demonstrate some basic <a href="http://au.autodesk.com/?nd=event_class&amp;session_id=8964&amp;jid=1725755">AutoCAD Messaging and Threading</a> concepts at Autodesk University last year, I thought you might like to see the App and the code behind it.</p>
<p>Here’s a snap shot of the installed App…</p>
<p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b016767b7cd40970b-pi"><img alt="image" border="0" height="209" src="/assets/image_660893.jpg" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" width="496" /></a></p>
<p>As you can see, there are seven <a href="http://wm.microsoft.com/ms/expression/wpf/WPFMKT_FINAL.wmv">WPF</a> gauges, all hosted in a <a href="http://au.autodesk.com/?nd=material&amp;session_material_id=5572">RibbonCompositeItem</a>, described left to right…</p>
<ul>
<li>Total number of objects being opened for read, per second </li>
<li>Total number of objects being closed, per second</li>
<li>Total number of objects being opened for write, per second</li>
<li>Total number of objects being erased, per second</li>
<li>Total number of objects being deepCloned, per second</li>
<li>Total number of view updates occurring, per second</li>
<li>Total number of DWG files being found on your hard drive, using a .NET Background worker thread.</li>
</ul>
<p>What you might find interesting about this App is the nice smooth interactive feel of the whole thing; the purpose of this demo App is to show how to keep dynamic UI working, whilst keeping the AutoCAD UI responsive. After you have seen it work, I recommend that you take a look at the source code that I have attached to see how it works (the source is&#0160;also included inside of the .bundle Contents/Resource folder) - I hope you like it.</p>
<p>Some topics to note inside the code…</p>
<ul>
<li>Check out the way I post updates to the gauges using <a href="http://msdn.microsoft.com/en-us/library/system.windows.threading.dispatcher.begininvoke(v=vs.95).aspx">Dispatcher.BeginInvoke</a>.</li>
<li>Check out the <strong>RelativeSource.Self</strong> Databinding style I’m using</li>
<li>Look at the power of our Overrule API!</li>
<li>Do not use worker threads with AutoCAD directly, AutoCAD is not multithread aware.</li>
</ul>
<p>Some things that the App doesn’t do at this time…</p>
<ul>
<li>The gauges only work for the start-up document, I have not coded any document switching.</li>
</ul>
<p>Finally, this App also shows how to “Autoload” CUI Custom Ribbon controls, for both AutoCAD 2012 and 2013... One of the features of the Autoloader is the ability for it to load WPF Dll’s away from the acad.exe directory; it does this by implementing an <a href="http://msdn.microsoft.com/en-us/library/system.appdomain.assemblyresolve.aspx">AssemblyResolve</a> event handler. In addition, the Autoloader does the full CUI Custom Ribbon control registration for you – niiiice.</p>
