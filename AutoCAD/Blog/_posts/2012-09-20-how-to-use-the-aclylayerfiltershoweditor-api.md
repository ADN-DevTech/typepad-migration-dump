<p>By <a href="http://adndevblog.typepad.com/autocad/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p><b>Q:</b></p>  <p>I am trying to call from my own arx application the &quot;AcLyLayerFilter::showEditor&quot; method in order to display the native AutoCAD layer filter editor, but it doesn't seem to work, nothing is displayed.</p>  <p><a name="section2"></a></p>  <p><b>A:</b></p>  <p>Unfortunately displaying this native dialog from API is not possible. The purpose of this method is to be called by AutoCAD itself on a custom filter that would derive from AcLyLayerFilter. Depending on the return type that you define in your overriden showEditor method you can display your own filter editor dialog or the native one, when the user is double clicking or invoking this editor on the custom filter.</p>  <p>The attached sample illustrates this behavior. To test it run command &quot;MyCommand1&quot;: it will add a new custom layer filter called &quot;CustomFilter&quot;. When you invoke the properties editor on this filter, the overriden showEditor method will be called.</p>  <p>Below is the code of the command method.
