<p>By <a href="http://adndevblog.typepad.com/autocad/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p><b>Q:</b></p>  <p>I have a large arx application which performs lengthy calculations. When the calculation is running, our clients would normally minimize the AutoCAD window down to the task bar (it can run for days). Occasionally, in order to monitor progress, the AutoCAD window is maximized but the screen within the window is not refreshed. How can I workaround this?</p>  <p><a name="section2"></a></p>  <p><b>A:</b></p>  <p>There are a couple of ways to do what you want.</p>  <p>The first way is multi-threading. I will say that multi-threading isn't recommended because it can have undesirable effects, especially when dealing with the AutoCAD drawing database, and is unsupported.</p>  <p>The second way is to implement your own Command Message Handler - recommended. When your program is executing it takes away the Message handling control from AutoCAD. Because of this when you minimize and maximize the AutoCAD window nothing gets repainted or updated because AutoCAD doesn't get the relevant messages that are sent from the operating system. So, what we need to do is to grab these messages in our 'calculation' loop and then spend some of our calculation time posting them back to AutoCAD and waiting for AutoCAD to process them.</p>  <p style="line-height: normal;
