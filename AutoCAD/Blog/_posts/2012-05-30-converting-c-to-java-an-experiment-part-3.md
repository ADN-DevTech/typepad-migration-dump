<p>By&#0160;<a href="http://adndevblog.typepad.com/autocad/stephen-preston.html">Stephen Preston</a></p>
<ul>
<li>In <a href="http://adndevblog.typepad.com/autocad/2012/05/converting-c-to-java-an-experiment-part-1.html" target="_self">part 1</a> of this series I created a simple task list prioritization tool for AutoCAD written in C#. This was based on two classes: Item and PrioritizerCore.</li>
<li>In <a href="http://adndevblog.typepad.com/autocad/2012/05/converting-c-to-java-an-experiment-part-1-1.html" target="_self">part 2</a>, I setup an Android app project in Eclipse, and migrated my Item class from C# to Java, discovering how very similar the two languages are.</li>
<li>Today, I&#39;ll migrate the PrioritizerCore class. (As for the Item class, the goal today is to get the code to compile in Eclipse - we&#39;ll check the logic still works when we hook the classes into a working Android app).</li>
</ul>
<p>We&#39;d already copied the PrioritizerCore C# code into a PrioritizerCore.java file in our Android project, so we can just jump in and start editing. Before making additional changes, we&#39;ll go through the code and make the same edits as we did for Item without further comment:</p>
<ul>
<li>Change string to String and bool to boolean.</li>
<li>Change LIst to ArrayList.</li>
<li>Change the Serializable attibute to the Java Serializable interface.</li>
<li>Change properties to get/set methods, and change calls to properties in the same way.</li>
<li>Change all function names to start with a lowercase character instead of uppercase.</li>
<li>USe intellisense to reference missing packages.</li>
</ul>
<p>&#0160;That removes the vast majority of compilation errors. Those that remain are new.</p>
<p>The first new error we encounter is the foreach loop - which is really handy for iterating the elements of a collection. The C# code looks like this:</p>
<p><span style="color: #ffffff;">&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;</span><span style="color: #000000;">foreach&#0160;</span><span style="color: #000000;">(</span><span style="color: #000000;">Item&#0160;i&#0160;in&#0160;m_ItemList</span><span style="color: #000000;">)</span></p>
<p>Java has an equivalent, but the syntax is slightly different. We change that line to:</p>
<p><span style="color: #ffffff;">&#0160;&#0160;&#0160;&#0160;&#0160;&#0160;</span><span style="color: #7f0055;"><strong>for&#0160;</strong></span><span style="color: #000000;">(</span><span style="color: #000000;">Item&#0160;i&#0160;:&#0160;m_ItemList</span><span style="color: #000000;">)</span></p>
<p>which I really like - its even more succinct than the C# :-).</p>
<p>The next error is:</p>
<p><span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #7f0055;"><strong>public&#0160;</strong></span><span style="color: #7f0055;"><strong>void&#0160;</strong></span><span style="color: #000000;">sort</span><span style="color: #000000;">()</span><br /> <span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #000000;">{</span><br /> <span style="color: #ffffff;">&#0160;&#0160;&#0160;&#0160;</span><span style="background-color: #ffff00;"><span style="color: #000000;">m_ItemList.sort</span><span style="color: #000000;">(</span><span style="color: #7f0055;"><strong>new&#0160;</strong></span><span style="color: #000000;">ItemComparer</span><span style="color: #000000;">())</span><span style="color: #000000;">;</span></span><br /> <span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #000000;">}</span></p>
<p>Again Java has an equivalent of the sort method and the equivalent of a Comparator. We change our code to:</p>
<p><span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #7f0055;"><strong>public&#0160;</strong></span><span style="color: #7f0055;"><strong>void&#0160;</strong></span><span style="color: #000000;">sort</span><span style="color: #000000;">()</span><br /> <span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #000000;">{</span><br /> <span style="color: #ffffff;">&#0160;&#0160;&#0160;&#0160;</span><span style="color: #000000;">Collections.sort</span><span style="color: #000000;">(</span><span style="color: #000000;">m_ItemList,&#0160;</span><span style="color: #7f0055;"><strong>new&#0160;</strong></span><span style="color: #000000;">ItemComparer</span><span style="color: #000000;">())</span><span style="color: #000000;">;</span><br /> <span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #000000;">}</span></p>
<p>As you&#39;d expect, the next error is in our custom comparator definition. We change&#0160;</p>
<p><span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #7f0055;"><strong>private&#0160;class&#0160;</strong></span><span style="color: #000000;">ItemComparer&#0160;:&#0160;IComparer&lt;Item&gt;</span></p>
<p>to</p>
<p><span style="color: #ffffff;">&#0160;&#0160;</span><span style="color: #7f0055;"><strong>private&#0160;class&#0160;</strong></span><span style="color: #000000;">ItemComparer&#0160;</span><span style="color: #7f0055;"><strong>implements&#0160;</strong></span><span style="color: #000000;">Comparator&lt;Item&gt;</span></p>
<p>&#0160;</p>

<p>Here&#39;s a summary of the migration issues we found today:</p>
<ul>
<li>Java has a &#39;foreach&#39; equivalent -&#0160;<strong>for&#0160;</strong>(Item&#0160;i&#0160;:&#0160;m_ItemList).</li>
<li>The Java &#39;sort&#39; method is a static method of the Collection class instead of an instance method of a specific collection.</li>
<li>The Java equivalent of the C# IComparer interface is IComparator (which I believe is more grammatically correct :-).</li>
</ul>
