<p>By <a href="http://adndevblog.typepad.com/autocad/gopinath-taget.html">Gopinath Taget</a></p>  <p>Consider this: If you derive a custom entity from AcDbPolyline, and select an instance of your entity in AutoCAD, OPM displays &quot;Polyline&quot;.</p>  <p>So, how can you change the type name that appears in the object properties palette window?</p>  <p>In essence, to change the &quot;Entity Type&quot; title in the OPM Dialog you *have* to use Static Properties. It is Static Properties which define the IOPMPropertyExtension::GetDisplayName(DISPID, BSTR*) function and it's in this function that (when receiving a DISPID of 0x401) you can change the title.</p>  <p>Also, if you were to derive your custom entity from AcDbEntity rather than AcDbPolyline then the OPM &quot;Entity Type&quot; would actually appear as the DXF_NAME in your ACRX_DXF_DEFINE_MEMBERS macro. This is because AcDbPolyline defines it's own Static Properties, and, it's own IOPMPropertyExtension::GetDisplayName which updates the DXF_NAME to &quot;Polyline&quot;. You need to override this behavior.
