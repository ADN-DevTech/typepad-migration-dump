<p>By <a href="http://adndevblog.typepad.com/autocad/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p><b>Q:</b></p>  <p>I am having a problem with the <em>AcDbPolyline::getSplitCurves()</em> function. The function is returning a set of entities which is incorrect for the input provided. I call <em>getSplitCurves()</em> on a polyline with two points on the polyline, expecting to get two polylines returned. Instead of getting two polylines, I only get a copy of the original polyline that has two extra vertices added.</p>  <p><a name="section2"></a></p>  <p><b>A:</b></p>  <p>The behavior of <em>getSplitCurves()</em> is affected by the order of the points in the <em>AcGePoint3dArray</em>. If the points are appended to the <em>AcGePoint3dArray</em> in the order that they would be if they were measured from the start point of the polyline then the expected behavior occurs. (two polylines are returned).&#160; The following code snippet uses this approach. It finds the distance on the&#160; curve from the start of the curve to the point, and then add the points to the array in the order of their distance from the start of the curve.&#160; </p>  <p style="line-height: normal;
