<p>By <a href="http://adndevblog.typepad.com/autocad/daniel-du.html">Daniel Du</a></p>  <p>As a practice of Windows Azure and WebGL, I did a simple project, creating an <a href="http://acadcommandwebviewer.cloudapp.net/">AutoCAD Command Usage Statistics “Magic Ball”</a> on Cloud. Hopefully it gives your some inspirations of the possibility of AutoCAD + Cloud integration. The idea is to monitor user’s command usage while he is drafting and save it up to cloud, then view the statistics in web page with a rotatable ball, I call it “magic ball”, it does not look like a ball just because I am lazy not to draft much to use many AutoCAD commands. <img alt="Smile" src="/assets/image_694622.jpg" /></p>  <p><a href="http://acadcommandwebviewer.cloudapp.net/"><img title="image" border="0" alt="image" src="/assets/image_155352.jpg" width="441" height="348" /></a></p>  <p align="center">(click the screen-shot to play with it.)</p>  <p>I posted some articles on Cloud and Mobile Devblog, I cross post it here if you are interested in how it is implemented.</p>  <h5><a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/03/an-autocad-command-usage-statistics-magic-ball-on-cloudpart-1.html">An AutoCAD Command Usage Statistics “Magic Ball” on Cloud–Part 1</a></h5>  <h5><a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/03/an-autocad-command-usage-statistics-magic-ball-on-cloudpart-2.html">An AutoCAD Command Usage Statistics “Magic Ball” on Cloud–Part 2</a></h5>  <h5><a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/03/an-autocad-command-usage-statistics-magic-ball-on-cloudpart-3.html">An AutoCAD Command Usage Statistics “Magic Ball” on Cloud–Part 3</a></h5>  <h5><a href="http://adndevblog.typepad.com/cloud_and_mobile/2013/03/an-autocad-command-usage-statistics-magic-ball-on-cloudpart-4.html">An AutoCAD Command Usage Statistics “Magic Ball” on Cloud–Part 4</a></h5>
