<p>By <a href="http://adndevblog.typepad.com/autocad/adam-nagy.html" target="_self">Adam Nagy</a></p>
<p>From inside my command I&#39;m starting a background thread that synchronizes with a database. Once that finishes I&#39;d like to keep using the AutoCAD .NET API to do some further modifications in the database. Unfortunately, when I&#39;m calling the AutoCAD API functions from this thread things do not seem to work, e.g. the MdiActiveDocument is null.</p>
<p><strong>Solution</strong></p>
<p>The AutoCAD API&#39;s do not support multithreading. You should only call the API functions from the main thread.</p>
<p>If you are in a different thread then you have to marshal the call to the main thread. The simplest way to achieve that is creating a System.Windows.Forms.Control object on the main thread and call its Invoke() function to start the function that is doing the end processing.</p>
<p>Here is a sample to demonstrate this.</p>
