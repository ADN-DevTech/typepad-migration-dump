<p>By <a href="http://adndevblog.typepad.com/autocad/virupaksha-aithal.html" target="_self">Virupaksha Aithal</a></p>
<p>AutoCAD 2017 SP1 is released now (For SP1 click <a href="https://knowledge.autodesk.com/support/autocad/downloads/caas/downloads/content/autocad-2017-service-pack-1.html?v=2017">here</a>). One of the important issues addressed in AutoCAD 2017 SP1 is debugging the .NET in using Visual studio 2015. Till now, developers were asked to use workaround as explained in blog <a href="http://adndevblog.typepad.com/autocad/2016/05/debugging-autocad-2017-using-visual-studio-2015.html">http://adndevblog.typepad.com/autocad/2016/05/debugging-autocad-2017-using-visual-studio-2015.html</a> . But with AutoCAD 2017 SP1, there is no need to use the workarounds to debug the .NET project. Also, now you can edit and continue debugging your .NET project.</p>
