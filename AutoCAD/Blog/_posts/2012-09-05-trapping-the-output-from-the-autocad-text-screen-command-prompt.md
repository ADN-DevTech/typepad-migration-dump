<p>By <a href="http://adndevblog.typepad.com/autocad/philippe-leefsma.html" target="_self">Philippe Leefsma</a></p>  <p><b>Q:</b></p>  <p>How can I obtain the textual output of commands in AutoCAD, such as MASSPROPS as   <br />well as hide the output for certain commands at the command prompt?</p>  <p><a name="section2"></a></p>  <p><b>A:</b></p>  <p>You can temporarily replace the AutoCAD host application services object with   <br />your own. The important callback is consoleMessage(), which you can overload to    <br />behave as you require - either to interpret or discard the calls made through    <br />acutPrintf() to display text on the command-line or text screen. There are other    <br />callbacks that require implementing, and these are to be forwarded to the    <br />original host application services object.</p>  <p>It is very important to understand that this mechanism should only be used for   <br />the shortest length of time possible. AutoCAD makes an assumption about its    <br />standard host application services object - it will cast the pointer to a    <br />non-published, derived class. It is therefore a matter of only replacing the    <br />object as long as is absolutely necessary.</p>  <p>This approach has also not been thoroughly tested for all commands; therefore,   <br />it should be used with caution and will need more complete testing on commands    <br />not tested by this sample. Refer to the attached sample.</p>  <p>The key point is to derive a class from <em>AcDbHostApplicationServices</em> and override some necessary methods.
