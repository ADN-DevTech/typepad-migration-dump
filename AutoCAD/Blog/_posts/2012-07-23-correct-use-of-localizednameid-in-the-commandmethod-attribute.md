<p>By <a href="http://adndevblog.typepad.com/autocad/stephen-preston.html">Stephen Preston</a></p>  <p>Our AutoCAD .NET Wizard project template downloadable from the <a href="http://www.autodesk.com/developautocad" target="_blank">AutoCAD Developer Center</a> includes some boilerplate code to demonstrate how to create various command types. The first command definition looks like this:</p>  <div style="font-family: consolas; background: white; color: black; font-size: 14pt">   <p style="margin: 0px"><font size="2">&#160;&#160;&#160;&#160;&#160;&#160;&#160; &lt;<span style="color: #2b91af">CommandMethod</span>(<span style="color: #a31515">&quot;MyGroup&quot;</span>, <span style="color: #a31515">&quot;MyCommand&quot;</span>, <span style="color: #a31515">&quot;MyCommandLocal&quot;</span>, <span style="color: #2b91af">CommandFlags</span>.Modal)&gt; _</font></p>    <p style="margin: 0px"><font size="2">&#160;&#160;&#160;&#160;&#160;&#160;&#160; <span style="color: blue">Public</span> <span style="color: blue">Sub</span> MyCommand() <span style="color: green">' This method can have any name</span></font></p>    <p style="margin: 0px"><font size="2">&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160; <span style="color: green">' Put your command code here</span></font></p>    <p style="margin: 0px"><font size="2">&#160;&#160;&#160;&#160;&#160;&#160;&#160; <span style="color: blue">End</span> <span style="color: blue">Sub</span></font></p> </div>  <p>A common misunderstanding is thinking that the “MyCommandLocal” string is actual localized command name – someone making that mistake will try to edit that string to create a new localized command name. This string is actually the Id of a string resource in the project, and it’s the string resource that contains the localized commandname. You can see this by clicking the ‘Show All Files’ button in the Visual Studio Solution Explorer and double clicking on the myCommands.resx file.</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b0177438de7e9970d-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="image" border="0" alt="image" src="/assets/image_109151.jpg" width="207" height="182" /></a></p>  <p>This opens up the resource file, where you can edit the resource string:</p>  <p><a href="http://adndevblog.typepad.com/.a/6a0167607c2431970b0177438de7f1970d-pi"><img style="background-image: none; border-bottom: 0px; border-left: 0px; padding-left: 0px; padding-right: 0px; display: inline; border-top: 0px; border-right: 0px; padding-top: 0px" title="image" border="0" alt="image" src="/assets/image_996253.jpg" width="437" height="98" /></a></p>  <p><a href="http://through-the-interface.typepad.com/through_the_interface/2009/06/registering-autocad-commands-with-localized-names-using-net.html" target="_blank">Kean covered this topic back in 2009</a>, and included instructions for creating additional localized resx files to allow your plug-in to support multiple languages. I’m restating it now because we still receive questions on this from time to time. </p>  <p>Once you’ve created your localized plug-in, you can set localized command names in your <a href="http://through-the-interface.typepad.com/through_the_interface/2011/05/adn-devcast-episode-6-autoloader.html" target="_blank">Autoloader</a> PackageContents.xml file too. For example, here is a &lt;Commands/&gt; group defining a &lt;Command/&gt; that has a global and local name:</p>  <blockquote>   <p><font size="2">&lt;Commands GroupName=&quot;<b>ADNPLUGINS</b>&quot;&gt;</font></p>    <p><font size="2">&#160; &lt;Command Local=&quot;<b>BATCHPUBLISH</b>&quot; Global=&quot;<b>BATCHPUBLISH</b>&quot; /&gt;</font></p>    <p><font size="2">&lt;/Commands&gt;</font></p> </blockquote>  <p>You can add additional per local localized names like this:</p>  <blockquote>   <p><font size="2">&lt;Commands GroupName=&quot;<b>ADNPLUGINS</b>&quot;&gt;</font></p>    <p><font size="2">&lt;Command LocalEnu=&quot;<b>BATCHPUBLISH_IN_ENGLISH</b>&quot; LocalDeu=&quot;<b>BATCHPUBLISH_IN_GERMAN</b>&quot; LocalFra=&quot;<b>BATCHPUBLISH_IN_FRENCH</b>&quot; Global=&quot;<b>BATCHPUBLISH</b>&quot; /&gt;</font></p>    <p><font size="2">&lt;/Commands&gt;</font></p> </blockquote>  <p>and so on …</p>
